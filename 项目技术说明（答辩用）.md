## 二、系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                    浏览器（Web 前端）                         │
│  HTML/CSS/JS · Leaflet地图 · Chart.js图表 · noVNC远程桌面    │
└──────────────────┬──────────────────┬────────────────────────┘
                   │ HTTP/REST        │ WebSocket
                   ▼                  ▼
┌──────────────────────────────────────────────────────────────┐
│                  Go 后端（Gin 框架）                          │
│  RESTful API · JWT/Session 认证 · 中间件 · WebSocket Hub     │
│  LLM 航线规划 · 高德地图集成 · 离线检测 · 数据同步引擎       │
├──────────────────────────────────────────────────────────────┤
│               SQLite 数据库（嵌入式）                         │
│  WAL 模式 · 自动迁移 · 外键约束                              │
└──────────────────────────────────────────────────────────────┘
                   ▲
                   │ HTTP POST（GPS/硬件/电池/飞行数据推送）
                   │
┌──────────────────────────────────────────────────────────────┐
│               hw-agent（设备端采集程序）                       │
│  MAVLink 飞控解析 · NMEA GPS串口 · 模拟数据 · IP地理定位      │
│  系统指标采集（CPU/内存/磁盘/温度）                           │
└──────────────────────────────────────────────────────────────┘
```

### 架构特点
- **前后端一体化部署**：Go 使用 `embed` 将前端静态文件编译进单一可执行文件，无需 Nginx
- **嵌入式数据库**：SQLite 零配置、零依赖，适合边缘/机载部署
- **事件驱动实时通信**：WebSocket 广播 GPS 位置变更、报警等事件
- **插件式数据源**：Agent 通过 Provider 接口支持 MAVLink / NMEA / 模拟 三种数据源

## 三、技术栈详解

### 3.1 后端：Go 语言

| 技术 | 用途 | 对应包/文件 |
|------|------|------------|
| **Gin** | HTTP 路由、中间件、JSON 序列化 | `github.com/gin-gonic/gin` |
| **gorilla/websocket** | 实时数据推送（GPS更新、报警） | `github.com/gorilla/websocket` |
| **modernc.org/sqlite** | 纯 Go SQLite 驱动（无 CGO） | `modernc.org/sqlite` |
| **gopsutil** | 跨平台系统指标采集（CPU/内存/磁盘/温度） | `github.com/shirou/gopsutil/v3` |
| **bcrypt** | 用户密码哈希加密 | `golang.org/x/crypto/bcrypt` |
| **embed** | 将前端文件嵌入二进制，单文件部署 | Go 标准库 `embed` |
| **context + signal** | 优雅关机（SIGINT/SIGTERM） | Go 标准库 |
| **goroutine + ticker** | 后台任务：离线检测、系统监控、数据同步 | Go 标准库 |

#### 知识点：为什么选 Go？
- **编译为单一二进制**：无需运行时环境，适合嵌入式/机载部署
- **天然并发**：goroutine 轻量级协程处理多设备并发数据推送
- **交叉编译**：一行命令即可编译 Linux ARM（树莓派/Jetson）目标
- **内存安全**：自带 GC，避免 C/C++ 常见的内存问题

### 3.2 前端：原生 Web 技术

| 技术 | 用途 | 引入方式 |
|------|------|---------|
| **HTML5 / CSS3 / ES6+ JavaScript** | 16 个功能模块页面 | 原生，无框架 |
| **Leaflet.js** | 交互式地图（GPS轨迹、围栏、标记点） | CDN |
| **Leaflet.markerCluster** | 地图标记聚合（同位置多无人机展开） | CDN |
| **Chart.js** | 数据可视化（饼图、折线图、柱状图） | CDN |
| **noVNC (RFB)** | 浏览器内远程桌面（VNC 协议） | 本地 `novnc-rfb.js` |
| **Fetch API + async/await** | 与后端 RESTful API 通信 | 原生 |
| **WebSocket API** | 实时数据推送（GPS 位置更新） | 原生 |
| **HTML5 `<video>`** | 视频监控流播放（HLS/MP4） | 原生 |
| **iframe** | 嵌入式视频播放（B站/YouTube/WebRTC页面） | 原生 |

#### 知识点：为什么不用 React/Vue？
- **零构建依赖**：无需 Node.js、npm、Webpack，降低部署复杂度
- **嵌入友好**：配合 Go `embed` 直接编译进二进制
- **学习/维护成本低**：团队成员可快速上手
- **性能足够**：16 个独立页面（非 SPA），无框架开销

### 3.3 数据库：SQLite

| 特性 | 说明 |
|------|------|
| **WAL 模式** | 写前日志，支持并发读 |
| **自动迁移** | 启动时自动建表 + ALTER TABLE 安全加列 |
| **外键约束** | `PRAGMA foreign_keys=ON`，保证数据完整性 |
| **纯 Go 驱动** | `modernc.org/sqlite`，无需安装 C 编译器 |
| **单文件存储** | 整个数据库是一个 `app.db` 文件，易于备份和迁移 |

#### 主要数据表
- `users` / `sessions` — 用户认证
- `drones` — 无人机注册信息
- `gps_devices` / `gps_history` — GPS 设备与轨迹
- `gps_fence_alerts` — 电子围栏报警
- `battery_records` / `battery_alerts` — 电池监控
- `devices` — 硬件设备
- `missions` / `mission_logs` — 飞行任务
- `flight_plans` — AI 航线规划
- `recordings` — 语音/交互记录
- `alerts` / `logs` — 系统告警与日志

### 3.4 设备端：hw-agent

| 技术 | 用途 |
|------|------|
| **MAVLink 协议解析** | 从 PX4/ArduPilot 飞控读取 GPS、电池、飞行状态 |
| **NMEA 0183 协议解析** | 从 GPS 串口模块或 gpsd 读取定位数据 |
| **IP 地理定位** | 模拟模式下通过公网 API 获取主机大致位置 |
| **Provider 接口模式** | 统一抽象 Simulated / MAVLink / NMEA 三种数据源 |
| **HTTP Client** | 定时向后端 POST 推送 GPS、电池、硬件指标 |

## 四、核心知识点

### 4.1 RESTful API 设计
- **资源导向**：`/api/drones`、`/api/gps/devices`、`/api/gps/devices/:id/history`
- **标准 HTTP 方法**：GET（查询）、POST（创建/操作）、PUT（更新）、DELETE（删除）
- **分页**：`?page=1&page_size=20`
- **过滤**：`?name=xxx&status=在线`
- **统一错误格式**：`{"error": "message"}`

### 4.2 用户认证与安全
- **bcrypt 密码哈希**：不存储明文密码，使用 `golang.org/x/crypto/bcrypt`
- **Session Token 认证**：登录后返回随机 token，存储在 Cookie 中
- **过期清理**：后台 goroutine 定期清理过期 Session
- **速率限制**：中间件实现请求频率控制，防止暴力破解
- **API Token 鉴权**：可选的 Bearer Token 用于 Agent 端认证

### 4.3 WebSocket 实时通信
- **发布-订阅模型**：`WSHub` 管理多个频道（GPS、告警等）
- **事件类型**：`gps_update`、`alert` 等
- **前端监听**：`new WebSocket(ws://host/api/gps/stream)`
- **断线重连**：前端自动检测断开并尝试重连

### 4.4 GPS 与地图技术
- **坐标系**：WGS-84（GPS 原始坐标）
- **Haversine 公式**：计算两点间球面距离（用于电子围栏判定）
- **电子围栏**：圆形围栏，实时判定设备是否越界
- **轨迹记录**：每次 GPS 推送写入 `gps_history`，支持历史轨迹回放
- **Leaflet MarkerCluster**：同位置多设备聚合显示 + 蛛网展开
- **地图可见性控制**：`map_visible` 字段控制设备是否显示在地图上
- **IP 地理定位**：通过 `ip-api.com` 获取主机大致位置（城市级精度 5~50km）

### 4.5 MAVLink 协议
- **用途**：无人机飞控（PX4/ArduPilot）的标准通信协议
- **传输层**：UDP（默认端口 14550）
- **解析消息类型**：
  - `GLOBAL_POSITION_INT`（MSG ID 33）→ GPS 位置
  - `SYS_STATUS`（MSG ID 1）→ 电池电压/电流/电量
  - `HEARTBEAT`（MSG ID 0）→ 飞行模式/状态
- **CRC 校验**：每条消息带 CRC16 校验确保数据完整性

### 4.6 NMEA 0183 协议
- **用途**：GPS 模块标准输出格式
- **常用语句**：
  - `$GPGGA` / `$GNGGA` → 定位信息（经纬度、海拔、精度）
  - `$GPRMC` / `$GNRMC` → 推荐最小定位信息（含速度、航向）
- **数据源**：串口（COM/ttyUSB）或 gpsd TCP 连接

### 4.7 LLM 智能航线规划
- **大语言模型集成**：调用 OpenAI 兼容 API（默认阿里云通义千问 qwen-plus）
- **Prompt Engineering**：构造结构化 JSON prompt，约束 LLM 输出航点列表
- **输入**：起点、目标点、任务动作、约束条件（限速/限高/禁飞区）
- **输出**：航点序列 + 预估距离/时间/电量消耗 + 风险警告
- **流式响应**：支持 SSE（Server-Sent Events）流式返回规划结果

### 4.8 系统监控
- **gopsutil 跨平台采集**：CPU 使用率、内存占用、磁盘容量、系统温度
- **阈值告警**：CPU > 85%、内存 > 85%、磁盘 > 90% 自动生成告警
- **后台定时任务**：`goroutine + time.Ticker` 周期性采集与判定

### 4.9 数据同步引擎
- **多节点同步**：支持主控端与边缘节点间的数据同步
- **增量同步**：基于时间戳的变更检测
- **冲突处理**：最后写入优先（Last-Write-Wins）策略

### 4.10 远程控制
- **VNC（noVNC）**：通过浏览器内 WebSocket→RFB 协议实现远程桌面
- **SSH**：前端模拟终端，通过后端代理建立 SSH 会话
- **设备关联**：远程控制目标绑定到注册的无人机/设备

### 4.11 高德地图集成
- **地理编码**：地址 → 经纬度坐标
- **逆地理编码**：经纬度 → 地址描述
- **用途**：辅助无人机任务规划时的地点输入

## 七、关键设计模式与工程实践

### 7.1 设计模式
| 模式 | 应用场景 |
|------|---------|
| **策略模式（Strategy）** | `DroneDataProvider` 接口统一 Simulated/MAVLink/NMEA 三种数据源 |
| **发布-订阅模式（Pub/Sub）** | `WSHub` 管理 WebSocket 频道广播 |
| **中间件模式（Middleware）** | Gin 中间件链：日志 → 速率限制 → 认证 → 业务处理 |
| **单例模式（Singleton）** | LLM Client、数据库连接、WebSocket Hub |

### 7.2 工程实践
| 实践 | 说明 |
|------|------|
| **Go embed 单文件部署** | 前端 + 后端编译为一个可执行文件 |
| **环境变量配置** | `.env` 文件 + `SC_*` / `LLM_*` 前缀环境变量 |
| **自动数据库迁移** | 启动时 `ALTER TABLE ADD COLUMN`（忽略已存在） |
| **优雅关机** | 捕获 SIGINT/SIGTERM → context cancel → HTTP server shutdown |
| **离线检测** | 后台定时任务检测 15 秒无 GPS 推送 → 标记设备离线 |
| **跨平台编译** | `GOOS=linux GOARCH=arm64 go build` 即可编译机载版本 |

## 九、部署与运行

### 最简启动（开发/演示）
```bash
# 1. 启动平台后端（含前端）
cd project
go run .
# 浏览器访问 http://localhost:8080

# 2.（可选）启动设备端 Agent（模拟模式）
go run ./cmd/agent/ -server http://127.0.0.1:8080 -id SJ-000 -simulate -push-interval 2
```

### 生产编译
```bash
# 编译平台后端（含前端嵌入）
go build -o smartcontrol .

# 编译 Agent（交叉编译 Linux ARM64 示例）
GOOS=linux GOARCH=arm64 go build -o hw-agent ./cmd/agent/

# 运行
./smartcontrol                    # 平台
./hw-agent -server http://... -id drone-01 -mavlink :14550   # 真实飞控
```

## 十、技术亮点总结（答辩用）

1. **全栈 Go 架构**：后端 + Agent 共用一套语言，编译为单文件，零依赖部署
2. **嵌入式数据库**：SQLite 纯 Go 驱动，适合边缘计算/机载环境
3. **多协议设备接入**：MAVLink（飞控）+ NMEA（GPS 串口）+ HTTP 推送，覆盖主流无人机
4. **AI 智能航线规划**：集成大语言模型（通义千问），自然语言描述任务自动生成航点
5. **实时地图可视化**：Leaflet + MarkerCluster + WebSocket，多设备实时位置跟踪
6. **浏览器内远程控制**：noVNC 实现零插件远程桌面，SSH 终端模拟
7. **完整监控告警链路**：GPS 围栏 → 自动检测 → 告警生成 → 前端实时推送
8. **前后端一体化**：Go embed 将 14 万行代码编译为单一可执行文件

# 仿真与智能决策专业知识

> 知识分类：领域专业知识 | 适用场景：仿真建模、CoT决策、推理解释

## 仿真模拟引擎

仿真引擎是无人机系统智能化的重要支撑，不仅服务算法训练，也服务方案验证、风险预演、调参、对抗测试和任务复盘。

### 仿真对象

- 机体动力学与运动学
- 控制指令与执行器响应
- 传感器模型：IMU、GNSS、相机、激光雷达、气压计
- 环境模型：地形、建筑、风场、障碍物、禁飞区
- 通信模型：链路时延、带宽、丢包
- 任务模型：巡检点、目标移动、区域覆盖
- 异常模型：电池衰减、单传感器失真、链路中断、临时限飞

### 仿真三种核心价值

1. **验证规划是否可执行**：在真实飞行前验证航线、约束、电量预算
2. **评估异常工况下系统鲁棒性**：注入故障、极端天气、通信中断等场景
3. **为智能决策算法提供训练与评估数据**：RL训练、策略评估、A/B测试

## 仿真场景设计

仿真不是越复杂越好，而是要有针对性。

- **基准场景**：验证基本功能正确性。标准航线、静态禁飞区、正常天气
- **复杂场景**：验证多约束综合表现。多障碍物、部分链路衰减、多任务竞争
- **极端场景**：暴露系统边界。大风突变、GNSS异常、低电返航、临时禁飞区刷新
- **对抗场景**：测试算法脆弱性。是否过度依赖单一传感器、是否存在脆弱策略、是否能识别异常输入

## 仿真指标体系

仿真结果不能停留在"看起来飞过去了"，应量化以下指标：

| 指标类别 | 具体指标 | 说明 |
| --- | --- | --- |
| 任务表现 | 任务成功率 | 成功完成的任务占比 |
| 任务表现 | 任务完成时间 | 从起飞到完成所有航点 |
| 效率指标 | 总航程与能耗 | 实际航程/理论最短航程 |
| 效率指标 | 轨迹跟踪误差 | 实际轨迹与规划轨迹的偏差 |
| 安全指标 | 最小安全裕度 | 距障碍物的最小距离 |
| 安全指标 | 误入限制区次数 | 应为零 |
| 安全指标 | 返航触发率 | 因异常触发返航的比例 |
| 通信指标 | 通信中断时长 | 累计失联时间 |
| 系统指标 | 告警触发频率 | 每小时告警次数 |
| 系统指标 | 决策耗时 | 从输入到决策输出的延迟 |
| 协同指标 | 资源冲突次数 | 多任务环境下设备/空域冲突 |

## CoT智能决策知识

CoT（Chain-of-Thought）智能决策是"链式推理驱动的任务辅助决策"。其价值不是让模型直接输出结论，而是让模型结合任务目标、设备状态、空域约束、历史经验、当前告警和日志上下文，给出有依据的决策建议。

### CoT适合处理的问题

- 当前任务是否建议继续执行
- 低电量场景下是返航、备降还是继续完成关键航点
- 多个告警同时出现时，哪个优先级更高
- 航线重规划应优先绕开禁飞区还是优先保证图传质量
- 某个异常更可能来自电池衰减、风场变化还是定位漂移
- 同时存在并发任务时，如何重新分配设备

### CoT推理依据材料

高质量CoT需要以下"推理依据材料"支撑：

1. **任务规则**：任务类型对应的执行标准和完成条件
2. **告警优先级矩阵**：不同告警的严重级别和处置优先顺序
3. **飞行安全边界**：电量阈值、风速阈值、高度限制、通信距离
4. **典型处置案例**：历史上类似场景的决策和结果
5. **电池策略**：不同电量水平下的操作建议
6. **应急SOP**：标准化应急操作流程
7. **历史任务复盘**：成功和失败案例的经验总结
8. **人工专家经验**：资深操作员的决策规则和经验判断

### CoT推理步骤模板

完整的CoT推理应包含以下步骤：

1. **态势感知**：收集当前设备状态、环境条件、任务状态、告警信息
2. **问题识别**：明确需要决策的核心问题和关键矛盾
3. **约束分析**：列出所有硬约束（不可违反）和软约束（可权衡）
4. **方案生成**：基于约束和目标，生成2-3个可行方案
5. **风险评估**：对每个方案评估成功概率、失败后果、资源消耗
6. **方案对比**：综合安全性、可行性、任务完成度、资源效率进行排序
7. **决策输出**：给出推荐方案、执行步骤、监控重点、降级条件
8. **应急预案**：推荐方案失败时的备选处置

## 决策解释模板

标准化解释模板让模型回答更稳定、更像工程系统：

### 返航决策解释模板

当前建议：立即返航。
主要依据：剩余电量低于安全返航阈值，且当前位置到备降点距离大于预设安全距离；同时图传链路质量下降，继续执行任务将增大失联风险。
次要依据：风速上升导致单位航程能耗增加，原始能耗模型已不再保守。
备选方案：若任务必须完成，可切换至最近备降点方案，但需人工确认地面环境。
风险提示：继续前飞存在电量不足导致迫降风险。

### 任务继续执行解释模板

当前建议：继续执行任务。
主要依据：剩余电量充足（>60%），返航裕量满足安全标准；当前风速在安全范围内；通信链路稳定。
监控重点：持续关注电量消耗速率和风速变化趋势。
降级条件：若电量下降至40%以下或风速超过6级，立即切换为返航模式。

### 故障处置解释模板

当前建议：暂停任务，悬停等待诊断。
故障现象：[具体症状描述]
可能原因：[原因1(概率)]、[原因2(概率)]、[原因3(概率)]
影响评估：[对飞行安全/任务完成/设备损伤的影响]
处置步骤：1.[步骤1] 2.[步骤2] 3.[步骤3]
预防措施：[后续如何避免此类问题]

### 告警优先级矩阵

| 优先级 | 告警类型 | 处置时限 | 操作要求 |
| --- | --- | --- | --- |
| P0-紧急 | 失控/碰撞风险/电池起火 | 立即 | 自动执行+人工确认 |
| P1-严重 | 低电返航/GNSS丢失/链路中断 | <30秒 | 系统建议+人工决策 |
| P2-警告 | 电池压差/风速接近限值/温度异常 | <5分钟 | 提示+监控 |
| P3-提示 | 载荷异常/轻微偏航/非关键传感器 | 任务结束前 | 记录+后续维护 |

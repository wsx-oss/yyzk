# 系统维护与监控指南

> 知识分类：平台操作知识 | 适用场景：系统维护、硬件监控、数据库管理

## 系统状态监控

平台提供实时系统监控功能（[NAV:monitor]），监控以下指标：

- **CPU 使用率**: 持续超过 80% 需要关注
- **内存占用**: 持续超过 85% 考虑扩容或优化
- **磁盘使用率**: 超过 90% 需要清理旧数据或扩容
- **网络流量**: 监控上下行带宽使用情况

## 阈值告警配置

系统支持通过环境变量配置监控阈值：

- `THRESHOLD_CPU`: CPU 使用率告警阈值（默认 80%）
- `THRESHOLD_MEM`: 内存占用告警阈值（默认 85%）
- `THRESHOLD_DISK`: 磁盘使用率告警阈值（默认 90%）
- `THRESHOLD_INTERVAL`: 检查间隔秒数（默认 30 秒）

超过阈值 10% 以上为严重告警（critical），否则为警告（warning）。

## 数据备份

系统每 24 小时自动备份一次数据库，保留最近 10 个备份。备份文件存储在服务器本地。手动备份可通过备份管理 API 触发。

## 日志管理

系统日志分为以下级别：

- **debug**: 调试信息，正常运行时不记录
- **info**: 正常操作记录
- **warning**: 需要关注的异常情况
- **error**: 错误，需要排查
- **fatal**: 致命错误，系统可能无法正常工作

查看日志请前往日志页面（[NAV:logs]）。AI 巡检会定期检查错误和致命级别日志，并生成通知。

## 硬件设备管理

硬件状态检测模块（[NAV:hardware]）监控连接的硬件设备：

- 设备在线/离线状态
- CPU、内存使用率
- 设备温度
- 网络连接状态

硬件设备超过 15 秒未上报数据会被自动标记为离线。

## 数据同步

多节点部署时，数据同步模块（[NAV:sync]）确保各节点数据一致性。同步策略包括：

- 实时同步：关键数据（无人机状态、GPS 位置）
- 定期同步：非关键数据（日志、历史记录）
- 冲突解决：以最新时间戳为准

## 性能优化建议

1. **数据库优化**: 定期清理过期的 GPS 轨迹和日志数据
2. **连接池**: WebSocket 连接数量控制在合理范围
3. **缓存策略**: 热点查询使用 StatsCache 缓存
4. **批量写入**: GPS 和电池数据使用 WriteBatcher 批量入库
5. **任务池**: 通过 goroutine 池控制并发，避免资源耗尽

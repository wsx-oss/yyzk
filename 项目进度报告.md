# 云翼智控（CloudControl）项目进度报告

> **项目名称**：云翼智控（CloudControl）—— 智能工具人机交互与远程管控系统
> **技术架构**：Go 1.25 + Gin + MySQL/SQLite + Redis + WebSocket + Leaflet + LLM + RAG + RL（Q-Learning）
> **报告周期**：2026 年 1 月 — 2026 年 4 月
> **文档用途**：项目阶段性建设总结与正式申报支撑材料

---

## 报告一：项目立项与基础架构搭建

| 项目               | 内容                       |
| ------------------ | -------------------------- |
| **报告日期** | 2026 年 1 月 15 日         |
| **当前进度** | 10%                        |
| **所处阶段** | 项目立项与基础架构建设阶段 |

### 一、已完成工作

1. **项目需求分析与总体方案设计**：围绕"智能工具人机交互与远程管控"核心场景完成需求梳理与可行性论证，明确平台建设目标、业务边界与核心功能布局。经充分调研，形成涵盖设备管理、飞行任务调度、状态监测、远程控制、智能辅助决策、仿真模拟训练等在内的 **22 个功能模块**总体规划蓝图，并输出需求分析与系统设计说明书。
2. **技术路线选型与系统架构确定**：完成技术选型论证，确定以 `Go 1.25 + Gin` 为后端核心服务框架，以 `Leaflet` 作为地图可视化引擎，以轻量静态 Web 页面为前端呈现载体。数据层采用 `SQLite/MySQL` 双驱动架构，通过 `AdaptSQL` 方言适配层实现 SQL 跨引擎自动转换，为后续从本地开发向云端生产环境的平滑迁移预留工程能力。
3. **基础服务框架搭建完成**：完成系统主入口（`main.go`）、基础路由注册、中间件体系及公共环境变量加载机制建设。集成 Panic 自动恢复、结构化访问日志、CORS 跨域处理、基于 IP 的请求限流等通用安全能力；同步实现用户注册、登录、登出、Token 校验与 `bcrypt` 密码加密机制，初步形成安全可用的访问控制体系。
4. **数据层与初始数据模型建设**：完成 `internal/db` 数据库连接封装、SQL 方言适配（`AdaptSQL`）及数据库迁移引擎设计，建立首批核心业务表结构与初始化迁移逻辑，为后续模块扩展、跨数据库兼容及统一数据治理奠定基础。
5. **核心功能原型验证**：形成无人机管理、GPS 实时位置展示（Leaflet 地图）、电池状态监控、系统资源监控等首批功能模块原型，成功完成基础数据采集、地图可视化、状态信息读取与 WebSocket 实时推送链路的端到端可用性验证。
6. **前端基础框架落地**：完成以 `dashboard.html` 为壳层、以 `iframe` 为模块承载的前端组织架构，抽离 `common.js`、`common.css` 等公共资源文件，建立统一页面布局与样式基线，为后续 22 个功能模块的快速扩展提供可复用前端基础。

### 二、存在问题

- 当前以 SQLite 为主数据库引擎，面对 GPS、电池等高频遥测数据的并发写入时锁竞争问题突出，承载能力存在瓶颈。
- 实时通信能力处于分散实现阶段，各业务模块独立维护 WebSocket 连接，缺乏统一的事件广播与连接生命周期管理机制。
- 前端模块框架虽具备可扩展性，但跨模块消息传递、全局状态共享及统一交互规范尚待建立。

### 三、下一步计划

- 按总体规划有序推进飞行任务、禁飞区、异常报警、维护日志、性能分析等重点业务模块建设，尽快形成平台主体功能框架。
- 设计并建设统一的 WebSocket 广播中心（`wshub`），实现多源实时数据的集中推送与连接治理。
- 开展高频遥测数据场景下的性能预研，引入批量写入、异步处理等优化思路。

### 四、备注

本阶段重点任务已按计划完成，项目已实现从概念设计向工程实现的顺利过渡，基础架构与核心原型均通过验证，具备进入主体功能密集开发阶段的条件。

## 报告二：核心业务模块开发与智能能力初步集成

| 项目               | 内容                                   |
| ------------------ | -------------------------------------- |
| **报告日期** | 2026 年 2 月 15 日                     |
| **当前进度** | 30%                                    |
| **所处阶段** | 主体业务模块开发与智能能力初步集成阶段 |

### 一、已完成工作

1. **飞行任务管理模块全面成型**：完成飞行任务创建流程重构，形成"航线规划 → 任务配置"两步向导式交互。支持动作点选（地图吸附最近航段）与间隔执行两种动作配置模式；实现任务进度反馈、飞行阶段状态机和执行状态追踪，初步建立从航线规划到任务闭环的全流程管理能力。
2. **禁飞区管控与航线安全能力落地**：完成圆形、多边形两类禁飞区的创建管理与 Leaflet 地图可视化叠加展示，并建立禁飞区纠偏引擎（NFZ Correction），在智能航线规划中自动绕行禁飞区域，为飞行安全控制提供刚性边界约束。
3. **大模型智能规划与推理能力接入**：接入 LLM 大语言模型航线规划能力，支持多候选路径方案生成与方案比选；完成 CoT（Chain-of-Thought）思维链分析模块基础建设（`internal/cot/`），面向调度分析、故障诊断、路径优化、任务规划等 6 类场景提供结构化智能推理。
4. **地图与位置服务能力增强**：接入高德地图 API（`internal/amap/`），实现正/逆地理编码与位置解析能力，提升航线规划、任务展示与地理信息处理的精度与交互体验。
5. **运维支撑类业务模块同步推进**：完成异常报警管理（分类/优先级/导入导出统计）、维护操作日志（操作审计/导入导出）、软件更新管理（自动/强制更新发布）、数据同步状态（18 张表全量/增量同步）、性能分析报告（响应时间/吞吐量/错误率/四维图表）等 5 个运维模块的主体建设，平台运维闭环逐步清晰。
6. **远程控制与设备接入能力扩展**：完成 VNC（noVNC 浏览器端）、SSH（xterm.js 终端）、RDP 三类远程连接协议的浏览器端接入，配套实现视频监控流统一查看与语音交互记录管理能力，平台对设备侧的控制与观测能力大幅增强。
7. **实时通信机制统一化改造**：建立 `wshub.go` 统一 WebSocket 事件广播中心，实现 GPS、电池、系统监控等多源实时数据的集中推送与连接生命周期管理，为后续多源数据融合打下基础。
8. **认证与接口规范体系完善**：补齐分页查询（全局 `page` / `page_size` 支持）、会话管理（Token 24h 自动过期）、API Token 鉴权等通用接口能力，平台接口体系逐步规范化。

### 二、存在问题

- LLM 大模型调用尚未建立独立的速率控制（RPM）与优先级调度机制，高并发场景下存在触发外部接口限流的风险。
- GPS、电池等高频遥测数据仍以逐条方式写入数据库，I/O 压力较大，对系统整体吞吐能力形成制约。
- CoT 思维链推理结果以纯文本形式呈现为主，尚缺少图形化、时间线式的可视化展示方式，用户分析效率有待提升。

### 三、下一步计划

- 推进高频数据链路优化，设计并实现批量写入（WriteBatcher）与 WebSocket 推送节流（Throttler）机制，从根本上改善数据库写入压力与实时推送稳定性。
- 完成备份与数据回滚、AI 智能助手、消息通知中心等关键平台支撑模块建设，进一步完善用户使用闭环。
- 启动统一并发任务框架设计，对巡检、智能规划、统计刷新等后台异步任务进行规范化调度与治理。

### 四、备注

本阶段平台主体业务框架已基本建立，22 个功能模块中已有 16 个完成主体开发。项目已由"单模块独立实现"逐步迈入"多模块协同与智能能力融合"阶段，为后续并发治理与高可用能力建设奠定了坚实基础。

## 报告三：并发框架建设与平台服务化能力提升

| 项目               | 内容                                 |
| ------------------ | ------------------------------------ |
| **报告日期** | 2026 年 3 月 10 日                   |
| **当前进度** | 50%                                  |
| **所处阶段** | 并发能力建设与平台服务化体系完善阶段 |

### 一、已完成工作

1. **数据库架构完成生产级升级**：完成从 SQLite 到 MySQL（Aiven 云数据库）的全量数据迁移与适配（编写 `migrate_sqlite_to_mysql.py` 迁移脚本，覆盖 1473+ 行存量数据）。建立基于环境变量 `SC_DB_DRIVER` 切换的双驱动运行机制，补齐连接池参数配置（25 最大连接 / 10 空闲连接）、会话时区统一（`Asia/Shanghai`）策略，显著提升系统在生产场景下的数据承载能力与一致性保障。
2. **统一并发任务框架（TaskPool）建成**：完成 `internal/taskpool/` 并发框架建设，形成 **IO 16 线程 + CPU 4 线程**双池架构，具备 4 级优先级队列（Critical / High / Normal / Low）、每任务独立超时控制（默认 30s）、Panic 自动恢复、故障计数及全局 + 按组指标采集等核心能力。配套实现 `SchedulePeriodic` 命名周期调度器，通过 9 项并发压测用例（含 5000 并发提交、混合压力测试）全部验证通过。
3. **高频数据处理链路优化落地**：基于并发框架配套开发 `WriteBatcher`（GPS 50 条/2s、电池 30 条/3s 批量落库）、`Throttler`（GPS 200ms、电池 500ms WebSocket 推送节流）与 `StatsCache`（6 类业务统计 10–15s 异步刷新）机制，有效降低数据库瞬时写入压力与 WebSocket 推送风暴。
4. **后台任务全面纳入并发框架统一调度**：将阈值告警检测、GPS/设备离线检测、6 个 AI 定时巡检任务、统计缓存刷新等后台任务全部从原始 `go func + ticker` 模式迁移至 TaskPool 调度执行；同时推动 LLM 航线规划进入 CPU 任务池、仿真遥测数据库写入进入 IO 任务池，后台服务运行逻辑更加清晰可控。
5. **备份与数据回滚体系建立**：完成自动备份（24h 周期 + 启动首次触发）、手动备份、按记录恢复、上传 SQL 文件恢复及异步恢复进度追踪能力建设，恢复前自动创建安全备份，平台数据安全保障机制初步健全。
6. **AI 智能助手与消息通知中心上线**：实现 AI 智能助手右下角浮窗，具备多轮上下文对话、快捷指令、知识库问答与模块快速跳转能力；同步完成消息通知中心建设，实现右上角铃铛入口、未读计数实时更新、类型筛选、AI 巡检自动触发通知及批量已读管理。
7. **并发治理可视化监控落地**：开发并发监控仪表盘页面（`concurrency.html`），实现 Worker 池实时状态展示、IO/CPU 池进度条、任务完成/失败趋势折线图、任务组环形分布图及详情表等可视化能力，系统运行态可观测性显著提升。

### 二、存在问题

- 在云端 MySQL 高延迟场景下（跨境网络 RTT > 200ms），并发任务池存在队列饱和风险（曾出现 1024/1024 持续满载），部分后台任务可能出现排队阻塞。
- 仿真快照、强化学习策略、知识库文档、备份文件、语音录音等关键数据仍依赖本地文件系统存储，不利于云端无状态部署与多实例水平扩展。
- 无人机离线等异常场景下，巡检与告警可能触发密集重复通知，对消息中心有效性和用户体验产生负面影响。

### 三、下一步计划

- 针对任务池队列饱和问题，引入 `TrySubmit` 非阻塞提交机制与遥测写入频率调优策略，提高系统在高延迟场景下的稳定性。
- 启动仿真模拟引擎与强化学习训练链路的设计与实现，构建从真实业务到模拟验证再到策略生成的技术闭环。
- 全面推动本地文件数据（知识库、仿真快照、RL 策略、备份 SQL、语音录音）向数据库持久化迁移，为云端部署扫清障碍。

### 四、备注

本阶段标志着项目已由"功能开发驱动"转向"平台能力建设驱动"，系统在并发治理、任务调度、数据链路优化和服务可观测性方面实现了阶段性跨越，平台工程化成熟度显著提升。

## 报告四：仿真引擎与数据全量持久化

| 项目               | 内容                                                 |
| ------------------ | ---------------------------------------------------- |
| **报告日期** | 2026 年 3 月 28 日                                   |
| **当前进度** | 65%                                                  |
| **所处阶段** | 仿真模拟引擎建设、数据全量数据库化与缓存体系集成阶段 |

### 一、已完成工作

1. **仿真模拟引擎主体建设完成**：完成 `internal/simulation/` 仿真引擎核心建设，实现仿真批次创建、启动、停止、删除及多实例并行管理能力。支持巡逻、巡检、配送、测绘、搜救等 5 类任务模板自动生成航点与任务配置；实现多实例防碰撞检测与自动航向/高度避让（`collisionAvoidanceLoop`）、禁飞区地理围栏实时检测（Ray-Casting 点在多边形内判定）、Leaflet 实时地图轨迹展示及 7×24 连续仿真运行机制（含 idle 充电与满电自动重启），显著增强了平台对复杂业务场景的模拟验证能力。
2. **强化学习训练链路打通**：建立基于 Q-Learning 的训练器与经验回放机制（`internal/simulation/rl.go`），设计 **11 维状态空间**（含速度、高度、电量、风速、任务进度、禁飞区状态等），定义完整的奖励函数体系（含禁飞区惩罚、碰撞惩罚、任务完成奖励等）。完成训练状态实时跟踪、训练历史持久化与策略导出接口（`/api/sim/rl/export-policy`），为后续形成"仿真训练 → 策略评估 → 策略应用"闭环奠定基础。
3. **数据全量数据库持久化完成**：将知识库文档、仿真快照、强化学习策略、备份内容、语音录音等历史依赖本地文件系统的数据统一迁移至数据库存储，新增 `knowledge_docs`、`data_store` 等关键表结构。仿真引擎与 RL 训练器通过 `SaveSnapshotFunc` / `LoadSnapshotFunc` / `SavePolicyFunc` / `LoadPolicyFunc` 回调接口实现数据库持久化，备份模块移除全部本地文件 I/O，录音模块实现上传直写数据库 BLOB。数据库表总数从 32 张扩展至 **34 张**，所有变更保持 SQLite / MySQL 双驱动兼容。
4. **Redis 缓存体系集成完成**：完成 `internal/cache/redis.go` 客户端封装（基于 `go-redis/v9`，连接池 20 连接 / 5 空闲），形成会话缓存加速（`session:<token>` 优先读 Redis）、分布式限流（`INCR + EXPIRE` 滑动窗口，500 次/分钟/IP）、统计缓存同步（6 类业务统计写入 Redis）、通知未读数缓存（30s TTL 自动失效）四大应用场景。Redis 不可用时全部功能自动降级为纯内存模式，系统稳定性不受影响。
5. **关键稳定性问题专项修复**：完成任务池饱和修复（新增 `TrySubmit` 非阻塞提交，队列使用率从 1024/1024 降至 0/1024）、仿真实例恢复致命 Bug 修复（Running 状态恢复后 goroutine 冻结问题）、电池仿真精度修复（`float64` 替代整数截断）、MySQL 迁移完整性审计（补全仿真 5 张表 DDL 与索引，确认 34 张表全部对齐），系统连续运行能力与工程可靠性有效提升。
6. **轨迹可视化表达能力提升**：新增 `curve-utils.js` 工具库，实现三次贝塞尔曲线（航线展示）与 Catmull-Rom 样条（实时/历史轨迹平滑）算法，配合弧度自适应策略（短距小弧度 → 长距大弧度、急转弯自动加大曲率），同步应用于 `simulation.html`、`gps.html`、`flight.html` 三个核心地图页面，飞行路径展示真实感与可读性显著增强。
7. **仿真统计看板形成闭环**：完成 `/api/sim/stats` 与仿真前端页面联动，支持仿真趋势图、任务类型分布图、RL 学习曲线等多维度指标展示，提升仿真过程可观测性与训练结果分析效率。

### 二、存在问题

- 在离线或外部依赖不可用场景下，巡检与告警机制仍可能形成集中通知风暴，通知抑制与异常折叠策略有待完善。
- LLM 大模型调用仍缺少独立 RPM 限流控制与细粒度监控指标，多入口并发调用下存在外部接口超限隐患。
- 前端部分静态资源（JS/CSS 库）仍依赖外部 CDN 加载，影响离线部署能力及弱网场景下的加载稳定性。
- 平台视觉设计语言尚未完全统一，页面圆角、毛玻璃效果、交互动效等细节一致性仍有提升空间。

### 三、下一步计划

- 建立离线状态检测与通知冷却抑制机制，从根本上解决离线场景下的通知风暴问题。
- 实现 LLM 独立 RPM 限流（令牌桶）与运行指标监测端点，增强智能能力模块在并发场景下的稳定性。
- 完成 RAG 检索增强生成能力的正式集成，提升 AI 智能助手的知识问答深度与准确度。
- 持续推进前端视觉规范统一与交互体验优化工作。

## 报告五：智能能力增强与阶段性成果整合

| 项目               | 内容                                           |

| **报告日期** | 2026 年 4 月 15 日                             |

### 一、已完成工作

1. **离线状态治理能力显著增强**：围绕无人机离线、数据库不可达、LLM 服务不可用等多类故障场景，建立前后端联动的健康检查机制（`/api/healthz` 返回 `db_reachable` / `redis_reachable` / `llm_reachable` / `patrol_online` / `uptime_s` 五维指标）。巡检模块新增 DB ping + LLM HEAD 预检逻辑，离线时自动跳过巡检；无人机下线通知引入 **5 分钟/台冷却期**，有效消除通知风暴；前端通知铃铛增加离线检测与暂停轮询机制，整体误报与重报率大幅下降。
2. **大模型调用治理体系完善**：基于 `golang.org/x/time/rate` 实现令牌桶 RPM 限流器，支持 `LLM_RPM`（默认 30 次/分钟）与 `LLM_TIMEOUT_SEC`（默认 60s）环境变量配置。新增 `/api/taskpool/llm-rpm` 独立监控端点，返回当前分钟调用量、上限、排队数与平均延迟；前端并发监控页面同步新增 LLM RPM 监控卡片，智能模块运行状态全面可观测。
3. **RAG 检索增强生成能力正式形成**：完成基于 BM25 关键词检索的 RAG 引擎建设（`internal/rag/rag.go`），实现倒排索引构建、文档分块与 TF-IDF 评分排序。知识库从 `knowledge_docs` 数据库表加载（含 20 篇领域文档覆盖平台总览、无人机操作、航线规划、电池安全、仿真指南、RL 策略、应急流程等领域），首次运行自动从 `knowledge_base/` 目录种子导入。AI 智能助手已完成 RAG 深度集成，对话时自动检索相关知识片段注入 Prompt，知识问答准确度显著提升。新增 `/api/ai/rag/search` 与 `/api/ai/rag/stats` 端点。
4. **地图服务体系升级**：完成仿真地图从高德底图向**天地图**体系迁移（支持 `TIANDITU_KEY` 环境变量配置），统一影像底图 + 标注图层加载方式，优化暗色主题适配效果，地图展示质量与全平台视觉一致性进一步提升。
5. **AI 智能助手交互体验优化**：围绕入口图标（科技感设计）、浮窗拖拽能力、毛玻璃视觉效果、窗口位置持久化等方面完成细致优化，智能助手在系统中的可用性与品牌识别度明显增强。
6. **前端毛玻璃视觉效果建设完成**：已在系统前端关键页面中完成毛玻璃（Glassmorphism）视觉效果落地，结合半透明卡片、背景模糊、光影层次与圆角样式优化，显著提升系统界面的科技感、层次感与整体展示品质，为后续统一视觉规范和项目答辩展示提供了更成熟的前端呈现基础。
7. **仿真异常日志表达能力增强**：实现异常日志等级筛选（Info / Warning / Critical）、计数徽标展示、列表自动滚动与异常条目高亮聚焦，运维人员可更高效地识别仿真过程中的风险事件与关键状态变化。
8. **RL 策略导出与任务联动能力打通**：围绕强化学习策略导出结果与仿真批次创建建立联动机制，导出策略可作为新批次创建的先验输入，增强训练成果的可复用性与 sim-to-real 应用闭环。
9. **低电量风险自动识别与处置落地**：在仿真实例中建立低电量异常自动检测机制（≤20% 注入 `low_battery` 警告、≤15% 升级 Critical 并强制返航），电量恢复 >20% 后自动清除异常标签，使仿真风险控制更贴近实际业务逻辑。
10. **登录与门户界面完成统一优化**：通过浮动无人机元素、光晕背景、毛玻璃卡片等视觉设计改造，提升系统登录入口的科技感与展示品质，平台整体视觉调性趋于统一。

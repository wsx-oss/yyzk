# 观察端和被控端的配置方法

> 目标：让未配置的观察端与被控端（全新笔记本/无人机控制终端）完成对本系统远程控制功能的测试与连接。  
> 支持协议：VNC（跨平台图形桌面）、SSH（Linux 终端）、RDP（Windows 远程桌面）  
> 架构：浏览器 → 本系统后端（WebSocket 代理）→ 目标被控端
>
> **新版本说明**：当前版本支持通过 **无人机管理模块** 注册无人机时自动创建关联的远程设备，无需手动添加。也可通过远程桌面控制页面手动新增独立设备。

## 角色定义

| 角色 | 设备 | 主要职责 | 所需配置 |
|------|------|----------|----------|
| **后端主机** | 运行本系统的服务器 | 提供 Web 界面与 VNC/SSH WebSocket 代理 | 启动本系统，确保 8080 端口可访问 |
| **被控端** | 目标主机/无人机控制终端 | 运行远程服务，接受远程控制 | 根据协议安装对应服务（VNC/SSH/RDP）并开放端口 |
| **观察端** | 测试人员浏览器 | 通过 Web 界面连接并控制被控端 | 访问系统页面，输入目标地址进行连接 |

## 网络拓扑与 IP 规划

```text
观察端浏览器 ──(HTTP/WS)──> 后端主机 :8080 ──(TCP)──> 被控端 :5900/22/3389
```

- **后端主机**：`192.168.1.10`（运行本系统）
- **被控端**：`192.168.1.20`（目标主机/无人机控制终端）
- **观察端**：任意可访问后端的浏览器

**关键要求**：

- 后端主机必须能直连被控端的对应端口（VNC: 5900、SSH: 22、RDP: 3389）
- 观察端只需访问后端的 8080 端口
- RDP 连接时，观察端需能直达被控端的 3389 端口（不经过后端代理）

## 后端主机（电脑A）部署与启动

> 说明：后端主机是运行本系统的电脑/服务器，它负责提供 Web 界面（HTTP 8080）以及 VNC/SSH WebSocket 代理（`/api/vnc/ws`、`/api/ssh/ws`）。

### 1. 获取项目代码

- **方式 A：复制文件夹**
  - 将整个项目文件夹复制到电脑A。

- **方式 B：Git 克隆（如有仓库）**
  - 在电脑A上 `git clone <仓库地址>`。

### 2. 进入后端项目目录

在电脑A上打开终端并进入 `project` 目录（该目录内有 `main.go` 与 `go.mod`）：

```bash
cd project
```

### 3. 安装依赖

```bash
go mod tidy
```

### 4. 启动服务

```bash
go run .
```

### 5. 确认启动成功

看到以下输出表示成功：

```text
listening on :8080
```

### 6. 验证后端可访问

在电脑A本机浏览器访问：

- `http://127.0.0.1:8080/api/healthz`

应返回：

```json
{"status":"ok"}
```

如果观察端在另一台机器上，改用：

- `http://<电脑A的IP>:8080/api/healthz`

并确保电脑A的防火墙允许入站访问 8080 端口（或你实际配置的监听端口）。

## 被控端配置—— VNC

> 适用于跨平台图形化远程桌面控制场景。如果被控端是 Linux 终端，推荐使用 SSH；如果是 Windows，也可使用 RDP。

### 被控端 VNC 配置（全新笔记本）

### 1. 安装 VNC 服务

#### Windows（推荐 TightVNC）

1. **下载安装**

   ```powershell
   # 访问 https://www.tightvnc.com/download.php 下载最新版
   # 安装时勾选 Server 和 Viewer，建议安装为服务
   ```

2. **设置密码**
   - 打开 "TightVNC Server" 配置界面
   - 设置 "Primary password"（控制密码）
   - 可选设置 "View-only password"（只读密码）
   - 勾选 "Allow control of this computer via password"

3. **启动服务**
   - 确保 "TightVNC Server" 服务状态为 "Running"
   - 建议设置为开机自启动

#### Linux（Ubuntu/Debian 示例）

```bash
sudo apt update
sudo apt install -y tigervnc-standalone-server
vncpasswd  # 设置 VNC 密码
vncserver :1 -geometry 1920x1080 -depth 24  # 启动 VNC 服务
```

#### macOS

1. 系统设置 → 共享 → 屏幕共享
2. 勾选 "屏幕共享"
3. 点击 "电脑设置" 设置 VNC 密码

### 2. 防火墙配置

#### Windows 防火墙（以管理员身份运行 PowerShell）

```powershell
# 允许 VNC 5900 端口入站
New-NetFirewallRule -DisplayName "VNC 5900" -Direction Inbound -Protocol TCP -LocalPort 5900 -Action Allow

# 验证规则是否生效
Get-NetFirewallRule -DisplayName "VNC 5900"
```

#### Linux 防火墙（ufw 示例）

```bash
sudo ufw allow 5900/tcp
sudo ufw reload
```

#### macOS 防火墙

1. 系统偏好设置 → 安全性与隐私 → 防火墙
2. 点击 "防火墙选项"
3. 添加 "屏幕共享" 或允许传入连接

### 3. 第三方安全软件配置

- **杀毒软件**：临时关闭或添加 5900 端口例外
- **企业防护**：在安全策略中放行 VNC 连接
- **VPN/代理**：确保不会拦截本地网络连接

### 4. 验证 VNC 服务

```powershell
# Windows
netstat -ano | findstr :5900

# Linux/macOS
ss -lnt | grep 5900
```

应看到类似 `TCP 0.0.0.0:5900` 的监听状态。

## 被控端配置—— SSH（Linux）

> 适用于 Linux/macOS 系统的命令行终端远程控制场景。系统通过浏览器内 xterm.js 终端实现 SSH 连接。

### 1. 确认 SSH 服务已安装并启动

#### Ubuntu/Debian

```bash
sudo apt update
sudo apt install -y openssh-server
sudo systemctl enable ssh
sudo systemctl start ssh
```

#### CentOS/RHEL

```bash
sudo yum install -y openssh-server
sudo systemctl enable sshd
sudo systemctl start sshd
```

#### macOS

macOS 自带 SSH 服务，只需在"系统设置 → 通用 → 共享 → 远程登录"中开启即可。

### 2. SSH 防火墙配置

#### Linux（ufw 示例）

```bash
sudo ufw allow 22/tcp
sudo ufw reload
```

#### Linux（firewalld 示例）

```bash
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
```

### 3. 验证 SSH 服务状态

```bash
ss -lnt | grep 22
```

应看到类似 `0.0.0.0:22` 的监听状态。

### 4. 在本系统中添加 SSH 设备

1. 登录系统，进入"远程桌面控制"模块
2. 点击"新增连接"，连接类型选择 **SSH**
3. 填写被控端 IP 地址、端口号 **22**、Linux 系统用户名
4. 点击"连接"时输入 Linux 系统密码
5. 系统在新标签页打开 xterm.js 终端，即可执行命令

> **提示**：如果已通过无人机管理模块注册了无人机并选择 SSH 协议，系统会自动创建对应的远程设备，无需手动添加。

## 被控端配置—— RDP（Windows）

> 适用于 Windows 系统的图形化远程桌面控制场景。系统生成 `.rdp` 文件，由本地 Windows 远程桌面客户端（mstsc.exe）打开。

### 1. 开启 Windows 远程桌面

1. 打开"设置 → 系统 → 远程桌面"
2. 开启"启用远程桌面"开关
3. 记录当前电脑名称（可选，用于连接标识）

> **注意**：Windows 10/11 **家庭版**不支持 RDP 服务端，需使用**专业版**或**企业版**。

### 2. RDP 防火墙配置

```powershell
# 允许 RDP 3389 端口入站（以管理员身份运行 PowerShell）
New-NetFirewallRule -DisplayName "RDP 3389" -Direction Inbound -Protocol TCP -LocalPort 3389 -Action Allow

# 验证规则是否生效
Get-NetFirewallRule -DisplayName "RDP 3389"
```

### 3. 验证 RDP 服务状态

```powershell
netstat -ano | findstr :3389
```

应看到类似 `TCP 0.0.0.0:3389` 的监听状态。

### 4. 在本系统中添加 RDP 设备

1. 登录系统，进入"远程桌面控制"模块
2. 点击"新增连接"，连接类型选择 **RDP**
3. 填写被控端 IP 地址、端口号 **3389**、Windows 登录用户名
4. 点击"连接"后，系统自动下载 `.rdp` 文件
5. 双击 `.rdp` 文件，Windows 自动调用 mstsc.exe 打开远程桌面
6. 在弹出的窗口中输入 Windows 登录密码即可连接

> **提示**：如果已通过无人机管理模块注册了无人机并选择 RDP 协议，系统会自动创建对应的远程设备，无需手动添加。

## 观察端配置（浏览器）

### 1. 基本要求

- **浏览器**：Chrome、Firefox、Edge 等现代浏览器
- **网络**：能访问后端主机的 8080 端口
- **插件**：无需额外插件（使用 HTML5 WebSocket）

### 2. 访问方式

#### 方式一：VNC 直接连接页（推荐快速测试）

1. 打开浏览器访问：`http://<后端IP>:8080/app/vnc.html`
2. 在目标地址输入框中填入：`<被控端IP>:5900`
3. 点击 "连接" 按钮
4. 如有密码弹窗，输入被控端设置的 VNC 密码

#### 方式二：通过设备管理页（推荐正式使用）

1. 访问：`http://<后端IP>:8080/app/login.html` 登录系统
2. 进入 "远程桌面控制" 模块
3. 点击 "新增连接"，填写设备信息：
   - 设备名称：自定义（如 "测试笔记本"、"无人机-01"）
   - IP 地址：被控端 IP
   - 连接类型：选择 **VNC** / **SSH** / **RDP**
   - 端口号：VNC 默认 5900、SSH 默认 22、RDP 默认 3389
   - 用户名：目标设备的系统用户名
4. 在设备列表中点击 "连接"：
   - **VNC**：新标签页打开 noVNC 远程桌面
   - **SSH**：弹出密码框后新标签页打开 xterm.js 终端
   - **RDP**：自动下载 `.rdp` 文件，双击用 mstsc.exe 打开

#### 方式三：通过无人机管理自动创建（推荐无人机场景）

1. 进入 "无人机管理" 模块，注册无人机时填写 IP、端口、协议等信息
2. 系统自动在 "远程桌面控制" 模块创建对应设备
3. 直接在设备列表中点击 "连接" 即可

### 3. 连接状态说明

| 状态 | 含义 | 操作建议 |
|------|------|----------|
| 未连接 | 初始状态 | 检查目标地址格式 |
| 正在连接 | WebSocket 建立中 | 等待或检查网络 |
| ✓ 已连接 | 连接成功 | 可开始远程控制 |
| ✗ 已断开 | 连接中断 | 检查网络或远程服务 |

### 4. 功能说明

- **VNC**：鼠标键盘完全控制被控端桌面，自动适应浏览器窗口大小
- **SSH**：浏览器内终端，支持 256 色显示、窗口自适应、光标闪烁等
- **RDP**：通过系统自带客户端连接，支持全屏、高分辨率
- **断开连接**：点击 "断开" 按钮或关闭页面

## 连通性验证流程

### 1. 后端 → 被控端验证

在后端主机上执行以下命令，根据使用的协议选择对应端口：

#### Windows 后端

```powershell
# VNC
Test-NetConnection 192.168.1.20 -Port 5900
# SSH
Test-NetConnection 192.168.1.20 -Port 22
# RDP
Test-NetConnection 192.168.1.20 -Port 3389
```

成功应显示：`TcpTestSucceeded : True`

#### Linux/macOS 后端

```bash
# VNC
nc -vz 192.168.1.20 5900
# SSH
nc -vz 192.168.1.20 22
# RDP
nc -vz 192.168.1.20 3389
```

成功应显示连接成功信息。

### 2. 观察端 → 后端验证

在观察端浏览器中：

1. 访问：`http://<后端IP>:8080/api/healthz`
2. 应返回：`{"status":"ok"}`

### 3. 完整链路测试

#### VNC 链路

1. 观察端访问 `/app/vnc.html`
2. 输入 `<被控端IP>:5900`
3. 打开浏览器开发者工具，查看 Network 标签
4. 应看到 WebSocket 连接到 `/api/vnc/ws`，状态为 101 Switching Protocols

#### SSH 链路

1. 在设备管理页添加 SSH 设备并点击 "连接"
2. 输入密码后应在新标签页打开终端
3. Network 标签应看到 WebSocket 连接到 `/api/ssh/ws`

#### RDP 链路

1. 在设备管理页添加 RDP 设备并点击 "连接"
2. 应自动下载 `.rdp` 文件
3. 双击文件后应弹出 Windows 远程桌面连接窗口

## 常见报错与排查

| 错误现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| **VNC 连接失败/502 错误** | 后端无法直连被控端 5900 | 在后端执行 `Test-NetConnection` 检查连通性 |
| **VNC 安全验证失败** | VNC 密码错误或不兼容 | 重新设置 VNC 密码，使用标准 VNC 软件 |
| **SSH 认证失败** | 用户名或密码错误 | 确认使用被控端的系统用户名和密码 |
| **SSH 连接被拒绝** | SSH 服务未启动或端口未开放 | 检查 `sshd` 服务状态和防火墙规则 |
| **RDP 文件下载后无法连接** | 被控端未开启远程桌面或防火墙拦截 | 检查 Windows 远程桌面设置和 3389 端口 |
| **RDP 提示"家庭版不支持"** | Windows 家庭版无 RDP 服务端 | 升级到专业版/企业版，或改用 VNC |
| **连接超时** | 防火墙或杀毒软件拦截 | 检查防火墙规则，临时关闭安全软件 |
| **noVNC 库加载失败** | CDN 被拦截或网络问题 | 换网络环境或使用 `/app/vnc_simple.html` |
| **一直"正在连接..."** | 网络丢包或远程服务异常 | 重启对应服务，检查网络质量 |
| **端口被占用** | 其他程序占用对应端口 | 使用 `netstat` 查找占用进程并停止 |

## 快速验证流程（推荐顺序）

### 步骤 1：被控端配置（根据协议选择）

**VNC 被控端（Windows）：**

```powershell
# 1. 安装 TightVNC（下载安装包）
# 2. 设置 VNC 密码
# 3. 启动 VNC 服务
# 4. 开放防火墙
New-NetFirewallRule -DisplayName "VNC 5900" -Direction Inbound -Protocol TCP -LocalPort 5900 -Action Allow
# 5. 验证监听
netstat -ano | findstr :5900
```

**SSH 被控端（Linux）：**

```bash
sudo apt install -y openssh-server && sudo systemctl enable ssh && sudo systemctl start ssh
sudo ufw allow 22/tcp && sudo ufw reload
ss -lnt | grep 22
```

**RDP 被控端（Windows）：**

```powershell
# 1. 设置 → 系统 → 远程桌面 → 启用远程桌面
# 2. 开放防火墙
New-NetFirewallRule -DisplayName "RDP 3389" -Direction Inbound -Protocol TCP -LocalPort 3389 -Action Allow
# 3. 验证监听
netstat -ano | findstr :3389
```

### 步骤 2：后端验证

```powershell
# 检查到被控端的连通性（根据协议选择端口）
Test-NetConnection 192.168.1.20 -Port 5900  # VNC
Test-NetConnection 192.168.1.20 -Port 22    # SSH
Test-NetConnection 192.168.1.20 -Port 3389  # RDP
```

### 步骤 3：观察端连接

**VNC：**

1. 访问：`http://192.168.1.10:8080/app/vnc.html`
2. 输入：`192.168.1.20:5900`，点击连接，输入密码

**SSH：**

1. 登录系统，进入远程桌面控制模块
2. 新增 SSH 设备，点击连接，输入密码

**RDP：**

1. 登录系统，进入远程桌面控制模块
2. 新增 RDP 设备，点击连接，双击下载的 `.rdp` 文件

## 一键检查脚本（被控端 Windows）

以管理员身份在 PowerShell 中执行：

```powershell
Write-Host "=== 远程服务配置检查 ===" -ForegroundColor Green

# 检查 VNC 服务
Write-Host "1. 检查 VNC 服务状态..."
Get-Service | Where-Object {$_.Name -like "*tightvnc*" -or $_.DisplayName -like "*vnc*"} | Format-Table Name, Status, DisplayName

# 检查端口监听
Write-Host "2. 检查关键端口监听..."
Write-Host "  VNC (5900):"
netstat -ano | findstr :5900
Write-Host "  RDP (3389):"
netstat -ano | findstr :3389

# 检查防火墙规则
Write-Host "3. 检查防火墙规则..."
Get-NetFirewallRule -DisplayName "VNC 5900" -ErrorAction SilentlyContinue | Format-Table DisplayName, Enabled, Direction

# 测试本地连接

Write-Host "=== 检查完成 ===" -ForegroundColor Green
```

## 故障排查清单

如果连接失败，请按以下顺序检查：

1. **被控端**
   - [ ] 对应远程服务已启动（VNC Server / SSH 服务 / Windows 远程桌面）
   - [ ] 防火墙已开放对应端口（VNC: 5900、SSH: 22、RDP: 3389）
   - [ ] 第三方安全软件已放行
   - [ ] IP 地址正确

2. **后端主机**
   - [ ] 能访问被控端的对应端口
   - [ ] 本系统服务正常运行
   - [ ] 8080 端口可访问

3. **观察端**
   - [ ] 浏览器能访问后端 8080 端口
   - [ ] 输入的目标地址和端口格式正确
   - [ ] 网络连接稳定
   - [ ] RDP 连接时能直达被控端 3389 端口

4. **网络环境**
   - [ ] 同一局域网或路由配置正确
   - [ ] 无 VPN/代理干扰
   - [ ] 带宽充足

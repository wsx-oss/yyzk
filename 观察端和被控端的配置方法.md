## 角色定义

| 角色 | 设备 | 主要职责 | 所需配置 |
|------|------|----------|----------|
| **后端主机** | 运行本系统的服务器 | 提供 Web 界面与 VNC WebSocket 代理 | 启动本系统，确保 8080 端口可访问 |
| **被控端** | 全新笔记本（目标主机） | 运行 VNC 服务，接受远程控制 | 安装 VNC 服务、配置防火墙、开放 5900 端口 |
| **观察端** | 测试人员浏览器 | 通过 Web 界面连接并控制被控端 | 访问系统页面，输入目标地址进行连接 |

## 网络拓扑与 IP 规划

```text
观察端浏览器 ──(HTTP/WS)──> 后端主机 :8080 ──(TCP)──> 被控端 :5900
```

- **后端主机**：`192.168.1.10`（运行本系统）
- **被控端**：`192.168.1.20`（全新笔记本）
- **观察端**：任意可访问后端的浏览器

**关键要求**：

- 后端主机必须能直连被控端的 5900 端口
- 观察端只需访问后端的 8080 端口

## 后端主机（电脑A）部署与启动

> 说明：后端主机是运行本系统的电脑/服务器，它负责提供 Web 界面（HTTP 8080）以及 VNC WebSocket 代理（`/api/vnc/ws`）。

### 1. 获取项目代码

- **方式 A：复制文件夹**
  - 将整个项目文件夹复制到电脑A。

- **方式 B：Git 克隆（如有仓库）**
  - 在电脑A上 `git clone <仓库地址>`。

### 2. 进入后端项目目录

在电脑A上打开终端并进入 `project` 目录（该目录内有 `main.go` 与 `go.mod`）：

```bash
cd project
```

### 3. 安装依赖

```bash
go mod tidy
```

### 4. 启动服务

```bash
go run .
```

### 5. 确认启动成功

看到以下输出表示成功：

```text
listening on :8080
```

### 6. 验证后端可访问

在电脑A本机浏览器访问：

- `http://127.0.0.1:8080/api/healthz`

应返回：

```json
{"status":"ok"}
```

如果观察端在另一台机器上，改用：

- `http://<电脑A的IP>:8080/api/healthz`

并确保电脑A的防火墙允许入站访问 8080 端口（或你实际配置的监听端口）。

## 被控端配置（全新笔记本）

### 1. 安装 VNC 服务

#### Windows（推荐 TightVNC）

1. **下载安装**

   ```powershell
   # 访问 https://www.tightvnc.com/download.php 下载最新版
   # 安装时勾选 Server 和 Viewer，建议安装为服务
   ```

2. **设置密码**
   - 打开 "TightVNC Server" 配置界面
   - 设置 "Primary password"（控制密码）
   - 可选设置 "View-only password"（只读密码）
   - 勾选 "Allow control of this computer via password"

3. **启动服务**
   - 确保 "TightVNC Server" 服务状态为 "Running"
   - 建议设置为开机自启动

#### Linux（Ubuntu/Debian 示例）

```bash
sudo apt update
sudo apt install -y tigervnc-standalone-server
vncpasswd  # 设置 VNC 密码
vncserver :1 -geometry 1920x1080 -depth 24  # 启动 VNC 服务
```

#### macOS

1. 系统设置 → 共享 → 屏幕共享
2. 勾选 "屏幕共享"
3. 点击 "电脑设置" 设置 VNC 密码

### 2. 防火墙配置

#### Windows 防火墙（以管理员身份运行 PowerShell）

```powershell
# 允许 VNC 5900 端口入站
New-NetFirewallRule -DisplayName "VNC 5900" -Direction Inbound -Protocol TCP -LocalPort 5900 -Action Allow

# 验证规则是否生效
Get-NetFirewallRule -DisplayName "VNC 5900"
```

#### Linux 防火墙（ufw 示例）

```bash
sudo ufw allow 5900/tcp
sudo ufw reload
```

#### macOS 防火墙

1. 系统偏好设置 → 安全性与隐私 → 防火墙
2. 点击 "防火墙选项"
3. 添加 "屏幕共享" 或允许传入连接

### 3. 第三方安全软件配置

- **杀毒软件**：临时关闭或添加 5900 端口例外
- **企业防护**：在安全策略中放行 VNC 连接
- **VPN/代理**：确保不会拦截本地网络连接

### 4. 验证 VNC 服务

```powershell
# Windows
netstat -ano | findstr :5900

# Linux/macOS
ss -lnt | grep 5900
```

应看到类似 `TCP 0.0.0.0:5900` 的监听状态。

## 观察端配置（浏览器）

### 1. 基本要求

- **浏览器**：Chrome、Firefox、Edge 等现代浏览器
- **网络**：能访问后端主机的 8080 端口
- **插件**：无需额外插件（使用 HTML5 WebSocket）

### 2. 访问方式

#### 方式一：直接连接页（推荐测试）

1. 打开浏览器访问：`http://<后端IP>:8080/app/vnc.html`
2. 在目标地址输入框中填入：`<被控端IP>:5900`
3. 点击 "连接" 按钮
4. 如有密码弹窗，输入被控端设置的 VNC 密码

#### 方式二：通过设备管理页（需登录）

1. 访问：`http://<后端IP>:8080/app/login.html` 登录系统
2. 进入 "远程桌面控制" 模块
3. 点击 "新增远程桌面"：
   - 设备名称：自定义（如 "测试笔记本"）
   - IP 地址：`192.168.1.20:5900`
   - 连接协议：选择 "VNC"
4. 在设备列表中点击 "连接"

### 3. 连接状态说明

| 状态 | 含义 | 操作建议 |
|------|------|----------|
| 未连接 | 初始状态 | 检查目标地址格式 |
| 正在连接 | WebSocket 建立中 | 等待或检查网络 |
| ✓ 已连接 | 连接成功 | 可开始远程控制 |
| ✗ 已断开 | 连接中断 | 检查网络或 VNC 服务 |

### 4. 功能说明

- **鼠标键盘**：完全控制被控端
- **缩放适应**：自动适应浏览器窗口大小
- **断开连接**：点击 "断开" 按钮或关闭页面

## 连通性验证流程

### 1. 后端 → 被控端验证

在后端主机上执行以下命令：

#### Windows 后端

```powershell
Test-NetConnection 192.168.1.20 -Port 5900
```

成功应显示：`TcpTestSucceeded : True`

#### Linux/macOS 后端

```bash
nc -vz 192.168.1.20 5900
# 或
telnet 192.168.1.20 5900
```

成功应显示连接成功信息。

### 2. 观察端 → 后端验证

在观察端浏览器中：

1. 访问：`http://<后端IP>:8080/api/healthz`
2. 应返回：`{"status":"ok"}`

### 3. 完整链路测试

1. 观察端访问 `/app/vnc.html`
2. 输入 `<被控端IP>:5900`
3. 打开浏览器开发者工具，查看 Network 标签
4. 应看到 WebSocket 连接到 `/api/vnc/ws`，状态为 101 Switching Protocols

## 常见报错与排查

| 错误现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| **连接失败/502 错误** | 后端无法直连被控端 5900 | 在后端执行 `Test-NetConnection` 检查连通性 |
| **安全验证失败** | VNC 密码错误或不兼容 | 重新设置 VNC 密码，使用标准 VNC 软件 |
| **连接超时** | 防火墙或杀毒软件拦截 | 检查防火墙规则，临时关闭安全软件 |
| **noVNC 库加载失败** | CDN 被拦截或网络问题 | 换网络环境或使用 `/app/vnc_simple.html` |
| **一直"正在连接..."** | 网络丢包或 VNC 服务异常 | 重启 VNC 服务，检查网络质量 |
| **端口被占用** | 其他程序占用 5900 | 使用 `netstat` 查找占用进程并停止 |

## 附录：关键命令与端口

### 端口说明

| 端口 | 用途 | 说明 |
|------|------|------|
| 8080 | 后端 Web 服务 | 观察端访问界面 |
| 5900 | VNC 服务 | 被控端监听端口 |
| 5901+ | 多显示器 VNC | 可选配置 |

### 关键端点

- **健康检查**：`http://<后端IP>:8080/api/healthz`
- **VNC 直连页**：`http://<后端IP>:8080/app/vnc.html`
- **设备管理页**：`http://<后端IP>:8080/app/modules/remote.html`
- **WebSocket 代理**：`ws://<后端IP>:8080/api/vnc/ws?target=<IP>:5900`

### 常用命令

```powershell
# Windows 检查端口监听
netstat -ano | findstr :5900

# Windows 检查防火墙规则
Get-NetFirewallRule -DisplayName "VNC 5900"

# Windows 测试连通性
Test-NetConnection <IP> -Port 5900

# Linux/macOS 检查端口监听
ss -lnt | grep 5900

# Linux/macOS 测试连通性
nc -vz <IP> 5900
```

## 一键检查脚本（被控端 Windows）

以管理员身份在 PowerShell 中执行：

```powershell
Write-Host "=== VNC 配置检查 ===" -ForegroundColor Green

# 检查 VNC 服务
Write-Host "1. 检查 VNC 服务状态..."
Get-Service | Where-Object {$_.Name -like "*tightvnc*" -or $_.DisplayName -like "*vnc*"} | Format-Table Name, Status, DisplayName

# 检查端口监听
Write-Host "2. 检查 5900 端口监听..."
netstat -ano | findstr :5900

# 检查防火墙规则
Write-Host "3. 检查防火墙规则..."
Get-NetFirewallRule -DisplayName "VNC 5900" | Format-Table DisplayName, Enabled, Direction

# 测试本地连接
Write-Host "4. 测试本地 VNC 连接..."
Test-NetConnection 127.0.0.1 -Port 5900

Write-Host "=== 检查完成 ===" -ForegroundColor Green
```

## 故障排查清单

如果连接失败，请按以下顺序检查：

1. **被控端**
   - [ ] VNC 服务已启动
   - [ ] 防火墙已开放 5900 端口
   - [ ] 第三方安全软件已放行
   - [ ] IP 地址正确

2. **后端主机**
   - [ ] 能访问被控端 5900 端口
   - [ ] 本系统服务正常运行
   - [ ] 8080 端口可访问

3. **观察端**
   - [ ] 浏览器能访问后端 8080 端口
   - [ ] 输入的目标地址格式正确
   - [ ] 网络连接稳定

4. **网络环境**
   - [ ] 同一局域网或路由配置正确
   - [ ] 无 VPN/代理干扰
   - [ ] 带宽充足

---

> **提示**：完成配置后，建议保存各设备的 IP 地址和 VNC 密码，便于后续测试使用。

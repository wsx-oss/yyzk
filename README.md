# 智能工具人机交互与远程操控系统

本 README 为本仓库的**团队协作开发规范**（面向课程设计/学生团队）。请所有成员在开始开发前通读并严格遵守，以避免协作混乱。

## 1. 项目介绍

**项目名称**：智能工具人机交互与远程操控系统

**项目概述**：

- 后端位于 `project/`，使用 Go + Gin 提供 API 服务，SQLite 持久化。
- 前端静态页面位于 `project/web/`，由后端内嵌静态资源并通过 `/app` 提供访问。
- 主要能力包含：用户认证、状态监控、音频管理、远程桌面（VNC WebSocket 代理）等。

**目标**：在多人协作下持续迭代功能，确保代码质量、提交可追溯、分支清晰、可稳定交付。

## 2. 团队分工原则

- **模块负责制**：每个功能模块设 1 名负责人（Owner），负责需求拆分、接口约定、代码评审合入。
- **接口优先**：跨模块协作时先对齐接口（API/数据结构/文件格式），再并行开发。
- **小步快跑**：一次提交只做一件事；一次 PR（合并请求）只解决一个主题。
- **评审必需**：合入必须需经过至少一人评审通过，由队长统一提交，确保能正常运行且无报错。
- **问题先同步**：发现阻塞问题（环境、接口、数据、冲突）先在群内同步，再继续开发。

## 3. Git 提交规范（强制）

### 3.1 提交信息格式（强制统一）

**所有提交信息必须使用如下格式：**

`M.D - 已完成的功能/修复内容`

- `M`：里程碑/大版本号（例如第 2 次阶段验收）
- `D`：该里程碑下的功能点序号（例如第 7 个功能点）

**示例（必须照抄这种风格）：**

- `2.7 - 已完成远程控制`
- `1.3 - 已修复登录态丢失问题`
- `3.1 - 已完成音频列表分页接口`

### 3.2 提交粒度与要求

- **一次提交只包含一个主题**（功能/修复/重构之一）。
- **提交前自检**：能编译/能运行/不破坏主流程。
- **避免提交无关文件**：如本地 IDE 配置、临时文件、个人笔记（除非团队约定）。

## 4. 代码规范

### 5.1 通用规范

- **命名清晰**：变量/函数表达含义，避免 `a/b/tmp/test` 等。
- **错误要处理**：不得吞掉错误；返回给前端的错误信息需可理解。
- **保持一致性**：同一模块内风格一致优先。

### 5.2 Go（后端）规范（适用于 `project/`）

- 代码格式：提交前运行 `gofmt`（必须）。
- 包结构：优先按 `internal/<领域>/...` 组织，避免在根目录堆文件。
- 路由与处理器：路由注册集中，handler 单一职责。
- 数据库访问：SQL 必须可读，避免在同一函数堆大量拼接逻辑。

### 5.3 前端（`project/web/`）规范

- 文件命名：小写 + 短横线，例如 `device-list.js`、`dashboard.html`。
- 不引入大体量依赖（如需引入，先在群内说明理由）。
- 与后端接口变更必须同步更新对应页面，并在 PR 描述中写清影响范围。

## 5. 文件目录规范

### 6.1 仓库根目录约定

- `project/`：核心可运行项目（后端 + 前端静态资源）。
- 其余 `*.md / *.docx / *.pdf / *.txt`：立项/说明/使用指南等资料（尽量不在核心代码目录内堆放）。

### 6.2 核心项目目录（参考）

以当前仓库实际结构为准（示意）：

```text
project/
  main.go
  go.mod
  internal/
    db/
    handlers/
    middleware/
    monitor/
  web/
  data/
```

### 6.3 新增文件/目录规则

- 新增模块优先放到 `project/internal/<模块>/...`。
- 不在仓库根目录新增零散代码文件（如确需新增，请先与队长确认）。

## 6. 注意事项

- **合并前先拉取**：合并/推送前先 `git pull`，避免大规模冲突。
- **冲突要及时解决**：冲突未及时解决，立即同步求助。
- **二进制大文件**：禁止频繁提交 `exe/zip/mp4` 等大文件。

## 7. 禁止行为（必须遵守）

- **禁止**绕过规范提交信息格式。
- **禁止**直接向 `main` 推送（含 Web IDE 直接编辑 main）。
- **禁止**在公共分支执行 `git push --force`。
- **禁止**把个人临时文件、无关生成物提交进仓库（如 `node_modules/`、本地数据库、临时日志等）。
- **禁止**未通知他人就改动公共接口（API/字段/URL），导致他人分支不可用。

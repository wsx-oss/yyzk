> 基于与学长的聊天记录 + 项目现有代码整理，目标：**让你的 CloudControl 平台收到真实飞控数据**。

> **补充说明（当前最新情况）：** 你们的后端已经部署到服务器，域名已可访问。下面文档中的“平台地址”优先使用你们现在实际可访问的线上地址。当前统一使用：`https://yyzk.online`。

## 一、整体数据流（先搞清楚数据怎么走的）

```
方案A（推荐，最简单）—— USB 直连：

  飞控(Pixhawk)
       │  USB 线
       ↓
  你的笔记本电脑
       │  识别为 COM 口（如 COM3）
       │  运行 hw-agent（读 COM 口的 MAVLink 数据）
       │  运行后端 Go 服务（端口 8080）
       ↓
  浏览器打开 https://yyzk.online


方案B —— 4G 模块 + 网线：

  飞控(Pixhawk)
       │  串口
       ↓
  4G 模块（装在无人机上，运行 uav_to_net3readpro.c 转发程序）
       │  网线（RJ45）
       ↓
  你的笔记本电脑
       │  接收 TCP 数据 → 需要一个中转脚本转为 UDP
       │  运行 hw-agent（监听 UDP 14550）
       │  运行后端 Go 服务（端口 8080）
       ↓
  浏览器打开 https://yyzk.online
```

**学长原话**："就是 4G 模块和 USB 线这两种方式"、"用电脑跑平台的话用，用网线连起来用局域网也行"

## 二、方案 A：USB 直连飞控（推荐首次测试用这个）

> 最简单，不需要 4G 模块，一根 USB 线搞定。适合桌面测试（飞控不需要装在无人机上）。

### 需要准备的东西

| 物品 | 说明 |
|------|------|
| 飞控板（Pixhawk） | 实验室应该有，巴掌大的绿色/黑色电路板 |
| USB 数据线 | Micro-USB 或 USB-C（看飞控型号），连飞控到电脑 |
| 你的笔记本电脑 | 已装好 Go 环境 |
| （可选）QGroundControl | 用来确认飞控是否正常工作 |

### 步骤 1：USB 连接飞控

1. 找到飞控板上的 **USB 接口**（通常在板子侧面，标注 USB 或有 USB 图标）
2. 用 USB 线一头插飞控，一头插你的笔记本
3. 飞控通电后会亮灯（有的会闪红绿灯）

### 步骤 2：确认 Windows 识别了飞控

1. 右键「此电脑」→「管理」→「设备管理器」→ 展开「端口(COM 和 LPT)」
2. 你应该能看到类似 **"USB Serial Device (COM3)"** 或 **"PX4 FMU (COM5)"**
3. **记住这个 COM 口号**（如 COM3），后面要用

> ⚠️ 如果没有出现 COM 口：
> - 换一根 USB 线试试（有的线只能充电不能传数据）
> - 可能需要安装驱动：去 https://firmware.ardupilot.org/Tools/Drivers/ 下载

### 步骤 3：（可选）用 QGroundControl 验证飞控

1. 打开 QGroundControl
2. 它会自动检测 USB 连接的飞控
3. 左上角显示"已连接"说明飞控正常
4. 你能在 QGC 里看到飞控的飞行模式、GPS 状态、电池信息等
5. **确认完后关闭 QGC**（QGC 会占用 COM 口，导致 hw-agent 无法读取）

### 步骤 4：启动你的 CloudControl 后端

**如果你已经把后端部署到了服务器并且域名可以访问，这一步在本机可以跳过。**

也就是说：
- **本地开发测试**：你自己电脑上运行 `go run .`
- **线上部署测试**：直接使用你们服务器域名访问平台，不需要本机再启动后端

```powershell
cd c:\Users\ASUS\Desktop\双创github\Intelligent-Tool-Human-Computer-Interaction-and-Remote-Control-System\project
go run .
```

看到 `listening on :8080` 即成功。浏览器访问 `https://yyzk.online/` 注册并登录。

**如果走线上环境：** 浏览器直接访问你们现在的线上地址进行注册和登录。

### 步骤 5：在平台注册无人机

1. 进入「无人机管理」→ 新增无人机
2. **名称**填 `drone-01`（这个名称很重要，后面要用）
3. IP 填 `127.0.0.1`，协议选 SSH，端口 22
4. 保存

### 步骤 6：编译并启动 hw-agent

```powershell
# 编译 agent
cd c:\Users\ASUS\Desktop\双创github\Intelligent-Tool-Human-Computer-Interaction-and-Remote-Control-System\project
go build -o hw-agent.exe ./cmd/agent/
```

然后启动（**把 COM3 换成你步骤 2 记下的实际 COM 口号**）：

```powershell
.\hw-agent.exe -server https://yyzk.online -id drone-01 -mavlink COM3 -push-interval 2
```

### 步骤 7：确认数据到了

Agent 启动后你应该看到类似输出：

```
[Agent] Data source: MAVLink (real flight controller on COM3)
[MAVLink] Listening on UDP COM3 for flight controller telemetry
[Push] hardware ok
[Push] gps ok          ← 有 GPS 定位后才会有非零坐标
[Push] battery ok
```

然后在浏览器里检查：
- **无人机管理**：drone-01 状态变为 online
- **GPS/位置信息**：地图上出现标记点
- **电池监控**：显示实时电压电量
- **硬件状态检测**：显示你电脑的 CPU/内存/温度

> 💡 **GPS 注意**：如果飞控没有连 GPS 模块或在室内，GPS 坐标会是 0。这是正常的，因为 GPS 需要在室外才能定位。但电池和飞行模式数据在室内就能收到。

## 三、方案 B：4G 模块 + 网线

> 这是学长实验室的实际部署方式。4G 模块装在无人机上，通过网线连到你的电脑。

### 需要准备的东西

| 物品 | 说明 |
|------|------|
| 4G 模块 | 实验室有，带 RJ45 网口的绿色电路板（学长发过照片） |
| 网线（RJ45） | 实验室有 |
| 飞控板 | 通过串口连到 4G 模块 |
| 你的笔记本电脑 | 需要有网口，或者带一个 USB 转 RJ45 转接器 |

### 数据流原理

学长原话：
- "飞控、吊舱这些直连4G模块，4G模块跟云网平台是无线网用 **TCP** 通信的"
- "这边的4G模块没公网IP呀，它就相当于一个转发，转发给云网平台"
- "你们这个用电脑跑起来，要用个网线把4G模块连接起来"

```
飞控 ──串口──→ 4G 模块 ──网线──→ 你的电脑
                  │
          运行转发程序
      (uav_to_net3readpro.c)
          通过 TCP 发数据
```

### 步骤 1：物理连线

1. 飞控的 **TELEM 串口**（或 USB）连到 4G 模块的串口输入
2. 4G 模块的 **RJ45 网口**用网线连到你的笔记本网口
3. 给 4G 模块和飞控供电

### 步骤 2：配置网络（让电脑和 4G 模块在同一网段）

1. 打开「控制面板」→ 「网络和共享中心」→ 「更改适配器设置」
2. 找到你的**有线以太网**适配器，右键→ 「属性」
3. 双击「Internet 协议版本 4 (TCP/IPv4)」
4. 选择「使用下面的 IP 地址」，手动配置：

| 设置项 | 值（示例） |
|--------|------------|
| IP 地址 | `192.168.1.100` |
| 子网掩码 | `255.255.255.0` |
| 默认网关 | 留空 |

> ⚠️ 具体 IP 网段要看 4G 模块的默认 IP 是多少，问学长或者查模块说明书。常见的是 `192.168.1.x` 或 `192.168.0.x`。

### 步骤 3：验证网络连通

```powershell
# 把 192.168.1.1 换成 4G 模块的实际 IP
ping 192.168.1.1
```

学长原话："只要能 ping 通无非就是建立连接发送数据的事情了"

### 步骤 4：接收 4G 模块转发的 TCP 数据

4G 模块通过 TCP 转发飞控数据。你需要一个简单的脚本把 TCP 数据转为 UDP（因为 hw-agent 的 MAVLink 模式监听 UDP 14550）。

用 PowerShell 快速验证能否收到数据：

```powershell
# 先试试能不能连上 4G 模块的 TCP 端口（端口号问学长，常见的有 8899、6666、14550 等）
Test-NetConnection 192.168.1.1 -Port 8899
```

如果能连通，用以下 Python 脚本做 TCP → UDP 转发：

```python
# tcp_to_udp_bridge.py
# 从 4G 模块的 TCP 端口读取 MAVLink 数据，转发到本地 UDP 14550
import socket, threading

TCP_HOST = "192.168.1.1"   # 4G 模块 IP（改成实际的）
TCP_PORT = 8899            # 4G 模块 TCP 端口（问学长）
UDP_TARGET = ("127.0.0.1", 14550)  # 本地 hw-agent 监听端口

def bridge():
    tcp_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    tcp_sock.connect((TCP_HOST, TCP_PORT))
    udp_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    print(f"Bridge: TCP {TCP_HOST}:{TCP_PORT} -> UDP {UDP_TARGET}")
    while True:
        data = tcp_sock.recv(4096)
        if not data:
            break
        udp_sock.sendto(data, UDP_TARGET)
        print(f"  forwarded {len(data)} bytes")

if __name__ == "__main__":
    bridge()
```

```powershell
python tcp_to_udp_bridge.py
```

### 步骤 5：启动 hw-agent

```powershell
.\hw-agent.exe -server https://yyzk.online -id drone-01 -mavlink :14550 -push-interval 2
```

这样 hw-agent 监听 UDP 14550，Python 脚本把 4G 模块的 TCP 数据转过来，数据流就通了。

### 步骤 6：验证

同方案 A 的步骤 7，在浏览器里确认各模块数据正常。

## 四、后端已经部署到服务器后，应该如何测试

> 这一节是你们现在最该看的：**后端已经在服务器上跑起来了，域名也能访问，接下来重点不是“本地起后端”，而是“让真实设备 / agent / 浏览器都连到线上服务”。**

### 4.1 先确认线上平台本身正常

先在浏览器打开你们的线上地址，至少确认以下几点：

1. 首页能打开，没有 502 / 404 / 白屏
2. 可以注册、登录
3. 登录后主要模块能正常进入
4. 刷新页面后仍然正常
5. 开发者工具里没有大量接口报错

建议优先手工测试以下地址是否正常：

- 平台首页：`https://yyzk.online/`
- 健康检查：`https://yyzk.online/api/healthz`
- 公共配置：`https://yyzk.online/api/config`

### 4.2 实机数据如何接入线上后端

核心原则只有一句话：

**飞控数据仍然在你手边的电脑/无人机上产生，但 `hw-agent` 要把数据推送到服务器域名，而不是本机 `127.0.0.1:8080`。**

所以线上联调时，数据链路变成：

```
飞控 / 4G 模块
      ↓
  你的电脑运行 hw-agent
      ↓
  hw-agent 把硬件 / GPS / 电池 / 任务数据推送到线上后端
      ↓
  浏览器访问 https://yyzk.online 查看实时结果
```

你们要重点确认以下 4 件事：

- `hw-agent` 能联网访问服务器域名
- 服务器对应端口已放行
- 平台里注册的无人机名称和 `-id` 一致
- 浏览器端能看到 online / GPS / 电池等数据变化

### 4.3 最推荐的线上联调顺序

建议你们按下面顺序测，不容易乱：

```
1. 浏览器打开线上平台首页
2. 测 /api/healthz 是否正常
3. 用 simulate 模式跑 hw-agent，验证推送链路
4. 成功后换成真实飞控 / 4G 数据
5. 浏览器确认无人机 online、GPS、电池都正常
6. 新增 server-01，协议选 SSH
7. 打开 SSH 终端，执行 pwd / ls / whoami
8. 最后再做完整演示
```

## 六、可能遇到的问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 设备管理器看不到 COM 口 | USB 线不支持数据传输 / 缺驱动 | 换数据线；安装 PX4 驱动 |
| Agent 报 "Failed to push" | 后端没启动 / 端口不对 | 确认 `go run .` 正在运行 |
| GPS 坐标全是 0 | 室内无 GPS 信号 | 正常现象，到室外才有；电池数据不受影响 |
| 平台上看不到无人机 online | `-id` 和注册名称不一致 | 确保完全一致，如都是 `drone-01` |
| QGC 打开后 agent 连不上 | QGC 占用了 COM 口 | 关闭 QGC 再启动 agent |
| 4G 模块 ping 不通 | IP 网段不对 | 问学长模块默认 IP，调整电脑 IP 到同网段 |
| TCP 连接被拒绝 | 端口号不对 | 问学长 4G 模块的 TCP 服务端口 |
| 浏览器里 SSH 终端打不开 | 目标机器没开 SSH / 22 端口没放行 / 用户名密码错误 | 先用云服务器本机做测试目标，再排查 SSH 服务与防火墙 |
| 平台能打开但终端连不上 | 浏览器能访问服务器，但服务器访问不到目标机器 | 记住 SSH 是“服务器去连目标机”，不是你电脑去连 |

# 系统状态监控 功能介绍

## 一、功能概述

系统状态监控是"CloudControl"的核心模块之一，负责对**运行后端服务的主服务器**进行实时状态采集与可视化展示。用户可通过该模块实时查看服务器的 CPU 使用率、内存使用率、磁盘使用率、网络带宽等关键指标，及时发现系统异常并采取相应措施，保障整个远程操控平台的稳定运行。

---

## 二、监控目标说明：为什么监控的是后端主机？

### 2.1 当前监控对象

本系统监控的是**部署并运行后端服务的主服务器（主机）**，而非远程被操控的设备。

### 2.2 设计原因

在"CloudControl"的架构中，**后端主服务器是整个系统的中枢节点**，承担以下核心职责：

1. **请求调度中心**：所有前端用户的操作请求（远程桌面控制、语音交互、视频监控等）都通过后端主机进行转发和调度。如果主机出现性能瓶颈或宕机，所有远程操控功能将全部瘫痪。
2. **数据存储与处理**：系统的日志记录、告警信息、音频文件、性能报告等数据均存储在主机的数据库中。主机的磁盘和内存状态直接影响数据的完整性和可用性。
3. **实时通信枢纽**：远程桌面控制（VNC代理）、实时监控数据推送（WebSocket）等功能依赖主机的网络带宽和 CPU 处理能力。
4. **稳定性保障**：在远程操控场景中，操控端（用户浏览器）与被操控端（远程设备）之间的通信链路必须经过主机。**监控主机的健康状态，就是在监控整个系统的可用性。**

### 2.3 关于远程设备的监控

远程被操控设备（如远程桌面目标机器）的状态监控属于不同的功能范畴：

- **远程设备的在线/离线状态**：由"远程桌面控制"模块中的设备管理功能负责，通过设备列表可查看各设备的连接状态。
- **远程设备的硬件信息**：由"硬件状态检测"模块负责采集和展示。
- **远程设备的异常告警**：由"异常报警"模块统一管理。

本系统采用**分模块职责分离**的设计理念：系统状态监控专注于保障平台自身（主服务器）的稳定运行，而对远程设备的管理和监控由各自对应的功能模块负责。这种设计避免了功能耦合，使每个模块的职责清晰明确。

## 三、技术实现

### 3.1 后端数据采集

后端使用 Go 语言编写，通过 `gopsutil` 库直接采集主机的系统指标：

| 指标         | 采集方式                | 说明                                  |
| ------------ | ----------------------- | ------------------------------------- |
| CPU 使用率   | `cpu.Percent()`       | 当前 CPU 总体使用百分比               |
| 内存使用率   | `mem.VirtualMemory()` | 已用内存占总内存的百分比              |
| 磁盘使用率   | `disk.Usage()`        | 根分区（Windows 为 C 盘）的使用百分比 |
| 系统负载     | `load.Avg()`          | 1 分钟平均负载（Linux/macOS）         |
| 网络流量     | `net.IOCounters()`    | 累计发送/接收字节数                   |
| 系统运行时间 | `host.Uptime()`       | 主机自上次启动以来的运行秒数          |

### 3.2 数据传输方式

- **HTTP API**（`GET /api/metrics/snapshot`）：一次性获取当前时刻的系统指标快照，用于页面初始化和手动刷新。
- **WebSocket**（`GET /api/metrics/stream`）：建立持久连接，后端每秒推送一次最新指标数据，实现卡片区域和趋势图表的实时动态更新。前端断线后会每 3 秒自动尝试重连。

### 3.3 硬件信息获取

通过 `GET /api/hardware/snapshot` 接口获取主机的硬件信息（主机名、操作系统、CPU 型号、内存总量等），用于在设备状态卡片中展示设备名称和 IP 地址。

## 四、功能详细说明

### 4.1 实时数据展示（自动更新）

页面加载后自动通过 WebSocket 连接后端，以下区域**实时自动更新**，无需用户操作：

- **设备状态卡片**：显示设备名称、设备 IP、CPU/内存/磁盘使用率进度条，以及设备在线状态。
- **网络状态卡片**：显示网络带宽、当前使用带宽、带宽使用率进度条、网络延迟和丢包率。
- **趋势折线图**：三个紧凑的折线图分别展示 CPU 使用率、内存使用率、磁盘使用率随时间的变化趋势，最多保留最近 20 个采样点。

### 4.2 历史记录表格（手动刷新）

历史记录表格**仅在用户点击"刷新"按钮时**才会新增一条记录，记录当前时刻的系统状态快照。这样设计的目的是：

- 避免实时数据每秒刷新导致表格行数爆炸式增长。
- 让用户主动选择需要记录的时间点，便于对比分析。
- 每条记录包含：监控时间、设备名称、CPU 使用率、内存使用率、磁盘使用率、网络带宽、状态、操作（查看详情）。

### 4.3 监控条件设置

| 设置项       | 说明                                                          |
| ------------ | ------------------------------------------------------------- |
| 监控时间段   | 选择开始日期和结束日期，筛选特定时间段的数据                  |
| 监控设备     | 选择要监控的设备（所有设备 / 主服务器），详见下方说明         |
| 监控指标     | 选择关注的指标（CPU / 内存 / 磁盘 / 网络带宽）                |
| 数据更新频率 | 选择数据获取频率（1 / 5 / 15 / 30 分钟，用于查询/扩展预留；实时 WebSocket 推送仍为每秒一次） |
| 实时监控开关 | 开启后通过 WebSocket 实时推送数据；关闭则断开连接（默认开启） |
| 告警通知开关 | 开启后当指标超过阈值时弹出告警提示                            |
| 告警阈值     | 自定义告警触发的百分比阈值（默认 80%）                        |

#### "监控设备"选项说明

"监控设备"下拉框提供以下两个选项：

- **主服务器**：指运行本系统后端服务的那台主机。系统通过 `gopsutil` 库直接采集该主机的 CPU、内存、磁盘、网络等指标。页面上显示的设备名称（如 `LAPTOP-QULPJ2AF`）和设备 IP（如 `127.0.0.1`）即为该主机的真实主机名和访问地址。选择此项时，监控数据来源于后端服务所在的本机。
- **所有设备**：在当前版本中，系统仅部署了一台后端主服务器，因此"所有设备"与"主服务器"展示的数据相同。该选项为系统扩展预留——当未来系统扩展为分布式多节点部署（例如多台服务器分别承担不同的服务职责）时，选择"所有设备"将汇总展示所有节点的监控数据，而选择具体设备则只展示该节点的数据。

> **总结**：当前系统为单机部署架构，"监控设备"下拉框中的两个选项实际指向同一台主机。该设计是为后续多设备扩展做的接口预留。

### 4.4 详情查看与操作

- **查看详情**：点击页面顶部"查看详情"按钮或表格中每行的"查看详情"按钮，弹出详情窗口。
- **详情内容**：设备名称、设备 IP、操作系统、CPU 型号、CPU 使用率、内存使用率、磁盘使用率、系统负载、网络发送/接收量、采集时间。
- **确认操作**：点击"确认"按钮可对当前设备状态进行标记确认。
- **关闭窗口**：点击"关闭"按钮或窗口外部区域即可关闭。

### 4.5 数据导出

点击"导出数据"按钮，将历史记录表格中的数据导出为 CSV 文件（UTF-8 编码，兼容 Excel），方便用户进行离线分析或存档。

### 4.6 告警通知

开启告警通知后，系统会在每次收到实时数据时检查 CPU、内存、磁盘使用率是否超过设定的阈值。一旦超过，页面右上角会弹出醒目的告警提示（Toast），帮助用户及时发现异常。

同时，后端也运行着独立的阈值告警协程，当指标超过预设阈值时会自动将告警记录写入数据库，可在"异常报警"模块中查看历史告警。

## 五、使用指南

1. **查看实时状态**：打开页面后，设备状态卡片、网络状态卡片和趋势图表会自动实时更新，无需任何操作。
2. **记录历史快照**：点击"刷新"按钮，当前时刻的系统状态会被记录到下方的历史表格中。多次点击可记录多个时间点的数据用于对比。
3. **导出数据**：先通过多次"刷新"积累历史记录，然后点击"导出数据"将表格保存为 CSV 文件。
4. **开启告警**：打开"是否启动告警通知"开关，并设置合适的阈值（如 80%），当系统指标超过阈值时会自动弹出提醒。
5. **查看详情**：点击"查看详情"按钮可查看完整的系统信息，包括硬件配置和当前各项指标的具体数值。
6. **关闭实时监控**：如不需要实时更新，可关闭"是否开启实时监控"开关，WebSocket 连接将断开，页面停止自动刷新。

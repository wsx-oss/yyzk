# 维护操作日志

## 1. 功能概述

维护操作日志模块是"智能工具人机交互与远程操控系统"的重要功能之一，用于记录、管理和追踪系统维护过程中的各项操作。该模块支持日志的新增、编辑、删除、搜索筛选、批量导入、导出等操作，并通过统计图表直观展示操作类型分布和操作结果分布，帮助管理人员全面掌握系统维护情况，便于追溯和审计操作行为。

## 2. 功能详细说明

### 2.1 日志管理功能

#### 新增日志

用户可点击页面右上角的 **"新增日志"** 按钮，打开新增维护操作日志的弹窗。在该窗口中，用户需填写以下信息：

| 字段     | 是否必填 | 说明                                                       |
| -------- | -------- | ---------------------------------------------------------- |
| 操作类型 | 是       | 操作类型：系统更新、设备维护、数据备份、系统重启           |
| 操作人员 | 是       | 执行操作的人员姓名                                         |
| 操作时间 | 否       | 操作执行的时间，默认为当前时间                             |
| 操作结果 | 是       | 操作结果：成功、失败                                       |
| 设备名称 | 否       | 与操作相关的设备名称                                       |
| 日志状态 | 是       | 日志状态：启用、禁用                                       |
| 操作详情 | 否       | 对该操作的详细描述信息                                     |

填写完成后，点击 **"确认"** 即可添加新的日志记录。数据会持久化保存到后端数据库，重启服务后不会丢失。

#### 编辑日志

在日志列表中，点击操作列的 **"编辑"** 按钮，系统会弹出编辑窗口，预填该日志的所有信息。用户可修改任意字段后点击 **"确认"** 保存更改。编辑操作支持部分更新，仅修改的字段会被更新到数据库。

#### 删除日志

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中永久移除该日志记录。

> **注意**：删除操作不可恢复，请谨慎操作。

#### 导入日志

点击 **"导入日志"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的文件进行批量导入。JSON 文件格式要求为数组格式，每个元素包含日志的各项字段信息。

导入时系统会自动校验数据：

- 未填写的日志状态默认为"启用"
- 未填写的操作时间默认为当前时间
- 导入完成后显示成功导入的记录数量

#### 导出日志

点击 **"导出日志"** 按钮，系统会将当前筛选条件下的所有日志数据导出为 CSV 文件，文件名格式为 `logs_YYYY-MM-DD.csv`。导出字段包括：操作类型、操作人员、操作时间、操作结果、设备名称、日志状态、操作详情。CSV 文件包含 BOM 头，支持 Excel 直接打开中文不乱码。

### 2.2 日志查询

#### 条件筛选

用户可在筛选区域中填写以下条件进行搜索：

| 条件         | 说明                                                       |
| ------------ | ---------------------------------------------------------- |
| 操作类型     | 按类型精确筛选（系统更新/设备维护/数据备份/系统重启/全部） |
| 操作人员     | 按人员姓名模糊匹配                                         |
| 操作时间范围 | 按开始日期和结束日期筛选                                   |
| 操作结果     | 按结果精确筛选（成功/失败/全部）                           |
| 设备名称     | 按设备名称模糊匹配                                         |
| 日志状态     | 通过 Toggle 开关切换筛选启用或禁用状态的日志               |

#### 搜索

填写筛选条件后，点击 **"搜索"** 按钮，系统将快速展示符合条件的日志列表。

> **说明**：所有筛选条件均为可选，支持组合筛选。筛选通过后端 API 参数传递执行。

### 2.3 日志展示与操作

#### 日志列表

页面以表格形式展示维护操作日志，包含以下列：

| 列名     | 说明                                                   |
| -------- | ------------------------------------------------------ |
| 操作类型 | 系统更新 / 设备维护 / 数据备份 / 系统重启             |
| 操作人员 | 执行操作的人员姓名                                     |
| 操作时间 | 操作执行的时间                                         |
| 操作结果 | 以彩色文字显示（绿色成功/红色失败）                    |
| 设备名称 | 与操作相关的设备名称                                   |
| 日志状态 | 启用 / 禁用                                           |
| 操作     | 编辑、删除按钮                                         |

表格支持分页显示，每页 20 条记录，底部显示总记录数和翻页控件。

### 2.4 统计信息展示

页面底部展示两个统计图表：

- **操作类型分布**（环形图）：以环形图形式展示不同操作类型（系统更新、设备维护、数据备份、系统重启等）的占比情况，图例显示在右侧，用户可直观了解各类操作的分布状况。
- **操作结果分布**（水平柱状图）：以水平柱状图形式展示操作成功和失败的数量对比，便于用户掌握操作结果的总体情况。

图表数据通过独立的统计接口获取，与列表数据分离，确保统计准确性。

### 2.5 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 可通过操作类型、操作人员等条件进行日志搜索。
2. 点击新增日志可手动添加维护操作日志。
3. 导入导出功能可方便批量处理日志数据。

## 3. 技术实现

### 3.1 后端接口

| 接口                 | 方法   | 说明                                                                                                     |
| -------------------- | ------ | -------------------------------------------------------------------------------------------------------- |
| `/api/logs/list`   | GET    | 获取日志列表（支持 op_type、operator、date_from、date_to、op_result、device_name、log_status 筛选 + 分页） |
| `/api/logs/append` | POST   | 新增日志记录（JSON 格式，含 op_type、operator、op_time、op_result、device_name、log_status、detail 等字段） |
| `/api/logs/:id`    | PUT    | 编辑指定 ID 的日志信息                                                                                   |
| `/api/logs/:id`    | DELETE | 删除指定 ID 的日志记录                                                                                   |
| `/api/logs/import` | POST   | 批量导入日志（接收 JSON 数组）                                                                           |
| `/api/logs/stats`  | GET    | 获取日志统计数据（操作类型分布 + 操作结果分布）                                                          |

### 3.2 数据存储

后端使用 SQLite 数据库的 `logs` 表存储日志信息，包含以下字段：

| 字段        | 类型     | 说明                                     |
| ----------- | -------- | ---------------------------------------- |
| id          | INTEGER  | 主键，自增                               |
| level       | TEXT     | 日志级别                                 |
| message     | TEXT     | 日志消息                                 |
| meta        | TEXT     | 元数据                                   |
| op_type     | TEXT     | 操作类型                                 |
| operator    | TEXT     | 操作人员                                 |
| op_time     | TEXT     | 操作时间                                 |
| op_result   | TEXT     | 操作结果（成功/失败）                    |
| device_name | TEXT     | 设备名称                                 |
| log_status  | TEXT     | 日志状态（启用/禁用）                    |
| detail      | TEXT     | 操作详情                                 |
| created_at  | DATETIME | 创建时间                                 |

### 3.3 前端技术

- 使用 Chart.js 绘制操作类型分布环形图和操作结果分布水平柱状图
- 使用模态弹窗实现新增和编辑日志交互，新增和编辑共用同一弹窗
- 使用 CSS Toggle Switch 实现日志状态的开关控件，匹配截图中的滑动开关样式
- 操作结果使用彩色文字区分显示（绿色成功/红色失败）
- 搜索筛选通过后端 API 参数传递执行，支持 7 种条件组合筛选
- 导入功能通过模态弹窗选择 JSON 文件，读取后解析并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 CSV 文件并触发下载，含 BOM 头支持中文
- 点击模态弹窗外部区域可关闭弹窗
- 删除操作前弹出确认窗口，防止误操作

## 4. 使用指南

1. **新增日志**：点击"新增日志"按钮，填写操作类型、操作人员、操作时间、操作结果、设备名称、日志状态、操作详情后点击"确认"。
2. **编辑日志**：点击日志的"编辑"按钮，在弹窗中修改信息后点击"确认"保存。
3. **删除日志**：点击"删除"按钮，在确认弹窗中确认后移除日志记录。
4. **搜索日志**：在筛选区域输入条件（操作类型、操作人员、时间范围、操作结果、设备名称、日志状态开关），点击"搜索"按钮快速定位。
5. **导入日志**：点击"导入日志"按钮，选择 JSON 文件进行批量导入。
6. **导出日志**：点击"导出日志"按钮，将日志列表导出为 CSV 文件保存。
7. **查看统计**：页面底部的图表实时展示操作类型和操作结果分布。

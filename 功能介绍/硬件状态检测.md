# 硬件状态检测

## 1. 功能概述

硬件状态检测模块是"CloudControl"的重要功能之一，用于**实时监控硬件设备的真实运行状态**。该模块支持两种数据上报方式：

- **拉取（Pull）模式**：主服务器连接目标设备上的 `hw-agent`（默认 `:9100`）并读取 `/metrics` 获取真实指标。
- **推送（Push）模式（可选）**：目标设备/无人机侧 Agent 主动 `POST /api/hardware/push` 上报指标（通过 `agent_id` 识别）。

系统支持硬件设备的添加、搜索筛选、实时数据刷新、导出、查看详情、删除等操作，并通过统计图表直观展示硬件状态分布和硬件类型分布。

## 2. 系统架构

```
┌─────────────────┐         HTTP GET          ┌─────────────────────┐
│   主服务器       │  ──────────────────────→  │  目标设备 A          │
│  (smartcontrol)  │  ←── JSON 硬件数据 ────  │  (hw-agent :9100)    │
│   :8080          │                           │  采集本机CPU/内存/   │
│                  │         HTTP GET          │  温度/网络带宽       │
│   浏览器访问     │  ──────────────────────→  ├─────────────────────┤
│   管理界面       │  ←── JSON 硬件数据 ────  │  目标设备 B          │
│                  │                           │  (hw-agent :9100)    │
└─────────────────┘                           └─────────────────────┘
```

**工作原理**：

1. 每台被监控的目标设备上运行 `hw-agent` 程序，它会采集本机的 CPU、内存、温度、网络等真实数据
2. `hw-agent` 在端口 9100 上暴露 HTTP 接口 `/metrics`，返回 JSON 格式的硬件指标
3. 主服务器在手动刷新/自动刷新时，通过 HTTP 请求各目标设备的 Agent 获取最新数据
4. 如果 Agent 可达，设备标记为"在线"并更新真实数据；如果不可达，标记为"离线"

> **补充**：为降低实时监控延迟，后端提供 `GET /api/hardware/items/live`，会并发拉取各 Agent 并直接返回合并结果；数据库会在后台异步更新，保证持久化数据也持续变新。

## 3. 目标设备配置（Agent 部署）

### 3.1 前提条件

- 目标设备与主服务器在同一局域网内，或通过 VPN/端口转发互通
- 目标设备的 **9100 端口**未被防火墙拦截
- 目标设备已安装 Go 语言环境（用于编译 Agent），或直接使用预编译的可执行文件

### 3.2 编译 Agent

在项目根目录下执行以下命令编译 Agent 程序：

```bash
# Windows
cd project
go build -o hw-agent.exe ./cmd/agent/

# Linux
cd project
go build -o hw-agent ./cmd/agent/
```

编译完成后会生成 `hw-agent.exe`（Windows）或 `hw-agent`（Linux）可执行文件。

### 3.3 部署到目标设备

将编译好的 `hw-agent` 可执行文件复制到目标设备上（通过 U 盘、SCP、共享文件夹等方式），然后在目标设备上运行：

```bash
# Windows（在命令提示符或 PowerShell 中运行）
.\hw-agent.exe

# Linux（在终端中运行）
chmod +x hw-agent
./hw-agent

# 自定义端口（默认 9100）
.\hw-agent.exe -port 9100
```

运行成功后会显示：

```
Hardware Agent started on :9100
  GET /metrics  - collect hardware metrics
  GET /health   - health check
```

### 3.4 验证 Agent 运行

在主服务器上用浏览器或命令行访问目标设备的 Agent 接口，确认返回数据：

```bash
# 将 192.168.1.100 替换为目标设备的实际 IP
curl http://192.168.1.100:9100/metrics
```

正常返回示例：

```json
{
  "hostname": "Server01",
  "cpu_usage": 23.5,
  "mem_usage": 61.2,
  "temperature": 45.0,
  "network_bandwidth": "186.8MB",
  "os": "windows",
  "cpu_model": "Intel(R) Core(TM) i5-7200U CPU @ 2.50GHz",
  "cpu_cores": 4,
  "mem_total_mb": 16282,
  "uptime": 86400,
  "timestamp": 1770738731
}
```

### 3.5 防火墙配置

如果目标设备有防火墙，需要放行 9100 端口：

```bash
# Windows（以管理员身份运行 PowerShell）
netsh advfirewall firewall add rule name="HW-Agent" dir=in action=allow protocol=TCP localport=9100

# Linux
sudo ufw allow 9100/tcp
# 或
sudo firewall-cmd --add-port=9100/tcp --permanent && sudo firewall-cmd --reload
```

### 3.6 监控本机（无需额外操作）

主服务器启动时会**自动在后台启动本机 Agent**（端口 9100），无需手动运行 `hw-agent.exe`。只需：

```bash
cd project
go run .
# 或
.\smartcontrol.exe
```

启动后日志会显示：

```
[Agent] Hardware Agent started on :9100
listening on :8080
```

在添加硬件时，IP 地址填写 `127.0.0.1` 即可采集本机的真实硬件数据。

### 3.7 监控远程设备

远程设备需要手动部署 Agent。将编译好的 `hw-agent.exe` 复制到远程设备并运行。

**完整示例：监控 3 台设备**

假设局域网内有 3 台设备：

| 设备       | IP 地址       | 角色                    |
| ---------- | ------------- | ----------------------- |
| 管理主机   | 192.168.1.10  | 运行主服务（自带Agent） |
| 办公室PC   | 192.168.1.101 | 运行 hw-agent.exe       |
| 机房服务器 | 192.168.1.102 | 运行 hw-agent.exe       |

操作步骤：

1. 在管理主机上运行 `.\smartcontrol.exe`（自动启动本机 Agent）
2. 将 `hw-agent.exe` 复制到办公室PC和机房服务器，分别运行
3. 在浏览器中打开管理界面，依次添加 3 台设备：
   - 名称"管理主机"，IP 填 `127.0.0.1`（或 `192.168.1.10`）
   - 名称"办公室PC"，IP 填 `192.168.1.101`
   - 名称"机房服务器"，IP 填 `192.168.1.102`
4. 系统会自动检测每个 IP 的 Agent 是否可达，只有 Agent 正常运行的设备才能添加成功
5. 添加成功后，所有设备的 CPU、内存、温度、网络数据会实时更新

> **注意**：每台设备的 IP 地址不同，系统通过 IP 区分不同设备。同一 IP 只需添加一次。

## 4. 功能详细说明

### 4.1 硬件管理功能

#### 添加硬件

用户可点击页面右上角的 **"添加硬件"** 按钮，打开添加硬件的弹窗。在该窗口中，用户需填写以下信息：

| 字段       | 是否必填 | 说明                                                   |
| ---------- | -------- | ------------------------------------------------------ |
| 硬件名称   | 是       | 硬件设备的标识名称，如"主服务器"、"核心交换机"         |
| 硬件类型   | 是       | 设备类型：服务器、路由器、交换机、存储设备、无人机     |
| 硬件IP地址 | 是       | 目标设备的 IP 地址，如 `192.168.1.100`，需符合 IP 格式 |
| 初始状态   | 是       | 设备当前状态：在线、离线、警告                         |
| 描述信息   | 否       | 对该硬件设备的补充说明，如"二楼机房核心交换机"         |

> **重要说明**：点击"确认添加"后，系统会**先检测目标 IP 上的 Agent 是否可达**（端口 9100）。如果 Agent 未运行或返回的数据全部为零，系统会**拒绝添加并给出提示**，避免添加无效设备。只有 Agent 正常运行且返回有效数据的设备才能成功添加。

填写完成后，点击 **"确认添加"**，系统会自动检测 Agent 并采集数据。

#### 搜索硬件

用户可在搜索表单中填写以下条件，点击 **"搜索"** 按钮进行筛选：

| 条件          | 说明                                                 |
| ------------- | ---------------------------------------------------- |
| 硬件名称      | 按名称模糊匹配                                       |
| 硬件类型      | 按类型精确筛选（服务器/路由器/交换机/存储设备/无人机/全部） |
| 硬件IP地址    | 按 IP 地址模糊匹配                                   |
| 检测时间范围  | 按检测时间范围筛选（开始日期 ~ 结束日期）            |
| 状态筛选      | 按状态精确筛选（在线/离线/警告/全部）                |
| 温度范围      | 按温度范围筛选（最小值 ~ 最大值，单位℃）             |
| CPU使用率范围 | 按 CPU 使用率范围筛选（最小值 ~ 最大值，单位%）      |

#### 刷新数据

点击 **"刷新数据"** 按钮，系统会**逐一连接所有已添加设备的 Agent**，采集最新的真实硬件数据。刷新完成后会在页面中央显示在线和离线设备的数量。

此外，页面右上角提供**自动刷新间隔设置**（下拉选择），支持以下选项：

- **关闭** — 不自动刷新
- **5秒 / 10秒（默认）/ 30秒 / 1分钟 / 5分钟** — 按所选间隔自动刷新

绿色脉冲圆点表示自动刷新正在运行，灰色表示已关闭。

#### 单设备刷新

在设备列表的操作列中，每个设备都有一个绿色的 **"刷新"** 按钮，点击后仅刷新该设备的数据，适用于需要快速确认某台设备状态的场景。

#### 导出数据

点击 **"导出数据"** 按钮，系统会将当前硬件列表中的所有数据导出为 CSV 文件（UTF-8 编码，含 BOM 头），方便在 Excel 中打开查看。

### 4.2 硬件状态监测功能

#### 硬件列表

页面以表格形式展示所有硬件设备的详细信息，包括以下列：

| 列名       | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| 硬件名称   | 设备标识名称                                           |
| 硬件类型   | 设备类型（服务器/路由器/交换机/存储设备/无人机）       |
| 硬件IP地址 | 设备 IP 地址                                           |
| 状态       | 设备状态，以彩色圆点标识（绿色在线/红色离线/黄色警告） |
| 温度       | **真实**设备温度（℃），从 Agent 采集                   |
| CPU使用率  | **真实** CPU 使用率（%），从 Agent 采集                |
| 内存使用率 | **真实**内存使用率（%），从 Agent 采集                 |
| 网络带宽   | **真实**网络带宽使用情况，从 Agent 采集                |
| 检测时间   | 最近一次成功采集数据的时间                             |
| 操作       | 查看详情、刷新、删除按钮                               |

#### 设备状态自动判定

- **在线**：Agent 可达，成功采集到数据
- **离线**：Agent 不可达（未运行、网络不通、端口被拦截等）

### 4.3 硬件详情功能

点击操作列的 **"查看详情"** 按钮，系统会弹出详情窗口，展示该硬件设备的完整信息。

#### 删除硬件

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中移除该硬件设备记录。

> **注意**：删除操作不可恢复，请谨慎操作。

### 4.4 硬件统计功能

页面底部展示两个统计图表：

- **硬件状态分布**（饼图）：以环形图形式展示在线、离线、警告三种状态的设备占比情况。
- **硬件类型分布**（柱状图）：以水平柱状图形式展示不同类型硬件的数量对比。

图表数据会随硬件列表的变化实时更新。

### 4.5 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 硬件状态检测用于实时监控硬件设备的运行状态，请定期刷新数据。
2. 若硬件状态显示为警告，请及时检查硬件设备。
3. 删除硬件信息后不可恢复，请谨慎操作。

## 5. 技术实现

### 5.1 Agent 程序

Agent 是一个独立的轻量级 Go 程序（`project/cmd/agent/main.go`），运行在每台被监控的目标设备上。

**采集的指标**：

| 指标            | 采集方式                               | 说明                   |
| --------------- | -------------------------------------- | ---------------------- |
| CPU 使用率      | `gopsutil/cpu.Percent()`，采样 1 秒    | 精确到小数点后 2 位    |
| 内存使用率      | `gopsutil/mem.VirtualMemory()`         | 精确到小数点后 2 位    |
| 温度            | `gopsutil/host.SensorsTemperatures()`  | 取最高传感器温度       |
| 网络带宽        | `gopsutil/net.IOCounters()`            | 累计发送+接收字节数    |
| 主机名          | `gopsutil/host.Info()`                 | 操作系统主机名         |
| CPU 型号/核心数 | `gopsutil/cpu.Info()` + `cpu.Counts()` | 处理器型号和逻辑核心数 |
| 内存总量        | `gopsutil/mem.VirtualMemory()`         | 总内存（MB）           |
| 系统运行时间    | `gopsutil/host.Info()`                 | 开机至今的秒数         |

**暴露的接口**：

| 接口       | 说明                             |
| ---------- | -------------------------------- |
| `/metrics` | 返回 JSON 格式的完整硬件指标数据 |
| `/health`  | 健康检查，返回 `{"status":"ok"}` |

### 5.2 后端接口

| 接口                              | 方法   | 说明                                                                                                       |
| --------------------------------- | ------ | ---------------------------------------------------------------------------------------------------------- |
| `/api/hardware/items`             | GET    | 获取硬件列表（支持 name、type、ip、status、date_from、date_to、temp_min、temp_max、cpu_min、cpu_max 筛选） |
| `/api/hardware/items`             | POST   | 添加新硬件设备，添加后自动连接 Agent 采集数据                                                              |
| `/api/hardware/items/check-agent` | POST   | 预检 Agent 可达性（添加前验证），返回是否可连接及指标数据                                                  |
| `/api/hardware/items/:id`         | GET    | 获取指定 ID 的硬件详情                                                                                     |
| `/api/hardware/items/:id`         | PUT    | 更新指定 ID 的硬件信息                                                                                     |
| `/api/hardware/items/:id`         | DELETE | 删除指定 ID 的硬件设备                                                                                     |
| `/api/hardware/items/:id/refresh` | POST   | 刷新单个设备，连接其 Agent 采集最新数据                                                                    |
| `/api/hardware/items/stats`       | GET    | 获取硬件统计数据（状态分布 + 类型分布）                                                                    |
| `/api/hardware/items/refresh`     | POST   | 批量刷新所有设备，逐一连接各 Agent 采集数据                                                                |
| `/api/hardware/items/live`        | GET    | 低延迟实时拉取（并发拉取所有 Agent 并直接返回合并结果；DB 异步更新）                                       |
| `/api/hardware/push`              | POST   | Push 上报硬件指标（通过 `agent_id` 识别；匹配则更新，不存在则自动创建）                                    |

### 5.3 数据存储

后端使用 SQLite 数据库的 `hardware_items` 表存储硬件信息，包含以下字段：

| 字段              | 类型     | 说明                                      |
| ----------------- | -------- | ----------------------------------------- |
| id                | INTEGER  | 主键，自增                                |
| name              | TEXT     | 硬件名称                                  |
| type              | TEXT     | 硬件类型（服务器/路由器/交换机/存储设备/无人机） |
| ip                | TEXT     | IP 地址                                   |
| status            | TEXT     | 设备状态（在线/离线/警告）                |
| temperature       | REAL     | 温度（℃），从 Agent 采集                  |
| cpu_usage         | REAL     | CPU 使用率（%），从 Agent 采集            |
| mem_usage         | REAL     | 内存使用率（%），从 Agent 采集            |
| network_bandwidth | TEXT     | 网络带宽，从 Agent 采集                   |
| description       | TEXT     | 描述信息                                  |
| detected_at       | DATETIME | 最近一次检测时间                          |
| created_at        | DATETIME | 创建时间                                  |
| updated_at        | DATETIME | 更新时间                                  |

### 5.4 前端技术

- 使用 Chart.js 绘制硬件状态分布饼图和硬件类型分布柱状图（紧凑布局）
- 使用模态弹窗实现添加硬件和查看详情交互
- 搜索筛选通过后端 API 参数传递执行，支持多条件组合筛选
- 导出功能生成 CSV 文件，支持 UTF-8 编码和 BOM 头，兼容 Excel 直接打开
- **自动刷新间隔可配置**（关闭/5秒/10秒/30秒/1分钟/5分钟），默认 10 秒
- 每个设备支持**单独刷新**按钮，快速获取单台设备最新数据
- 添加硬件前**预检 Agent 可达性**，无效 IP 不予添加
- 操作提示信息在**页面中上方居中**显示，醒目且不遮挡内容

### 5.5 数据采集流程

```
1. 用户添加硬件（填写 IP 地址）
   ↓
2. 前端先调用 check-agent 接口预检 Agent 可达性
   ↓
3a. Agent 不可达或数据全零 → 拒绝添加，提示用户检查目标设备
3b. Agent 可达且数据有效 → 继续添加
   ↓
4. 后端插入记录并自动采集真实数据 → 状态设为"在线"
   ↓
5. 前端按用户设定的间隔自动触发批量刷新
   ↓
6. 后端逐一连接所有设备的 Agent → 更新数据库 → 前端刷新表格和图表
```

## 6. 使用指南

1. **启动系统**：在 `project` 目录下运行 `go run .` 或 `.\smartcontrol.exe`，主服务和本机 Agent 会同时启动。
2. **部署远程 Agent**：如需监控远程设备，将 `hw-agent.exe` 复制到目标设备并运行（详见第 3.7 节）。
3. **添加硬件**：点击"添加硬件"按钮，填写硬件名称、类型、目标设备 IP 地址后点击"确认添加"。系统会**先检测 Agent 是否可达**，不可达则拒绝添加。
4. **查看实时数据**：添加成功后，表格中会显示真实的 CPU 使用率、内存使用率、温度、网络带宽。
5. **设置自动刷新**：右上角下拉选择自动刷新间隔（关闭/5秒/10秒/30秒/1分钟/5分钟），默认每 10 秒自动更新。
6. **手动刷新**：点击"刷新数据"按钮批量刷新所有设备，或点击单个设备的"刷新"按钮刷新指定设备。
7. **搜索硬件**：在搜索表单中输入筛选条件，点击"搜索"按钮快速定位设备。
8. **导出数据**：点击"导出数据"按钮，将硬件列表导出为 CSV 文件保存。
9. **查看详情**：点击设备的"查看详情"按钮，在弹窗中查看完整的设备信息。
10. **删除硬件**：点击"删除"按钮，在确认弹窗中确认后移除设备记录。
11. **查看统计**：页面底部的图表实时展示硬件状态和类型分布。

# 远程桌面控制

## 1. 功能概述

远程桌面控制模块是"智能工具人机交互与远程操控系统"的核心功能之一，用于管理和操控远程设备的桌面连接。该模块支持远程连接的新增、搜索、刷新、连接、断开、删除等操作，并通过统计图表直观展示连接状态分布和连接类型使用情况，帮助管理人员高效管理远程设备。

> **新特性**：在当前版本中，远程设备可通过 **无人机管理模块** 自动创建——注册无人机时系统会自动在 `devices` 表中生成对应的远程设备记录（包含 IP、端口、协议等信息），无需手动添加。当然，用户仍可通过本页面手动新增独立的远程连接。

## 2. 功能详细说明

### 2.1 连接管理功能

#### 新增连接

用户可点击页面右上角的 **"新增连接"** 按钮，打开新增远程连接的弹窗。在该窗口中，用户需填写以下信息：

| 字段     | 是否必填 | 属于谁     | 说明                                                                            |
| -------- | -------- | ---------- | ------------------------------------------------------------------------------- |
| 设备名称 | 是       | 目标设备   | 远程设备的标识名称，如"Server01"、"测试机A"                                     |
| IP地址   | 是       | 目标设备   | 目标设备的局域网或公网 IP 地址，如 `192.168.1.100`                            |
| 连接类型 | 是       | 连接协议   | 连接目标设备所用的协议：RDP（Windows远程桌面）、VNC（跨平台）、SSH（Linux终端） |
| 端口号   | 是       | 目标设备   | 目标设备上远程服务监听的端口号。VNC 默认 5900，RDP 默认 3389，SSH 默认 22       |
| 用户名   | 是       | 目标设备   | 登录目标设备的系统账户用户名（即目标电脑上的操作系统用户名）                    |
| 密码     | 否       | 目标设备   | 登录目标设备的系统账户密码（即目标电脑上的操作系统密码）                        |
| 自动连接 | 否       | 本系统设置 | 启用后，系统启动时会自动尝试连接该设备                                          |
| 记录日志 | 否       | 本系统设置 | 启用后，系统会记录该设备的连接/断开等操作日志                                   |
| 描述信息 | 否       | 备注       | 对该设备的补充说明，如"二楼机房主服务器"、"无人机控制终端"                      |

> **重要说明**：用户名、密码、端口号均为**目标远程设备**的信息，而非本系统的登录信息。例如，要连接一台 Windows 电脑，需填写该电脑的 Windows 登录用户名和密码。

填写完成后，点击 **"确认"** 即可添加新的远程连接。数据会持久化保存到后端数据库，重启服务后不会丢失。

> **提示**：如果通过无人机管理模块注册了无人机，系统会自动创建对应的远程设备，无需在此手动添加。自动创建的设备会继承无人机注册时填写的 IP 地址、端口、协议和用户名等信息。

#### 搜索连接

用户可在搜索表单中填写以下条件，点击 **"搜索"** 按钮进行筛选：

| 条件     | 说明                               |
| -------- | ---------------------------------- |
| 设备名称 | 按名称模糊匹配                     |
| IP地址   | 按 IP 地址模糊匹配                 |
| 端口号   | 按端口号精确匹配                   |
| 连接类型 | 按协议精确筛选（RDP/VNC/SSH/全部） |
| 用户名   | 按用户名模糊匹配                   |
| 描述     | 按描述信息模糊匹配                 |

#### 刷新连接

点击 **"刷新"** 按钮，系统会重新从后端加载远程连接列表，获取最新的设备状态和连接信息。

#### 断开所有连接

当用户需要断开所有已建立的远程连接时，点击 **"断开所有连接"** 按钮，系统会弹出确认窗口，确认后将断开所有在线连接，同时带宽使用进度条会归零。

> **注意**：断开连接后，设备将无法进行远程控制，需重新点击"连接"按钮恢复。

### 2.2 设备状态监测功能

#### 设备状态统计

页面展示了四个统计卡片，让用户直观了解当前远程连接的整体情况：

| 统计项   | 说明                                       |
| -------- | ------------------------------------------ |
| 在线设备 | 当前处于连接状态的设备数量                 |
| 离线设备 | 当前未连接的设备数量                       |
| 总连接数 | 系统中所有已添加的设备总数                 |
| 带宽使用 | 在线设备占总设备的百分比，反映当前网络负载 |

#### 设备状态显示

在远程连接列表中，每个设备都显示了其状态：

- **绿色圆点 + "在线"**：设备已连接，可进行远程操控
- **红色圆点 + "离线"**：设备未连接，需点击"连接"按钮建立连接

#### 带宽使用进度条

页面中部展示一个带宽使用进度条，以可视化方式展示当前在线设备占比。进度条会随设备连接/断开状态实时更新。

### 2.3 远程连接操作功能

#### 连接设备

对于列表中的离线设备，用户可点击 **“连接”** 按钮，系统会根据设备的连接类型自动选择对应的连接方式：

- **VNC 连接**：系统会在新标签页打开内置的 noVNC 客户端，通过 WebSocket 代理连接到目标设备的 VNC Server，在浏览器内直接操控远程桌面
- **SSH 连接**：系统会弹出密码输入框，确认后在新标签页打开内置的 SSH 终端（基于 xterm.js），通过 WebSocket 代理连接到目标设备的 SSH 服务，在浏览器内直接执行命令行操作
- **RDP 连接**：系统会自动生成一个 `.rdp` 连接文件并下载到本地，用户双击该文件即可通过 Windows 自带的远程桌面客户端（mstsc.exe）连接目标设备

> **前提条件**：目标设备必须已开机、运行对应的远程服务（VNC Server / RDP 服务 / SSH 服务），且与本机网络互通。

#### 断开设备连接

对于已连接的在线设备，操作列会显示 **"断开"** 按钮，点击后即可断开该设备的远程连接。

#### 删除设备

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中移除该设备记录。

> **注意**：删除操作不可恢复，请谨慎操作。如果设备是通过无人机注册自动创建的，建议通过无人机管理模块删除无人机（会级联删除关联设备），而非在此单独删除设备记录。

### 2.4 连接统计功能

页面底部展示两个统计图表：

- **连接状态分布**（饼图）：以环形图形式展示在线和离线设备的占比情况，用户可直观了解当前远程连接的状态分布。
- **连接类型分布**（柱状图）：以水平柱状图形式展示不同连接类型（RDP、VNC、SSH）的使用数量，帮助用户了解各种连接协议的使用情况。

图表数据会随设备列表的变化实时更新。

### 2.5 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 请确保设备的IP地址和端口号正确，以保证连接成功。
2. 不同的连接类型可能需要不同的配置，请根据实际情况设置。
3. 断开连接后，设备将无法进行远程控制，需重新点击"连接"按钮恢复。

## 3. 技术实现

### 3.1 后端接口

| 接口                            | 方法      | 说明                                                                                                |
| ------------------------------- | --------- | --------------------------------------------------------------------------------------------------- |
| `/api/devices`                | GET       | 获取所有设备列表（支持 name、protocol 查询参数）                                                    |
| `/api/devices`                | POST      | 新增远程连接设备（JSON 格式）                                                                       |
| `/api/devices/:id`            | DELETE    | 删除指定 ID 的设备                                                                                  |
| `/api/devices/:id/status`     | POST      | 设置设备状态（online/offline）                                                                      |
| `/api/devices/disconnect-all` | POST      | 断开所有在线连接                                                                                    |
| `/api/vnc/ws`                 | WebSocket | VNC 代理，将浏览器 WebSocket 转发到目标设备的 VNC Server                                            |
| `/api/ssh/ws`                 | WebSocket | SSH 代理，后端通过 `golang.org/x/crypto/ssh` 连接目标设备，将终端 I/O 通过 WebSocket 桥接到浏览器 |

### 3.2 数据存储

后端使用 SQLite 数据库的 `devices` 表存储设备信息，包含以下字段：

| 字段              | 类型     | 说明                       |
| ----------------- | -------- | -------------------------- |
| id                | INTEGER  | 主键，自增                 |
| name              | TEXT     | 设备名称                   |
| ip                | TEXT     | IP 地址                    |
| protocol          | TEXT     | 连接协议（VNC/RDP/SSH）    |
| port              | INTEGER  | 端口号                     |
| username          | TEXT     | 目标设备登录用户名         |
| password          | TEXT     | 目标设备登录密码           |
| auto_connect      | INTEGER  | 是否自动连接（0/1）        |
| log_enabled       | INTEGER  | 是否记录日志（0/1）        |
| description       | TEXT     | 描述信息                   |
| status            | TEXT     | 设备状态（online/offline） |
| created_at        | DATETIME | 创建时间                   |
| updated_at        | DATETIME | 更新时间                   |
| last_connected_at | DATETIME | 最后连接时间               |

### 3.3 前端技术

- 使用 Chart.js 绘制连接状态分布饼图和连接类型分布柱状图
- 使用模态弹窗实现新增连接和删除确认交互
- 搜索筛选在前端内存中执行，响应速度快
- 设备列表每 8 秒自动刷新，保持状态同步
- VNC 连接通过 noVNC 库实现浏览器内远程桌面
- SSH 连接通过 xterm.js 库实现浏览器内终端，支持终端窗口自适应和 256 色显示
- RDP 连接通过前端生成 `.rdp` 文件并触发下载，由系统自带客户端打开

### 3.4 连接协议说明

| 协议 | 默认端口 | 适用场景         | 目标设备要求                                  |
| ---- | -------- | ---------------- | --------------------------------------------- |
| VNC  | 5900     | 跨平台远程桌面   | 安装并启动 VNC Server（如 TightVNC、RealVNC） |
| RDP  | 3389     | Windows 远程桌面 | 开启 Windows 远程桌面功能                     |
| SSH  | 22       | Linux 终端控制   | 安装并启动 SSH 服务（如 OpenSSH）             |

### 3.5 三种连接方式的具体实现步骤

#### 3.5.1 VNC 连接实现步骤

**目标设备准备（被控端）：**

1. 在目标设备上安装 VNC Server 软件（推荐 TightVNC 或 RealVNC）
2. 启动 VNC Server，设置连接密码
3. 确保 VNC Server 监听端口（默认 5900）未被防火墙拦截
4. 确保目标设备与本机在同一局域网，或通过 VPN/端口转发互通

**本系统操作（控制端）：**

1. 在远程桌面控制页面点击 **“新增连接”**
2. 填写设备名称、目标设备 IP 地址、连接类型选择 **VNC**、端口号填写 **5900**（或自定义端口）
3. 填写用户名（可填任意标识，VNC 协议主要依赖密码认证）
4. 点击 **“确认”** 保存设备信息
5. 在设备列表中点击 **“连接”** 按钮
6. 系统自动在新标签页打开 noVNC 客户端页面（`vnc.html`）
7. noVNC 通过 WebSocket 连接到后端 `/api/vnc/ws` 接口
8. 后端通过 `net.Dial("tcp", target)` 建立到目标设备 VNC Server 的 TCP 连接
9. 后端将 WebSocket 和 TCP 连接双向桥接，实现浏览器内远程桌面显示和操控

**技术流程图：**

```
浏览器(noVNC) ←WebSocket→ Go后端(VNCProxyWS) ←TCP→ 目标设备(VNC Server:5900)
```

---

#### 3.5.2 SSH 连接实现步骤

**目标设备准备（被控端）：**

1. 目标设备需为 Linux 系统（或其他支持 SSH 的系统）
2. 确保 SSH 服务已安装并启动（`sudo systemctl start sshd`）
3. 确保 SSH 端口（默认 22）未被防火墙拦截
4. 确保目标设备与本机网络互通

**本系统操作（控制端）：**

1. 在远程桌面控制页面点击 **“新增连接”**
2. 填写设备名称、目标设备 IP 地址、连接类型选择 **SSH**、端口号填写 **22**（或自定义端口）
3. 填写目标设备的 **Linux 系统用户名**（如 `root`、`ubuntu` 等）
4. 点击 **“确认”** 保存设备信息
5. 在设备列表中点击 **“连接”** 按钮
6. 系统弹出密码输入框，输入目标设备的 **Linux 系统密码**
7. 系统自动在新标签页打开 SSH 终端页面（`ssh.html`）
8. 前端 xterm.js 通过 WebSocket 连接到后端 `/api/ssh/ws` 接口
9. 后端使用 `golang.org/x/crypto/ssh` 库建立到目标设备的 SSH 连接
10. 后端请求 PTY（伪终端）并启动 Shell，将终端的 stdin/stdout/stderr 与 WebSocket 双向桥接
11. 用户在浏览器终端中输入的命令通过 WebSocket 发送到后端，后端转发到 SSH 会话的 stdin
12. 目标设备的命令输出通过 SSH 会话的 stdout 返回后端，后端通过 WebSocket 推送到浏览器终端显示

**技术流程图：**

```
浏览器(xterm.js) ←WebSocket→ Go后端(SSHProxyWS) ←SSH协议→ 目标设备(SSH Server:22)
                                        │
                              golang.org/x/crypto/ssh
                              │─ ssh.Dial() 建立连接
                              │─ session.RequestPty() 请求伪终端
                              │─ session.Shell() 启动 Shell
                              └─ stdin/stdout/stderr 双向桥接
```

**SSH 终端特性：**

- 支持 xterm-256color 终端类型，完整显示彩色输出
- 支持终端窗口自动调整大小（通过 `\x01` + JSON 消息发送 resize 指令）
- 支持光标闪烁、滚动、选择复制等标准终端操作
- 连接状态实时显示（连接中/已连接/已断开）

---

#### 3.5.3 RDP 连接实现步骤

**目标设备准备（被控端）：**

1. 目标设备需为 Windows 系统（Windows 10/11 专业版或企业版）
2. 开启远程桌面功能：设置 → 系统 → 远程桌面 → 启用远程桌面
3. 确保 RDP 端口（默认 3389）未被防火墙拦截
4. 确保目标设备与本机网络互通

**本系统操作（控制端）：**

1. 在远程桌面控制页面点击 **“新增连接”**
2. 填写设备名称、目标设备 IP 地址、连接类型选择 **RDP**、端口号填写 **3389**（或自定义端口）
3. 填写目标设备的 **Windows 登录用户名**（如 `Administrator`、`user` 等）
4. 点击 **“确认”** 保存设备信息
5. 在设备列表中点击 **“连接”** 按钮
6. 系统自动生成一个 `.rdp` 连接文件，包含以下配置信息：
   - `full address` — 目标设备 IP 和端口
   - `username` — 登录用户名
   - `screen mode id` — 全屏模式
   - `desktopwidth/desktopheight` — 分辨率 1920×1080
   - `session bpp` — 32 位色彩
   - `prompt for credentials` — 提示输入密码
7. 浏览器自动下载该 `.rdp` 文件
8. 用户双击下载的 `.rdp` 文件，Windows 系统自动调用 `mstsc.exe`（远程桌面连接客户端）打开
9. 在弹出的 Windows 远程桌面窗口中输入目标设备的 Windows 密码，即可连接

**技术流程图：**

```
本系统前端 → 生成 .rdp 文件 → 浏览器下载 → 用户双击 → mstsc.exe ←RDP协议→ 目标设备(RDP:3389)
```

> **说明**：RDP 是微软的私有协议，没有成熟的纯浏览器实现方案，因此采用生成 `.rdp` 文件 + 系统自带客户端的方式，这是业界通用做法，兼容性最好。

---

#### 3.5.4 三种连接方式对比

| 特性               | VNC                 | SSH                | RDP                      |
| ------------------ | ------------------- | ------------------ | ------------------------ |
| 是否在浏览器内完成 | ✅ 是               | ✅ 是              | ❌ 否（需系统客户端）    |
| 操控方式           | 图形化桌面          | 命令行终端         | 图形化桌面               |
| 后端代理           | TCP 转发            | SSH 库连接         | 无（前端生成文件）       |
| 前端库             | noVNC               | xterm.js           | 无（系统自带 mstsc.exe） |
| 适用系统           | 跨平台              | Linux/macOS        | Windows                  |
| 默认端口           | 5900                | 22                 | 3389                     |
| 带宽消耗           | 高（图形传输）      | 低（纯文本）       | 中高（图形传输，有压缩） |
| 密码输入时机       | VNC Server 内部认证 | 点击连接时弹出输入 | 打开 .rdp 文件后输入     |

## 4. 使用指南

1. **添加设备**：点击“新增连接”按钮，填写目标设备的名称、IP、端口、用户名等信息后点击“确认”。
2. **连接 VNC 设备**：点击“连接”按钮，系统自动在新标签页打开远程桌面，直接在浏览器内操控。
3. **连接 SSH 设备**：点击“连接”按钮，输入密码后系统在新标签页打开终端，可直接执行 Linux 命令。
4. **连接 RDP 设备**：点击“连接”按钮，系统自动下载 `.rdp` 文件，双击即可用 Windows 远程桌面连接。
5. **断开连接**：点击在线设备的“断开”按钮，或点击“断开所有连接”一键断开全部。
6. **搜索设备**：在搜索表单中输入条件，点击“搜索”按钮快速定位设备。
7. **删除设备**：点击“删除”按钮，在确认弹窗中确认后移除设备。
8. **查看统计**：页面底部的图表实时展示连接状态和类型分布。

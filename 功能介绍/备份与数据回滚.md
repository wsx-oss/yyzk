# 备份与数据回滚 功能介绍

## 一、功能概述

备份与数据回滚是"CloudControl"的数据安全模块，提供**自动备份、手动备份、数据恢复与回滚、备份文件管理**能力。该模块保障在数据库迁移、仿真压测、系统升级等高风险操作中，数据可随时回退到安全状态。

---

## 二、设计背景

### 2.1 为什么需要备份与回滚？

1. **MySQL 迁移保障**：从 SQLite 迁移到 MySQL 后，需要定期备份防止云端数据库意外丢失
2. **仿真压测兜底**：大规模模拟无人机群产生海量数据，异常写入可能污染生产数据
3. **操作可逆**：误删无人机、误修改任务配置、误清告警等操作均需要回退能力
4. **合规审计**：备份记录提供完整的数据变更审计轨迹

### 2.2 技术方案

- **后端**：`internal/handlers/backup.go`，基于 `mysqldump` 原理实现纯 Go SQL 备份
- **前端**：`web/modules/backup.html`，备份列表 + 手动触发 + 一键恢复 + 进度追踪
- **存储**：备份文件保存在 `data/backups/` 目录，SQL 格式
- **记录**：`backup_records` 表持久化所有备份和恢复操作

---

## 三、核心功能

### 3.1 自动备份

- **周期**：24 小时自动执行一次
- **首次备份**：服务启动后立即执行首次备份
- **保留策略**：默认保留最近 10 份，可通过清理接口调整
- **失败告警**：备份失败自动写入 `notifications` 表，前端铃铛提醒

### 3.2 手动备份

- **触发方式**：前端"立即备份"按钮 或 `POST /api/backup/manual`
- **实时反馈**：操作后即时 Toast 提示
- **记录**：备份完成后自动写入 `backup_records`（含大小、行数、耗时）

### 3.3 数据恢复

支持两种恢复方式：

| 方式 | 说明 |
|------|------|
| 从备份记录恢复 | 选择历史备份 ID，一键恢复到该时间点 |
| 上传 SQL 文件恢复 | 上传外部 SQL 备份文件进行恢复 |

恢复流程：
1. 检查备份文件有效性
2. 自动创建 `pre_restore` 安全备份（恢复前快照）
3. 异步执行恢复（前端显示进度弹窗）
4. 恢复完成后刷新备份列表

### 3.4 备份管理

| 功能 | 说明 |
|------|------|
| 备份列表 | 展示所有备份记录（时间、类型、状态、大小、行数、耗时） |
| 下载备份 | 下载 SQL 备份文件到本地 |
| 删除备份 | 删除单条备份记录及对应文件 |
| 批量清理 | 保留最近 N 份，清理旧备份 |
| 自动备份开关 | 启用/停用自动备份 |

## 四、数据库表

### backup_records

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| type | VARCHAR(20) | 备份类型：auto / manual / pre_restore |
| status | VARCHAR(20) | 状态：running / success / failed |
| file_path | TEXT | 备份文件路径 |
| file_size | BIGINT | 文件大小（字节） |
| row_count | INT | 备份行数 |
| operator | VARCHAR(100) | 操作人 |
| error_msg | TEXT | 失败原因 |
| created_at | DATETIME | 创建时间 |
| finished_at | DATETIME | 完成时间 |

---

## 五、API 接口

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/backup/list` | 备份记录列表 |
| GET | `/api/backup/status` | 备份状态概览（总数/成功/失败/空间） |
| POST | `/api/backup/manual` | 手动触发备份 |
| POST | `/api/backup/restore/:id` | 从指定备份恢复数据 |
| POST | `/api/backup/restore-upload` | 上传 SQL 文件恢复数据 |
| GET | `/api/backup/restore-progress` | 恢复进度查询（轮询） |
| DELETE | `/api/backup/:id` | 删除备份记录及文件 |
| GET | `/api/backup/download/:id` | 下载备份文件 |
| POST | `/api/backup/auto-toggle` | 开关自动备份 |
| POST | `/api/backup/cleanup` | 清理旧备份（保留最近 N 份） |

---

## 六、前端页面

`backup.html` 提供完整的备份管理界面：

1. **状态概览卡片**：备份总数、成功数、失败数、已用空间
2. **操作按钮区**：立即备份、清理旧备份、开关自动备份
3. **备份列表表格**：时间、类型标签、状态标签、大小、行数、耗时、操作（恢复/下载/删除）
4. **恢复进度弹窗**：实时显示恢复阶段和进度百分比
5. **确认弹窗**：删除、恢复、清理等危险操作均需二次确认

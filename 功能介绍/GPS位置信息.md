# GPS/位置信息

## 1. 功能概述

GPS/位置信息模块是“CloudControl”新增的功能模块，专门用于实时采集、展示和管理无人机的GPS位置数据。该模块支持无人机的添加、编辑、删除、位置推送、历史轨迹查看等操作，并集成了电子围栏功能，当无人机超出预设围栏范围时自动触发报警。通过地图可视化界面，操作人员可直观掌握所有无人机的实时位置分布和移动状态。本模块与“飞行任务管理”模块联动，创建飞行任务时可选择具体的无人机设备执行任务。

## 2. 功能详细说明

### 2.1 设备管理功能

#### 添加设备

用户可点击页面右上角的 **“添加设备”** 按钮，打开添加无人机的弹窗。在该窗口中，用户需填写以下信息：

| 字段         | 是否必填 | 说明                                                       |
| ------------ | -------- | ---------------------------------------------------------- |
| 无人机名称   | 是       | 无人机的名称                                                |
| 初始经度     | 否       | 无人机初始经度坐标                                         |
| 初始纬度     | 否       | 无人机初始纬度坐标                                         |
| 初始海拔     | 否       | 无人机初始海拔高度（米）                                   |
| 电子围栏     | 否       | 是否启用电子围栏（开关控件）                               |
| 围栏中心经度 | 否       | 电子围栏中心点经度（启用围栏时填写）                       |
| 围栏中心纬度 | 否       | 电子围栏中心点纬度（启用围栏时填写）                       |
| 围栏半径     | 否       | 电子围栏半径（米，默认500m）                               |

填写完成后，点击 **“确认添加”** 即可添加新的无人机。新设备默认状态为“在线”，初始位置会记录到历史轨迹中。添加的无人机可在“飞行任务管理”模块中被选择执行飞行任务。

#### 编辑设备

在设备列表中，点击操作列的 **“编辑”** 按钮，系统会弹出编辑窗口，预填该无人机的所有信息。用户可修改无人机名称、电子围栏设置等字段后点击 **“确认保存”** 保存更改。

#### 删除设备

点击操作列的 **“删除”** 按钮，系统弹出确认窗口，确认后将从系统中永久移除该无人机及其所有历史轨迹、围栏报警记录，同时级联删除该无人机在"电池监控"模块中的所有电池记录和电池报警。

> **注意**：删除操作不可恢复，且会同步清除该无人机的电池监控数据，请谨慎操作。

### 2.2 GPS数据采集与推送

#### 手动位置推送

点击设备列表中的 **“推送”** 按钮，可模拟Agent上报GPS数据。推送弹窗中可设置以下参数：

| 参数     | 说明                       |
| -------- | -------------------------- |
| 经度     | 设备当前经度               |
| 纬度     | 设备当前纬度               |
| 海拔     | 设备当前海拔（米）         |
| 速度     | 设备当前速度（米/秒）      |
| 航向角   | 设备当前航向（0-360度）    |
| 精度     | GPS定位精度（米）          |

推送位置时，系统会自动：

- 更新设备的实时位置信息
- 将位置记录写入历史轨迹
- 检查电子围栏，超出范围时自动生成围栏报警

> **说明**：推送按钮会自动在当前位置附近随机生成坐标，方便快速测试。

#### Agent 自动推送（实时采集）

无人机端运行 `hw-agent` 程序并启用 Push 模式后，Agent 会**自动采集 GPS 位置数据**并定时推送到主控端，无需手动操作。

```bash
# 在无人机端运行（每2秒自动推送硬件+GPS+电池数据）
./hw-agent -server http://主控端IP:8080 -push-interval 2
```

Agent 推送 GPS 数据时通过 `POST /api/gps/push` 接口，使用 `agent_id`（默认为主机名）标识自身。后端会自动根据 `agent_id` 匹配已有的 GPS 设备（按 `name` 字段匹配），如果不存在则**自动创建**新的 GPS 设备记录。

推送的数据包括：经纬度、海拔、速度、航向角、精度。每次推送都会自动记录历史轨迹并检查电子围栏。

> **数据流**：无人机 hw-agent → `POST /api/gps/push` → 自动匹配/创建设备 → 更新位置 → 记录历史 → 检查围栏

### 2.3 实时监控

点击页面右上角的 **"开启实时监控"** 按钮，系统会通过 **WebSocket 长连接**订阅 GPS 事件流（`WS /api/gps/stream`）。当后端收到新的 GPS 上报（手动推送或 Agent Push）时，会广播 `gps_update` 事件，前端收到事件后自动刷新设备列表、地图标记、统计数据和围栏报警记录，实现**低延迟的实时跟踪**。

- 开启后按钮显示为 **"🟢 实时监控中（WebSocket）"**，再次点击即可关闭
- WebSocket 断线后自动每 3 秒尝试重连
- 实时监控期间，地图上的无人机标记会随着 GPS 上报自动更新
- 统计卡片（在线/离线数、报警数等）会在事件触发时自动刷新

在线/离线判定：

- 设备最近 30 秒有上报视为“在线”（前端显示），长时间无上报会显示为“离线”。
- 后端会在后台定时检测：若超过 60 秒未上报，会把 `gps_devices.status` 标记为 `离线`，并将关联无人机状态置为 `offline`。

### 2.4 实时位置地图

#### 地图展示

页面中部展示基于 Leaflet + OpenStreetMap 的实时位置地图，功能包括：

- **设备标记**：在线无人机显示为绿色圆点，离线无人机显示为红色圆点
- **弹出信息**：点击无人机标记可查看名称、坐标、速度、状态等信息
- **电子围栏**：启用围栏的设备会在地图上显示紫色虚线圆圈
- **自动缩放**：地图会自动调整视野以包含所有设备标记
- **图例说明**：地图右上角显示在线/离线/围栏的图例

### 2.5 电子围栏

#### 围栏设置

在添加或编辑设备时，可通过开关控件启用电子围栏。启用后需设置：

- **围栏中心**：围栏中心点的经纬度坐标
- **围栏半径**：围栏的半径范围（米）

#### 围栏报警

当无人机位置超出围栏范围时，系统会自动：

1. 在 `gps_fence_alerts` 表中生成一条围栏报警记录
2. 同时在全局 `alerts` 表中生成一条报警，可在“异常报警”模块中查看
3. 报警信息包含无人机名称、触发位置、围栏中心、围栏半径、偏离距离等

围栏距离计算使用 **Haversine 公式**，精确计算地球表面两点间的距离。

#### 围栏报警管理

在“围栏报警记录”标签页中，可查看所有围栏报警记录，支持：

- 查看报警详情（设备名称、触发位置、围栏参数、偏离距离）
- 确认报警（将报警标记为已确认）
- 分页浏览

3. 启用电子围栏后，当设备位置超出围栏范围时，系统会自动生成围栏报警记录。
4. 点击地图上的设备标记可查看设备详细信息，点击"轨迹"可查看历史移动轨迹。

## 3. 技术实现

### 3.1 后端接口

| 接口                              | 方法   | 说明                                                                           |
| --------------------------------- | ------ | ------------------------------------------------------------------------------ |
| `/api/gps/devices`              | GET    | 获取无人机列表（支持 name、status 筛选 + 分页）                                |
| `/api/gps/devices`              | POST   | 添加无人机（JSON 格式）                                                        |
| `/api/gps/devices/stats`        | GET    | 获取GPS统计数据（设备总数、在线/离线数、围栏启用数、未处理报警数）             |
| `/api/gps/devices/:id`          | GET    | 获取指定设备详情                                                               |
| `/api/gps/devices/:id`          | PUT    | 编辑指定设备信息                                                               |
| `/api/gps/devices/:id`          | DELETE | 删除指定设备及其历史数据和报警记录                                             |
| `/api/gps/devices/:id/push`     | POST   | 推送GPS位置数据（自动记录历史、检查围栏）                                      |
| `/api/gps/devices/:id/history`  | GET    | 获取指定设备的位置历史轨迹（支持 limit 参数）                                  |
| `/api/gps/push`                 | POST   | Agent推送GPS数据（通过agent_id自动匹配/创建设备，记录历史，检查围栏）          |
| `/api/gps/fence-alerts`         | GET    | 获取围栏报警记录列表（分页）                                                   |
| `/api/gps/fence-alerts/:id/ack` | POST   | 确认围栏报警                                                                   |
| `/api/gps/stream`              | WS     | WebSocket 实时事件流（推送 `gps_update` 事件）                                 |

### 3.2 数据存储

后端使用 SQLite 数据库存储GPS数据，涉及三张表：

#### gps_devices 表

| 字段          | 类型     | 说明                                     |
| ------------- | -------- | ---------------------------------------- |
| id            | INTEGER  | 主键，自增                               |
| name          | TEXT     | 设备名称                                 |
| device_type   | TEXT     | 设备类型（固定为“无人机”）              |
| latitude      | REAL     | 当前纬度                                 |
| longitude     | REAL     | 当前经度                                 |
| altitude      | REAL     | 当前海拔（米）                           |
| speed         | REAL     | 当前速度（米/秒）                        |
| heading       | REAL     | 当前航向角（度）                         |
| accuracy      | REAL     | GPS精度（米）                            |
| status        | TEXT     | 设备状态（在线/离线）                    |
| fence_enabled | INTEGER  | 是否启用电子围栏（0/1）                  |
| fence_lat     | REAL     | 围栏中心纬度                             |
| fence_lng     | REAL     | 围栏中心经度                             |
| fence_radius  | REAL     | 围栏半径（米）                           |
| last_update   | DATETIME | 最后位置更新时间                         |
| created_at    | DATETIME | 创建时间                                 |

索引：`idx_gps_devices_name`、`idx_gps_devices_status`

#### gps_history 表

| 字段       | 类型     | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| id         | INTEGER  | 主键，自增                               |
| device_id  | INTEGER  | 关联的设备 ID（外键）                    |
| latitude   | REAL     | 纬度                                     |
| longitude  | REAL     | 经度                                     |
| altitude   | REAL     | 海拔（米）                               |
| speed      | REAL     | 速度（米/秒）                            |
| heading    | REAL     | 航向角（度）                             |
| created_at | DATETIME | 记录时间                                 |

索引：`idx_gps_history_device`、`idx_gps_history_created`

#### gps_fence_alerts 表

| 字段         | 类型     | 说明                                     |
| ------------ | -------- | ---------------------------------------- |
| id           | INTEGER  | 主键，自增                               |
| device_id    | INTEGER  | 关联的设备 ID（外键）                    |
| device_name  | TEXT     | 设备名称                                 |
| latitude     | REAL     | 触发位置纬度                             |
| longitude    | REAL     | 触发位置经度                             |
| fence_lat    | REAL     | 围栏中心纬度                             |
| fence_lng    | REAL     | 围栏中心经度                             |
| fence_radius | REAL     | 围栏半径（米）                           |
| distance     | REAL     | 偏离距离（米）                           |
| message      | TEXT     | 报警消息                                 |
| acknowledged | INTEGER  | 是否已确认（0/1）                        |
| created_at   | DATETIME | 报警时间                                 |

索引：`idx_gps_fence_alerts_device`

### 3.3 前端技术

- 使用 Leaflet + OpenStreetMap 实现地图可视化，展示无人机实时位置和电子围栏
- 使用 Chart.js 绘制设备状态分布和围栏状态分布环形图
- 使用 Tab 标签页切换设备列表和围栏报警记录
- 使用模态弹窗实现添加/编辑设备、推送位置、查看详情、查看轨迹等交互
- 设备详情弹窗中嵌入独立地图，展示设备位置和围栏范围
- 历史轨迹弹窗中以折线绘制移动路径，标注起点和最新位置
- 电子围栏使用 Toggle Switch 开关控件，启用后动态显示围栏参数输入框
- 围栏距离计算使用 Haversine 公式（后端实现）
- 搜索筛选通过后端 API 参数传递执行，支持 2 种条件组合筛选（名称、状态）
- 删除操作前弹出确认窗口，防止误操作
- 实时监控功能通过 WebSocket 事件驱动实现（`WS /api/gps/stream`），GPS 更新时即时推送并刷新
- WebSocket 断线后自动重连（3秒间隔），连接状态实时显示在按钮上

## 4. 使用指南

1. **添加设备**：点击"添加设备"按钮，填写设备名称、类型、初始坐标，可选启用电子围栏后点击"确认添加"。
2. **手动推送位置**：点击设备的"推送"按钮，在弹窗中设置坐标和运动参数后点击"推送"，模拟GPS数据上报。
3. **Agent自动推送**：在无人机端运行 `./hw-agent -server http://主控端IP:8080 -push-interval 2`，Agent会自动采集GPS数据并推送，首次推送时自动创建设备。
4. **开启实时监控**：点击"开启实时监控"按钮，系统通过 WebSocket 实时接收 GPS 更新事件，配合 Agent 推送实现低延迟实时跟踪。
5. **查看地图**：地图自动显示所有设备的实时位置，点击标记查看详情。
6. **查看详情**：点击"详情"按钮，在弹窗中查看设备完整信息和位置地图。
7. **查看轨迹**：在详情弹窗中点击"查看轨迹"，查看设备的历史移动路径。
8. **编辑设备**：点击"编辑"按钮，修改设备信息和围栏设置。
9. **删除设备**：点击"删除"按钮，确认后移除设备及所有历史数据。
10. **围栏报警**：切换到"围栏报警记录"标签页，查看和确认围栏报警。
11. **搜索设备**：在筛选区域输入条件，点击"搜索"按钮快速定位。
12. **查看统计**：页面底部图表实时展示设备状态和类型分布。

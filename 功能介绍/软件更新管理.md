# 软件更新管理

## 1. 功能概述

软件更新管理模块是"CloudControl"的重要功能之一，用于发布、管理和追踪软件的更新信息。

> **说明**：当前版本的“软件更新”主要管理的是**更新元数据**（名称/版本/描述/状态等）与“检查最新版本”能力；不包含自动下载二进制包并执行安装的完整流水线。

## 2. 功能详细说明

### 2.1 更新管理功能

#### 新增更新

用户可点击页面右上角的 **"新增更新"** 按钮，打开新增软件更新的弹窗。在该窗口中，用户需填写以下信息：

| 字段       | 是否必填 | 说明                                                       |
| ---------- | -------- | ---------------------------------------------------------- |
| 更新名称   | 是       | 更新的标识名称，如"安全补丁 v2.1"、"功能升级包"            |
| 版本号     | 是       | 更新的版本号，如 `1.0.0`、`2.3.1`                         |
| 更新类型   | 是       | 更新类型：功能更新、安全更新、性能优化                     |
| 更新大小   | 否       | 更新包的大小，如 `20MB`、`150MB`                          |
| 状态       | 是       | 更新状态：待发布、已发布、已撤回                           |
| 发布时间   | 否       | 更新的发布日期                                             |
| 自动更新   | 否       | 启用后，系统会自动下载并安装该更新                         |
| 强制更新   | 否       | 启用后，用户下次启动软件时会强制下载并安装该更新           |
| 更新描述   | 否       | 对该更新的详细说明，如更新内容、修复的问题等               |

填写完成后，点击 **"确认"** 即可添加新的软件更新。数据会持久化保存到后端数据库，重启服务后不会丢失。

#### 编辑更新

在更新列表中，点击操作列的 **"编辑"** 按钮，系统会弹出编辑窗口，预填该更新的所有信息。用户可修改任意字段后点击 **"确认"** 保存更改。编辑操作支持部分更新，仅修改的字段会被更新到数据库。

#### 删除更新

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中永久移除该更新记录。

> **注意**：删除操作不可恢复，请谨慎操作。

#### 搜索更新

用户可在搜索表单中填写以下条件，点击 **"搜索"** 按钮进行筛选：

| 条件         | 说明                                           |
| ------------ | ---------------------------------------------- |
| 更新名称     | 按名称模糊匹配                                 |
| 更新版本号   | 按版本号模糊匹配                               |
| 更新类型     | 按类型精确筛选（功能更新/安全更新/性能优化/全部） |
| 更新状态     | 按状态精确筛选（待发布/已发布/已撤回/全部）     |
| 更新大小     | 按大小模糊匹配                                 |
| 发布时间     | 按发布日期筛选                                 |
| 自动更新开关 | 通过 Toggle 开关筛选启用自动更新的记录         |
| 强制更新开关 | 通过 Toggle 开关筛选启用强制更新的记录         |

> **说明**：所有筛选条件均为可选，支持组合筛选。Toggle 开关仅在启用时参与筛选。

### 2.2 导入导出功能

#### 导入更新

点击 **"导入更新"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的文件进行批量导入。JSON 文件格式要求为数组格式，每个元素包含更新的各项字段信息。

导入时系统会自动校验数据：
- 更新名称和版本号为必填项，缺失的记录会被跳过
- 未填写的状态默认为"待发布"，未填写的类型默认为"功能更新"
- 导入完成后显示成功导入的记录数量

#### 导出更新

点击 **"导出更新"** 按钮，系统会将当前列表中的所有更新数据导出为 JSON 文件，文件名格式为 `软件更新_YYYY-MM-DD.json`。导出字段包括：更新名称、版本号、类型、大小、状态、发布时间、自动更新、强制更新、描述。

### 2.3 更新列表

页面以表格形式展示所有软件更新的详细信息，包括以下列：

| 列名       | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| 更新名称   | 更新的标识名称                                         |
| 更新版本号 | 版本号（加粗显示）                                     |
| 更新类型   | 功能更新 / 安全更新 / 性能优化                         |
| 更新大小   | 更新包大小                                             |
| 更新状态   | 以彩色徽章显示（绿色已发布/蓝色待发布/红色已撤回）     |
| 发布时间   | 更新的发布日期                                         |
| 自动更新   | 以标签显示（蓝色"启用"/灰色"禁用"）                   |
| 强制更新   | 以标签显示（蓝色"启用"/灰色"禁用"）                   |
| 操作       | 编辑、删除按钮                                         |

### 2.4 更新统计功能

页面底部展示两个统计图表：

- **更新状态分布**（饼图）：以环形图形式展示待发布、已发布、已撤回三种状态的更新占比情况，用户可直观了解当前软件更新的发布状况。
- **更新类型分布**（柱状图）：以水平柱状图形式展示不同类型更新（功能更新、安全更新、性能优化）的数量对比，帮助用户了解更新的类型构成。

图表数据会随更新列表的变化实时更新。

### 2.5 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 软件更新管理用于发布和管理软件的更新信息，请谨慎操作。
2. 已发布的更新撤回后无法再次发布，已撤回的更新无法再次发布。
3. 强制更新会在用户下次启动软件时自动下载并安装软件。

## 3. 技术实现

### 3.1 后端接口

| 接口                       | 方法   | 说明                                                                                                     |
| -------------------------- | ------ | -------------------------------------------------------------------------------------------------------- |
| `/api/updates/list`      | GET    | 获取更新列表（支持 name、version、type、status、size、publish_date、auto_update、force_update 筛选 + 分页） |
| `/api/updates/add`       | POST   | 新增软件更新（JSON：`name`/`version`/`description`/`status`/`type`/`size`/`auto_update`/`force_update`/`publish_date`） |
| `/api/updates/:id`       | PUT    | 编辑指定 ID 的更新信息（支持部分更新）                                                                   |
| `/api/updates/:id`       | DELETE | 删除指定 ID 的更新记录                                                                                   |
| `/api/updates/import`    | POST   | 批量导入更新（接收 JSON 数组）                                                                           |
| `/api/updates/stats`     | GET    | 获取更新统计数据（状态分布 + 类型分布）                                                                  |
| `/api/updates/check`     | GET    | 检查最新版本（支持 current 参数对比）                                                                    |

### 3.2 数据存储

后端使用 SQLite 数据库的 `updates` 表存储更新信息，包含以下字段：

| 字段         | 类型     | 说明                                     |
| ------------ | -------- | ---------------------------------------- |
| id           | INTEGER  | 主键，自增                               |
| name         | TEXT     | 更新名称                                 |
| version      | TEXT     | 版本号                                   |
| description  | TEXT     | 更新描述                                 |
| status       | TEXT     | 更新状态（待发布/已发布/已撤回）         |
| type         | TEXT     | 更新类型（功能更新/安全更新/性能优化）   |
| size         | TEXT     | 更新大小                                 |
| auto_update  | INTEGER  | 是否自动更新（0/1）                      |
| force_update | INTEGER  | 是否强制更新（0/1）                      |
| publish_date | TEXT     | 发布时间                                 |
| created_at   | DATETIME | 创建时间                                 |
| updated_at   | DATETIME | 更新时间                                 |

### 3.3 前端技术

- 使用 Chart.js 绘制更新状态分布饼图和更新类型分布柱状图
- 使用模态弹窗实现新增和编辑更新交互，新增和编辑共用同一弹窗
- 使用 CSS Toggle Switch 实现自动更新和强制更新的开关控件
- 搜索筛选通过后端 API 参数传递执行，支持 8 种条件组合筛选
- 导入功能通过隐藏的 `<input type="file">` 实现文件选择，读取后解析 JSON 并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 JSON 文件并触发下载
- 删除操作前弹出确认窗口，防止误操作

## 4. 使用指南

1. **新增更新**：点击"新增更新"按钮，填写更新名称、版本号、类型、状态等信息后点击"确认"。
2. **编辑更新**：点击更新的"编辑"按钮，在弹窗中修改信息后点击"确认"保存。
3. **删除更新**：点击"删除"按钮，在确认弹窗中确认后移除更新记录。
4. **搜索更新**：在搜索表单中输入筛选条件（名称、版本号、类型、状态、大小、发布时间、自动/强制更新开关），点击"搜索"按钮快速定位。
5. **导入更新**：点击"导入更新"按钮，选择 JSON 文件进行批量导入。
6. **导出更新**：点击"导出更新"按钮，将更新列表导出为 JSON 文件保存。
7. **查看统计**：页面底部的图表实时展示更新状态和类型分布。

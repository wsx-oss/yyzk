# 电池监控

## 1. 功能概述

电池监控模块是"CloudControl"新增的功能模块，专门用于实时监控无人机电池的电量、电压、电流、温度、健康度等关键指标。该模块与"GPS/位置信息"模块联动，基于已添加的无人机设备进行电池数据采集和监控。当电池电量过低、温度过高或健康度不足时，系统会自动生成报警通知，帮助操作人员及时发现电池异常，保障飞行安全。

> **数据联动**：当在"GPS/位置信息"模块中删除某台无人机时，该无人机对应的所有电池记录和电池报警将被级联删除，确保数据一致性。

## 2. 功能详细说明

### 2.1 电池数据上报

#### 上报电池数据

用户可点击页面右上角的 **"上报电池数据"** 按钮，打开上报弹窗。在该窗口中，用户需填写以下信息：

| 字段         | 是否必填 | 说明                                     |
| ------------ | -------- | ---------------------------------------- |
| 无人机设备   | 是       | 从GPS模块中选择要上报电池数据的无人机     |
| 电量(%)      | 否       | 当前电池电量百分比（0-100）              |
| 电压(V)      | 否       | 当前电池电压                             |
| 电流(A)      | 否       | 当前放电电流                             |
| 温度(°C)     | 否       | 当前电池温度                             |
| 健康度(%)    | 否       | 电池健康度百分比（0-100）                |
| 充电次数     | 否       | 电池累计充电循环次数                     |
| 剩余飞行时间 | 否       | 预估剩余飞行时间                         |

填写完成后，点击 **"确认上报"** 即可提交电池数据。系统会自动判断电池状态并在异常时生成报警。

#### 快速上报

在实时电池状态表格中，点击 **"上报"** 按钮可快速为该无人机上报新数据，系统会自动在当前数据基础上随机生成测试数据，方便快速测试。

> **说明**：实际部署时，电池数据由Agent端自动采集并通过API推送，无需手动上报。

#### Agent 自动推送（实时采集）

无人机端运行 `hw-agent` 程序并启用 Push 模式后，Agent 会**自动采集电池数据**并定时推送到主控端，无需手动操作。

```bash
# 在无人机端运行（每2秒推送硬件+GPS数据，每10秒推送电池数据）
./hw-agent -server http://主控端IP:8080 -push-interval 2
```

Agent 推送电池数据时通过 `POST /api/battery/push` 接口，使用 `agent_id`（默认为主机名）标识自身。后端会自动根据 `agent_id` 匹配 GPS 模块中已注册的无人机设备（按 `name` 字段匹配），然后插入电池记录并自动判断状态、生成报警。

> **前提**：Agent 需要先推送 GPS 数据（自动创建设备），之后电池数据才能正确关联到该设备。Agent 的推送顺序已自动处理这一依赖关系。

> **数据流**：无人机 hw-agent → `POST /api/battery/push` → 匹配 GPS 设备 → 插入电池记录 → 自动判断状态 → 触发报警

### 2.2 实时监控

点击页面右上角的 **"开启实时监控"** 按钮，系统通过 **WebSocket 长连接**订阅电池事件流（`/api/battery/stream`）。当后端有新的电池数据上报或报警确认时，服务端会即时推送事件到前端，前端收到事件后自动刷新数据，实现**零延迟的真正实时监控**。

- 开启后按钮显示 **"🟢 实时监控中（WebSocket）"**，再次点击即可关闭
- WebSocket 断线后自动每 3 秒尝试重连
- 实时监控期间，电池电量、状态、统计卡片、报警列表等在数据变化时自动更新

### 2.3 实时电池状态

#### 状态总览

"实时电池状态"标签页展示每台无人机的最新电池数据，包括：

- 电量（带彩色进度条）
- 电压、电流、温度
- 健康度
- 电池状态（正常/电量低/严重不足/温度过高/健康度低）
- 充电次数、剩余飞行时间
- 最后更新时间

#### 状态自动判定

系统根据上报数据自动判定电池状态：

| 条件               | 状态     |
| ------------------ | -------- |
| 电量 ≤ 10%         | 严重不足 |
| 电量 ≤ 20%         | 电量低   |
| 温度 ≥ 50°C        | 温度过高 |
| 健康度 ≤ 50%       | 健康度低 |
| 其他               | 正常     |

### 2.4 电池报警

#### 自动报警触发

当上报的电池数据满足以下条件时，系统自动生成报警：

- **电量 ≤ 20%**：生成"电量低"报警
- **电量 ≤ 10%**：生成"电量严重不足"报警，提示立即返航
- **温度 ≥ 50°C**：生成"温度过高"报警
- **健康度 ≤ 50%**：生成"健康度低"报警，建议更换电池

报警同时写入全局 `alerts` 表，可在"异常报警"模块中查看。

#### 报警管理

在"电池报警"标签页中，可查看所有电池报警记录，支持：

- 查看报警详情（无人机名称、报警类型、电量、电压、温度、报警信息）
- 确认报警（将报警标记为已确认）
- 分页浏览

### 2.5 电池详情与历史趋势

点击 **"详情"** 按钮，弹出电池详情弹窗，展示：

- 当前电池完整信息（电量、电压、电流、温度、健康度、状态、充电次数、剩余时间）
- **历史趋势图表**：以折线图展示该无人机的电量、温度、健康度随时间的变化趋势（最近50条记录）

### 2.6 查询筛选

用户可在筛选区域中填写以下条件进行搜索：

| 条件       | 说明                                               |
| ---------- | -------------------------------------------------- |
| 无人机设备 | 按设备精确筛选                                     |
| 电池状态   | 按状态精确筛选（正常/电量低/严重不足/温度过高/健康度低） |

### 2.7 统计信息展示

#### 统计卡片

页面顶部展示 5 个统计卡片，实时显示：

| 卡片         | 说明                     | 颜色 |
| ------------ | ------------------------ | ---- |
| 监控设备数   | 有电池记录的无人机总数   | 蓝色 |
| 正常         | 电池状态正常的无人机数   | 绿色 |
| 电量低/严重  | 电量低或严重不足的无人机数 | 橙色 |
| 平均电量     | 所有无人机的平均电量     | 天蓝 |
| 未处理报警   | 未确认的电池报警数       | 红色 |

#### 统计图表

页面底部展示两个统计图表：

- **电池状态分布**（环形图）：展示正常、电量低、严重不足、温度过高、健康度低各状态的占比
- **平均电量与健康度**（水平柱状图）：展示所有无人机的平均电量和平均健康度

### 2.8 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 先在"GPS/位置信息"模块中添加无人机设备（或通过 Agent Push 自动创建），然后在此处查看电池监控数据。
2. 点击"开启实时监控"按钮，系统通过 WebSocket 实时接收数据变化事件，配合Agent推送实现零延迟实时监控。
3. 当电量低于20%、温度超过50°C或健康度低于50%时，系统会自动生成电池报警。
4. 点击"详情"可查看该无人机的电池历史趋势图表，点击"上报"可快速为该无人机上报新数据。

## 3. 技术实现

### 3.1 后端接口

| 接口                                | 方法   | 说明                                                         |
| ----------------------------------- | ------ | ------------------------------------------------------------ |
| `/api/battery/records`            | GET    | 获取电池记录列表（支持 device_id、status 筛选 + 分页）       |
| `/api/battery/report`             | POST   | 上报电池数据（JSON 格式）                                    |
| `/api/battery/latest`             | GET    | 获取每台无人机的最新电池状态                                 |
| `/api/battery/history/:device_id` | GET    | 获取指定无人机的电池历史记录（支持 limit 参数）              |
| `/api/battery/stats`              | GET    | 获取电池监控统计数据                                         |
| `/api/battery/alerts`             | GET    | 获取电池报警记录列表（分页）                                 |
| `/api/battery/alerts/:id/ack`     | POST   | 确认电池报警                                                 |
| `/api/battery/push`               | POST   | Agent推送电池数据（通过agent_id匹配GPS设备，自动判断状态和报警）     |
| `/api/battery/stream`             | WS     | WebSocket 实时事件流（数据变化时即时推送 `battery_update`/`battery_alert_ack` 事件） |

### 3.2 数据存储

后端使用 SQLite 数据库存储电池数据，涉及两张表：

#### battery_records 表

| 字段           | 类型     | 说明                                     |
| -------------- | -------- | ---------------------------------------- |
| id             | INTEGER  | 主键，自增                               |
| device_id      | INTEGER  | 关联的无人机设备 ID（来自 gps_devices 表） |
| device_name    | TEXT     | 无人机名称                               |
| voltage        | REAL     | 电压（V）                                |
| current_val    | REAL     | 电流（A）                                |
| level          | INTEGER  | 电量百分比（0-100）                      |
| temperature    | REAL     | 温度（°C）                               |
| health         | INTEGER  | 健康度百分比（0-100）                    |
| status         | TEXT     | 电池状态（正常/电量低/严重不足/温度过高/健康度低） |
| charge_cycles  | INTEGER  | 充电循环次数                             |
| remaining_time | TEXT     | 剩余飞行时间                             |
| created_at     | DATETIME | 记录时间                                 |

索引：`idx_battery_records_device`、`idx_battery_records_created`

#### battery_alerts 表

| 字段         | 类型     | 说明                                     |
| ------------ | -------- | ---------------------------------------- |
| id           | INTEGER  | 主键，自增                               |
| device_id    | INTEGER  | 关联的无人机设备 ID                      |
| device_name  | TEXT     | 无人机名称                               |
| level        | INTEGER  | 触发时电量                               |
| voltage      | REAL     | 触发时电压                               |
| temperature  | REAL     | 触发时温度                               |
| alert_type   | TEXT     | 报警类型（电量低/电量严重不足/温度过高/健康度低） |
| message      | TEXT     | 报警消息                                 |
| acknowledged | INTEGER  | 是否已确认（0/1）                        |
| created_at   | DATETIME | 报警时间                                 |

索引：`idx_battery_alerts_device`

### 3.3 前端技术

- 使用 Chart.js 绘制电池状态分布环形图和平均电量/健康度柱状图
- 详情弹窗中使用折线图展示电量、温度、健康度的历史趋势
- 电量显示使用彩色进度条（绿色 > 50% / 蓝色 20-50% / 橙色 10-20% / 红色 ≤ 10%）
- 使用 Tab 标签页切换实时状态、历史记录和电池报警
- 无人机设备列表从 GPS 模块 API 动态加载
- 快速上报功能自动在当前数据基础上随机生成测试数据
- 搜索筛选通过后端 API 参数传递执行
- 删除操作前弹出确认窗口，防止误操作
- 实时监控功能通过 WebSocket 事件驱动实现（`/api/battery/stream`），数据变化时即时推送，零延迟更新
- WebSocket 断线自动重连（3秒间隔），连接状态实时显示在按钮上

## 4. 使用指南

1. **添加无人机**：先在"GPS/位置信息"模块中添加无人机设备（或通过 Agent Push 自动创建）。
2. **手动上报数据**：点击"上报电池数据"按钮，选择无人机并填写电池参数后提交。
3. **Agent自动推送**：在无人机端运行 `./hw-agent -server http://主控端IP:8080 -push-interval 2`，Agent会自动采集电池数据并推送。
4. **开启实时监控**：点击"开启实时监控"按钮，系统通过 WebSocket 实时接收数据变化事件，配合Agent推送实现零延迟实时监控。
5. **查看状态**：在"实时电池状态"标签页查看所有无人机的最新电池状态。
6. **查看详情**：点击"详情"按钮，查看电池完整信息和历史趋势图表。
7. **快速上报**：点击"上报"按钮，快速为该无人机上报新的电池数据。
8. **查看报警**：切换到"电池报警"标签页，查看和确认电池报警记录。
9. **筛选查询**：在筛选区域选择无人机或电池状态，点击"搜索"快速定位。
10. **查看统计**：页面底部图表实时展示电池状态分布和平均指标。

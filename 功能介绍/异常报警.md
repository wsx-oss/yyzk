# 异常报警

## 1. 功能概述

异常报警模块是"CloudControl"的重要功能之一，用于记录、管理和追踪系统运行过程中产生的各类异常报警信息。

报警来源分为两类：

- **手动创建**：在报警页面直接新增一条报警记录。
- **系统自动生成**：其他业务模块在检测到异常时会自动写入全局 `alerts` 表，例如：GPS 围栏越界会写入一条 `category=围栏报警` 的告警记录。

该模块支持报警的新增、查看详情、标记已解决、批量导入、导出、清空已解决报警等操作，并通过统计图表直观展示报警类型分布和优先级分布，帮助管理人员及时发现和处理系统异常。

## 2. 功能详细说明

### 2.1 报警管理功能

#### 新增报警记录

用户可点击页面右上角的 **"新增报警记录"** 按钮，打开新增报警的弹窗。在该窗口中，用户需填写以下信息：

| 字段       | 是否必填 | 说明                                                       |
| ---------- | -------- | ---------------------------------------------------------- |
| 报警时间   | 否       | 报警发生的时间，默认为当前时间                             |
| 报警类型   | 是       | 报警类型：系统故障、网络异常、设备故障                     |
| 报警优先级 | 是       | 优先级等级：高、中、低                                     |
| 报警状态   | 是       | 当前状态：未解决、处理中、已解决                           |
| 关联设备   | 否       | 与报警相关的设备名称                                       |
| 报警描述   | 是       | 对该报警的详细描述信息                                     |

填写完成后，点击 **"确认新增"** 即可添加新的报警记录。数据会持久化保存到后端数据库，重启服务后不会丢失。

#### 查看详情

在报警列表中，点击操作列的 **"查看详情"** 按钮，系统会弹出详情窗口，展示该条报警的完整信息，包括报警时间、报警类型、报警优先级、报警状态、关联设备、报警描述和创建时间。

#### 标记已解决

对于未解决的报警记录，点击操作列的 **"标记已解决"** 按钮，确认后系统会将该报警状态更新为"已解决"。已解决的报警不再显示"标记已解决"按钮。

#### 清空已解决报警

点击页面右上角的 **"清空已解决报警"** 按钮，系统弹出确认窗口，确认后将永久删除所有状态为"已解决"的报警记录。

> **注意**：清空操作不可恢复，请谨慎操作。

#### 搜索报警

用户可在筛选区域中填写以下条件，点击 **"搜索"** 按钮进行筛选：

| 条件           | 说明                                           |
| -------------- | ---------------------------------------------- |
| 报警时间范围   | 按开始日期和结束日期筛选                       |
| 报警类型       | 按类型精确筛选（系统故障/网络异常/设备故障/全部） |
| 报警优先级     | 按优先级精确筛选（高/中/低/全部）               |
| 报警状态       | 按状态精确筛选（未解决/已解决/处理中/全部）     |
| 关联设备       | 按设备名称模糊匹配                             |
| 报警描述关键词 | 按描述内容模糊匹配                             |

> **说明**：所有筛选条件均为可选，支持组合筛选。筛选通过后端 API 参数传递执行。

### 2.2 导入导出功能

#### 导入报警

点击 **"导入"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的文件进行批量导入。JSON 文件格式要求为数组格式，每个元素包含报警的各项字段信息。

导入时系统会自动校验数据：
- 导入完成后显示成功导入的记录数量
- 未填写的状态默认为"未解决"

#### 导出报警

点击 **"导出"** 按钮，系统会将当前筛选条件下的所有报警数据导出为 CSV 文件，文件名格式为 `alerts_YYYY-MM-DD.csv`。导出字段包括：报警时间、报警类型、报警优先级、报警状态、关联设备、报警描述。CSV 文件包含 BOM 头，支持 Excel 直接打开中文不乱码。

### 2.3 报警列表

页面以表格形式展示所有异常报警的详细信息，包括以下列：

| 列名       | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| 报警时间   | 报警发生的时间                                         |
| 报警类型   | 系统故障 / 网络异常 / 设备故障                         |
| 报警优先级 | 以彩色徽章显示（红色高/黄色中/蓝色低）                 |
| 报警状态   | 以彩色徽章显示（绿色已解决/红色未解决/黄色处理中）     |
| 关联设备   | 与报警相关的设备名称                                   |
| 报警描述   | 报警的描述信息（超长文本自动截断）                     |
| 操作       | 查看详情、标记已解决按钮                               |

表格支持分页显示，每页 20 条记录，底部显示总记录数和翻页控件。

### 2.4 报警统计功能

页面底部展示两个统计图表：

- **报警类型分布**（环形图）：以环形图形式展示不同报警类型（系统故障、网络异常、设备故障等）的占比情况，图例显示在右侧，用户可直观了解各类报警的分布状况。
- **报警优先级分布**（水平柱状图）：以水平柱状图形式展示不同优先级（高、中、低）的报警数量对比，帮助用户了解报警的紧急程度分布。

图表数据通过独立的统计接口获取，与列表数据分离，确保统计准确性。

### 2.5 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 可根据报警时间范围、类型、优先级等条件进行筛选查询。
2. 标记已解决的报警记录将不再显示在未解决列表中。
3. 清空已解决报警将永久删除已解决的报警记录，请谨慎操作。

## 3. 技术实现

### 3.1 后端接口

| 接口                          | 方法   | 说明                                                                                                     |
| ----------------------------- | ------ | -------------------------------------------------------------------------------------------------------- |
| `/api/alerts/list`          | GET    | 获取报警列表（支持 date_from、date_to、category、priority、status、device、keyword 筛选 + 分页）         |
| `/api/alerts/new`           | POST   | 新增报警记录（JSON：`category`/`severity`/`message`/`alert_time`/`priority`/`device`/`description`/`status`） |
| `/api/alerts/ack/:id`       | POST   | 确认指定 ID 的报警（仅设置 `acknowledged=1`，不改变 `status`）                                           |
| `/api/alerts/resolve/:id`   | POST   | 将指定 ID 的报警标记为已解决                                                                             |
| `/api/alerts/import`        | POST   | 批量导入报警（接收 JSON 数组）                                                                           |
| `/api/alerts/clear-resolved`| DELETE | 清空所有已解决的报警记录                                                                                 |
| `/api/alerts/stats`         | GET    | 获取报警统计数据（类型分布 + 优先级分布）                                                                |

### 3.2 数据存储

后端使用 SQLite 数据库的 `alerts` 表存储报警信息，包含以下字段：

| 字段         | 类型     | 说明                                     |
| ------------ | -------- | ---------------------------------------- |
| id           | INTEGER  | 主键，自增                               |
| category     | TEXT     | 报警类型                                 |
| severity     | TEXT     | 报警等级                                 |
| message      | TEXT     | 报警消息                                 |
| acknowledged | INTEGER  | 是否已确认（0/1）                        |
| alert_time   | TEXT     | 报警时间                                 |
| priority     | TEXT     | 优先级（高/中/低）                       |
| device       | TEXT     | 关联设备名称                             |
| description  | TEXT     | 报警描述                                 |
| status       | TEXT     | 报警状态（未解决/处理中/已解决）         |
| created_at   | DATETIME | 创建时间                                 |

### 3.3 前端技术

- 使用 Chart.js 绘制报警类型分布环形图和优先级分布水平柱状图
- 使用模态弹窗实现新增报警和查看详情交互
- 优先级和状态使用彩色徽章区分显示
- 搜索筛选通过后端 API 参数传递执行，支持 7 种条件组合筛选
- 导入功能通过模态弹窗选择 JSON 文件，读取后解析并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 CSV 文件并触发下载，含 BOM 头支持中文
- 点击模态弹窗外部区域可关闭弹窗
- 删除/清空操作前弹出确认窗口，防止误操作

## 4. 使用指南

1. **新增报警**：点击"新增报警记录"按钮，填写报警时间、类型、优先级、状态、设备、描述后点击"确认新增"。
2. **查看详情**：点击报警的"查看详情"按钮，在弹窗中查看完整报警信息。
3. **标记已解决**：点击"标记已解决"按钮，确认后将报警状态更新为已解决。
4. **搜索报警**：在筛选区域输入条件（时间范围、类型、优先级、状态、设备、关键词），点击"搜索"按钮快速定位。
5. **导入报警**：点击"导入"按钮，选择 JSON 文件进行批量导入。
6. **导出报警**：点击"导出"按钮，将报警列表导出为 CSV 文件保存。
7. **清空已解决**：点击"清空已解决报警"按钮，确认后永久删除所有已解决的报警记录。
8. **查看统计**：页面底部的图表实时展示报警类型和优先级分布。

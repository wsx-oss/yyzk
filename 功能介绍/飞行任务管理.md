# 飞行任务管理

## 1. 功能概述

飞行任务管理模块是"智能工具人机交互与远程操控系统"新增的功能模块，用于创建、管理和跟踪无人机飞行任务的全生命周期。该模块支持任务的创建、编辑、删除、状态跟踪、日志记录、批量导入、导出等操作，并通过统计图表直观展示任务状态分布和飞行阶段分布，帮助操作人员全面掌握飞行任务执行情况，实现对飞行任务从创建到完成的全流程管控。本模块与“GPS/位置信息”模块联动，创建任务时可选择具体的无人机设备执行任务。

## 2. 功能详细说明

### 2.1 任务管理功能

#### 创建任务

用户可点击页面右上角的 **"创建任务"** 按钮，打开创建飞行任务的弹窗。在该窗口中，用户需填写以下信息：

| 字段       | 是否必填 | 说明                                                       |
| ---------- | -------- | ---------------------------------------------------------- |
| 任务名称   | 是       | 飞行任务的名称                                             |
| 执行无人机 | 否       | 从GPS/位置信息模块中选择执行该任务的无人机设备             |
| 飞行路线   | 是       | 飞行路线描述，例如：A基地 → B区域 → C目标点 → A基地        |
| 任务目标   | 否       | 任务目标描述，例如：区域巡检、物资运输等                   |
| 预计时长   | 否       | 预计飞行时长，例如：2小时30分                               |
| 任务描述   | 否       | 对该任务的详细描述信息                                     |

填写完成后，点击 **"确认创建"** 即可添加新的飞行任务。新创建的任务默认状态为"待起飞"，当前阶段为"待命"，进度为 0%。数据会持久化保存到后端数据库，重启服务后不会丢失。

#### 编辑任务

在任务列表中，点击操作列的 **"编辑"** 按钮，系统会弹出编辑窗口，预填该任务的所有信息。用户可修改任务名称、飞行路线、任务目标、预计时长、任务描述等字段后点击 **"确认保存"** 保存更改。

> **注意**：已完成的任务不可编辑，编辑按钮将被禁用。

#### 删除任务

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中永久移除该飞行任务及其关联的所有任务日志。

> **注意**：删除操作不可恢复，请谨慎操作。

#### 导入任务

点击 **"导入"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的文件进行批量导入。JSON 文件格式要求为数组格式，每个元素包含任务的各项字段信息。

导入时系统会自动校验数据：

- 任务名称为空的记录将被跳过
- 新导入的任务默认状态为"待起飞"，阶段为"待命"
- 导入完成后显示成功导入的记录数量

JSON 格式示例：

```json
[
  {
    "name": "巡检任务A",
    "route": "A→B→A",
    "target": "区域巡检",
    "estimated_duration": "1小时",
    "description": "例行巡检"
  }
]
```

#### 导出任务

点击 **"导出"** 按钮，系统会将当前所有飞行任务数据导出为 JSON 文件，文件名格式为 `flight_missions_YYYY-MM-DD.json`。导出字段包括：任务名称、飞行路线、任务目标、预计时长、任务描述、任务状态、当前阶段、进度。

### 2.2 任务状态跟踪

#### 飞行阶段状态机

飞行任务的核心功能是阶段状态跟踪，支持以下 6 个飞行阶段的顺序推进：

```
待命 → 起飞 → 巡航 → 执行任务 → 返航 → 降落
```

每个阶段对应的任务状态和进度如下：

| 飞行阶段 | 任务状态 | 进度  |
| -------- | -------- | ----- |
| 待命     | 待起飞   | 0%    |
| 起飞     | 飞行中   | 10%   |
| 巡航     | 飞行中   | 30%   |
| 执行任务 | 飞行中   | 60%   |
| 返航     | 返航中   | 80%   |
| 降落     | 已完成   | 100%  |

#### 阶段变更操作

在任务详情弹窗中，底部显示阶段控制按钮。用户只能将任务推进到下一个阶段（不可跳跃、不可回退）：

- 已经过的阶段按钮显示为绿色（已完成）
- 当前阶段按钮显示为蓝色（当前）
- 下一个阶段按钮可点击
- 其余阶段按钮禁用

阶段变更时系统会自动：

- 更新任务状态和进度
- 在"起飞"阶段记录开始时间
- 在"降落"阶段记录结束时间
- 写入一条任务日志记录

### 2.3 任务查询

#### 条件筛选

用户可在筛选区域中填写以下条件进行搜索：

| 条件     | 说明                                               |
| -------- | -------------------------------------------------- |
| 任务名称 | 按名称模糊匹配                                     |
| 任务状态 | 按状态精确筛选（待起飞/飞行中/返航中/已完成/全部） |
| 任务目标 | 按目标模糊匹配                                     |

#### 搜索

填写筛选条件后，点击 **"搜索"** 按钮，系统将快速展示符合条件的任务列表。

> **说明**：所有筛选条件均为可选，支持组合筛选。筛选通过后端 API 参数传递执行。

### 2.4 任务展示

#### 统计卡片

页面顶部展示 5 个统计卡片，实时显示：

| 卡片     | 说明                   | 颜色   |
| -------- | ---------------------- | ------ |
| 任务总数 | 所有飞行任务的总数     | 蓝色   |
| 待起飞   | 状态为"待起飞"的任务数 | 紫色   |
| 飞行中   | 状态为"飞行中"的任务数 | 天蓝色 |
| 返航中   | 状态为"返航中"的任务数 | 橙色   |
| 已完成   | 状态为"已完成"的任务数 | 绿色   |

#### 任务列表

页面以表格形式展示飞行任务，包含以下列：

| 列名       | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| 任务名称   | 任务名称（加粗显示）                                   |
| 执行无人机 | 关联的无人机名称（未指定时显示灰色“未指定”）           |
| 飞行路线   | 飞行路线描述                                           |
| 任务目标   | 任务目标描述                                           |
| 预计时长 | 预计飞行时长                                           |
| 当前阶段 | 以彩色标签显示当前飞行阶段                             |
| 任务状态 | 以彩色标签显示任务状态                                 |
| 进度     | 以进度条 + 百分比显示任务完成进度                      |
| 创建时间 | 任务创建时间                                           |
| 操作     | 详情、编辑、删除按钮                                   |

表格支持分页显示，每页 20 条记录，底部显示总记录数和翻页控件。

#### 任务详情

点击 **"详情"** 按钮，弹出任务详情弹窗，展示以下信息：

- 任务基本信息（名称、执行无人机、路线、目标、预计时长、描述）
- 当前阶段和任务状态（彩色标签）
- 任务进度（进度条）
- 开始时间和结束时间
- 阶段控制按钮（状态跟踪）
- 任务日志时间线（按时间倒序显示所有阶段变更记录）

### 2.5 统计信息展示

页面底部展示两个统计图表：

- **任务状态分布**（环形图）：以环形图形式展示不同任务状态（待起飞、飞行中、返航中、已完成）的占比情况，图例显示在底部，用户可直观了解各状态任务的分布状况。
- **飞行阶段分布**（水平柱状图）：以水平柱状图形式展示各飞行阶段（待命、起飞、巡航、执行任务、返航、降落）的任务数量，便于用户掌握任务在各阶段的分布情况。

图表数据通过独立的统计接口获取，与列表数据分离，确保统计准确性。

### 2.6 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 创建任务后，可通过"状态跟踪"按钮控制飞行阶段：待命 → 起飞 → 巡航 → 执行任务 → 返航 → 降落。
2. 每次阶段变更会自动记录到任务日志中，可在详情中查看完整日志。
3. 支持批量导入/导出任务数据（JSON格式）。

## 3. 技术实现

### 3.1 后端接口

| 接口                              | 方法   | 说明                                                                                     |
| --------------------------------- | ------ | ---------------------------------------------------------------------------------------- |
| `/api/flight/missions`          | GET    | 获取任务列表（支持 name、status、target 筛选 + 分页）                                    |
| `/api/flight/missions`          | POST   | 创建飞行任务（JSON 格式，含 name、device_id、route、target、estimated_duration、description 字段） |
| `/api/flight/missions/stats`    | GET    | 获取任务统计数据（状态分布 + 阶段分布）                                                  |
| `/api/flight/missions/import`   | POST   | 批量导入任务（接收 JSON 数组）                                                           |
| `/api/flight/missions/:id`      | GET    | 获取指定 ID 的任务详情                                                                   |
| `/api/flight/missions/:id`      | PUT    | 编辑指定 ID 的任务信息                                                                   |
| `/api/flight/missions/:id`      | DELETE | 删除指定 ID 的任务记录及关联日志                                                         |
| `/api/flight/missions/:id/phase`| POST   | 更新指定任务的飞行阶段（JSON 格式，含 phase 字段）                                       |
| `/api/flight/missions/:id/logs` | GET    | 获取指定任务的操作日志列表                                                               |

### 3.2 数据存储

后端使用 SQLite 数据库存储飞行任务数据，涉及两张表：

#### flight_missions 表

| 字段               | 类型     | 说明                                     |
| ------------------ | -------- | ---------------------------------------- |
| id                 | INTEGER  | 主键，自增                               |
| name               | TEXT     | 任务名称                                 |
| device_id          | INTEGER  | 关联的无人机设备 ID（来自 gps_devices 表） |
| route              | TEXT     | 飞行路线                                 |
| target             | TEXT     | 任务目标                                 |
| estimated_duration | TEXT     | 预计时长                                 |
| status             | TEXT     | 任务状态（待起飞/飞行中/返航中/已完成）  |
| current_phase      | TEXT     | 当前阶段（待命/起飞/巡航/执行任务/返航/降落） |
| progress           | INTEGER  | 任务进度（0-100）                        |
| start_time         | TEXT     | 起飞时间                                 |
| end_time           | TEXT     | 降落时间                                 |
| description        | TEXT     | 任务描述                                 |
| created_at         | DATETIME | 创建时间                                 |
| updated_at         | DATETIME | 更新时间                                 |

索引：`idx_flight_missions_name`、`idx_flight_missions_status`、`idx_flight_missions_created`

#### mission_logs 表

| 字段       | 类型     | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| id         | INTEGER  | 主键，自增                               |
| mission_id | INTEGER  | 关联的飞行任务 ID（外键）                |
| phase      | TEXT     | 阶段名称                                 |
| message    | TEXT     | 日志消息                                 |
| created_at | DATETIME | 创建时间                                 |

索引：`idx_mission_logs_mission`

### 3.3 前端技术

- 使用 Chart.js 绘制任务状态分布环形图和飞行阶段分布水平柱状图
- 使用模态弹窗实现创建和编辑任务交互，创建和编辑共用同一弹窗
- 任务状态使用彩色标签区分显示（紫色待起飞/蓝色飞行中/橙色返航中/绿色已完成）
- 飞行阶段使用彩色标签区分显示（灰色待命/紫色起飞/蓝色巡航/橙色执行任务/深橙返航/绿色降落）
- 进度条根据进度值动态变色（蓝色 < 60% / 橙色 60%-99% / 绿色 100%）
- 搜索筛选通过后端 API 参数传递执行，支持 3 种条件组合筛选
- 导入功能通过模态弹窗选择 JSON 文件，读取后解析并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 JSON 文件并触发下载
- 任务详情弹窗中集成阶段控制按钮和任务日志时间线
- 点击模态弹窗外部区域可关闭弹窗
- 删除操作前弹出确认窗口，防止误操作

## 4. 使用指南

1. **创建任务**：点击“创建任务”按钮，填写任务名称、选择执行无人机、填写飞行路线、任务目标、预计时长、任务描述后点击“确认创建”。
2. **编辑任务**：点击任务的"编辑"按钮，在弹窗中修改信息后点击"确认保存"。
3. **删除任务**：点击"删除"按钮，在确认弹窗中确认后移除任务记录及关联日志。
4. **状态跟踪**：点击"详情"按钮，在详情弹窗底部使用阶段控制按钮逐步推进飞行阶段。
5. **搜索任务**：在筛选区域输入条件（任务名称、任务状态、任务目标），点击"搜索"按钮快速定位。
6. **导入任务**：点击"导入"按钮，选择 JSON 文件进行批量导入。
7. **导出任务**：点击"导出"按钮，将任务列表导出为 JSON 文件保存。
8. **查看统计**：页面底部的图表实时展示任务状态分布和飞行阶段分布。
9. **查看日志**：在任务详情弹窗中查看该任务的完整阶段变更日志。

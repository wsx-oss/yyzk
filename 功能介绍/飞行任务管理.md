# 飞行任务管理

## 1. 功能概述

飞行任务管理模块用于创建、管理和跟踪无人机飞行任务的全生命周期。该模块支持任务的创建、编辑、删除、状态跟踪、日志记录、批量导入、导出等操作，并通过统计图表直观展示任务状态分布和飞行阶段分布，帮助操作人员全面掌握飞行任务执行情况，实现对飞行任务从创建到完成的全流程管控。

本模块与“GPS/位置信息 / 电池信息”模块深度联动：

- 任务创建时可选择具体无人机设备执行任务（`device_id` 绑定到 `gps_devices.id`，通常来自无人机注册表 `drones.linked_gps_device_id`）
- 任务列表与详情会聚合展示该无人机的 **最新 GPS 实时位置**（经纬度/速度/航向/高度/更新时间）与 **最新电池信息**（电量等）
- 前端通过 **WebSocket 实时推送** 获得低延迟更新：
  - 飞行任务事件：`WS /api/flight/stream`
  - GPS 位置事件：`WS /api/gps/stream`

为提升可读性，系统支持 **逆地理编码**：将实时坐标转换为人类可理解的地名（`POST /api/amap/regeocode`，需配置 `AMAP_KEY`）。

此外，当前版本在创建/编辑任务弹窗内集成了 **地图选点/地址解析/智能航线规划** 能力：

- 可通过地图点击选择起点/终点坐标
- 支持地址解析（地理编码）辅助定位
- 支持调用“智能航线规划”接口生成建议航线/航点，并可一键采用到任务路线中

## 2. 功能详细说明

### 2.1 任务管理功能

#### 创建任务

用户可点击页面右上角的 **"创建任务"** 按钮，打开创建飞行任务的弹窗。在该窗口中，用户需填写以下信息：

| 字段       | 是否必填 | 说明                                                   |
| ---------- | -------- | ------------------------------------------------------ |
| 任务名称   | 是       | 飞行任务的名称                                         |
| 执行无人机 | 否       | 从GPS/位置信息模块中选择执行该任务的无人机设备         |
| 飞行路线   | 是       | 飞行路线描述，例如：A基地 → B区域 → C目标点 → A基地 |
| 任务目标   | 否       | 任务目标描述，例如：区域巡检、物资运输等               |
| 预计时长   | 否       | 预计飞行时长，例如：2小时30分                          |
| 任务描述   | 否       | 对该任务的详细描述信息                                 |

填写完成后，点击 **"确认创建"** 即可添加新的飞行任务。新创建的任务默认状态为"待起飞"，当前阶段为"待命"，进度为 0%。数据会持久化保存到后端数据库，重启服务后不会丢失。

> **说明**：执行无人机的下拉选项来源于“无人机管理”模块的统一注册表，前端会使用 `GET /api/drones` 获取无人机列表，并将其中 `linked_gps_device_id` 作为可选的 `device_id`。

#### 编辑任务

在任务列表中，点击操作列的 **"编辑"** 按钮，系统会弹出编辑窗口，预填该任务的所有信息。用户可修改任务名称、飞行路线、任务目标、预计时长、任务描述等字段后点击 **"确认保存"** 保存更改。

> **注意**：已完成的任务不可编辑，编辑按钮将被禁用。

#### 删除任务

点击操作列的 **"删除"** 按钮，系统弹出确认窗口，确认后将从系统中永久移除该飞行任务及其关联的所有任务日志。

> **注意**：删除操作不可恢复，请谨慎操作。

#### 导入任务

点击 **"导入"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的文件进行批量导入。JSON 文件格式要求为数组格式，每个元素包含任务的各项字段信息。

导入时系统会自动校验数据：

- 任务名称为空的记录将被跳过
- 新导入的任务默认状态为"待起飞"，阶段为"待命"
- 导入完成后显示成功导入的记录数量

JSON 格式示例：

```json
[
  {
    "name": "巡检任务A",
    "route": "A→B→A",
    "target": "区域巡检",
    "estimated_duration": "1小时",
    "description": "例行巡检"
  }
]
```

#### 导出任务

点击 **"导出"** 按钮，系统会将当前所有飞行任务数据导出为 JSON 文件，文件名格式为 `flight_missions_YYYY-MM-DD.json`。导出字段包括：任务名称、飞行路线、任务目标、预计时长、任务描述、任务状态、当前阶段、进度。

### 2.2 智能航线规划（可选）

创建/编辑任务弹窗内提供“智能规划”能力，辅助生成飞行路线（route）与航点建议，典型流程：

1. 通过地图点击选择起点（左键）与终点（右键），或输入地址/坐标
2. 调用地址解析接口将地址转换为坐标（可选）
3. 调用智能规划接口生成航线与解释
4. 一键采用规划结果填充到任务路线中

相关接口：

- `POST /api/amap/geocode`：地址 → 坐标
- `POST /api/flight/missions/plan`：创建规划任务
- `GET /api/flight/missions/plans`：规划任务列表
- `GET /api/flight/missions/plan/status`：规划能力状态（LLM/高德是否可用、当前模型与 BaseURL）
- `GET /api/flight/missions/plan/:id`：规划详情
- `POST /api/flight/missions/plan/:id/adopt`：采用规划结果
- `POST /api/flight/missions/plan/:id/discard`：丢弃规划结果

### 2.3 任务状态跟踪

#### 飞行阶段状态机

飞行任务的核心功能是阶段状态跟踪，支持以下 6 个飞行阶段的顺序推进：

```
待命 → 起飞 → 巡航 → 执行任务 → 返航 → 降落
```

每个阶段对应的任务状态和进度如下：

| 飞行阶段 | 任务状态 | 进度 |
| -------- | -------- | ---- |
| 待命     | 待起飞   | 0%   |
| 起飞     | 飞行中   | 10%  |
| 巡航     | 飞行中   | 30%  |
| 执行任务 | 飞行中   | 60%  |
| 返航     | 返航中   | 80%  |
| 降落     | 已完成   | 100% |

#### 阶段变更操作

在任务详情弹窗中，底部显示阶段控制区域。

系统同时支持“自动阶段上报”和“离线手动控制”两种模式：

- **设备在线（规则 A 生效）**：当任务已绑定设备，且该设备 GPS 最近更新时间在 60 秒内（视为在线），阶段由无人机/Agent 自动上报，前端阶段按钮将只读展示，**禁止手动修改**。
- **设备离线或未绑定**：允许用户手动推进阶段，用于演示/应急/无飞控接入时的人工管控。

当允许手动控制时，用户只能将任务推进到下一个阶段（不可跳跃、不可回退）：

- 已经过的阶段按钮显示为绿色（已完成）
- 当前阶段按钮显示为蓝色（当前）
- 下一个阶段按钮可点击
- 其余阶段按钮禁用

阶段变更时系统会自动：

- 更新任务状态和进度
- 在"起飞"阶段记录开始时间
- 在"降落"阶段记录结束时间
- 写入一条任务日志记录

### 2.4 任务查询

#### 条件筛选

用户可在筛选区域中填写以下条件进行搜索：

| 条件     | 说明                                               |
| -------- | -------------------------------------------------- |
| 任务名称 | 按名称模糊匹配                                     |
| 任务状态 | 按状态精确筛选（待起飞/飞行中/返航中/已完成/全部） |
| 任务目标 | 按目标模糊匹配                                     |

后端接口同时支持按创建时间范围筛选（用于按天/按周查询）：

- `date_from`：开始日期（例如 `2026-02-01`）
- `date_to`：结束日期（例如 `2026-02-23`，服务端会自动补齐到 `23:59:59`）

#### 搜索

填写筛选条件后，点击 **"搜索"** 按钮，系统将快速展示符合条件的任务列表。

> **说明**：所有筛选条件均为可选，支持组合筛选。筛选通过后端 API 参数传递执行。

页面以表格形式展示飞行任务，包含以下列：

#### 任务详情

点击 **"详情"** 按钮，弹出任务详情弹窗，展示以下信息：

- 飞行阶段与任务状态（合并彩色标签）
- **无人机实时位置卡片**（当任务绑定设备时显示）：
  - 地名（逆地理编码，可选）
  - 坐标（纬度/经度）
  - 速度/高度/航向
  - 电量
  - 最后更新时间
- 任务进度（进度条）
- 开始时间和结束时间
- 阶段控制（设备在线时只读自动、离线时允许手动推进）
- 任务日志时间线（按时间倒序显示所有阶段变更记录）

### 2.6 实时位置与电量展示（核心增强）

为提升任务监控的直观性，任务列表与任务详情会展示任务关联无人机的最新实时信息：

- **GPS 实时位置**：来自 `gps_devices` 的最新点位（纬度、经度、速度、航向、高度、更新时间）
- **电池电量**：来自 `battery_records` 的最新记录（列表显示电量百分比，详情可显示更丰富信息）
- **逆地理编码（可选）**：坐标 → 地名（需要配置 `AMAP_KEY`）

实时更新方式：

- HTTP 聚合列表/详情接口首次加载（包含最新 GPS + 电池）
- WebSocket 持续推送 GPS 更新（`WS /api/gps/stream`），前端会在不刷新整页的情况下实时更新“当前位置”列与详情展示

### 2.7 统计信息展示

页面底部展示两个统计图表：

- **任务状态分布**（环形图）：以环形图形式展示不同任务状态（待起飞、飞行中、返航中、已完成）的占比情况，图例显示在底部，用户可直观了解各状态任务的分布状况。
- **飞行阶段分布**（水平柱状图）：以水平柱状图形式展示各飞行阶段（待命、起飞、巡航、执行任务、返航、降落）的任务数量，便于用户掌握任务在各阶段的分布情况。

图表数据通过独立的统计接口获取，与列表数据分离，确保统计准确性。

### 2.8 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 创建任务后，在设备离线或未绑定时，可通过"状态跟踪"按钮手动推进阶段：待命 → 起飞 → 巡航 → 执行任务 → 返航 → 降落。
2. 每次阶段变更会自动记录到任务日志中，可在详情中查看完整日志。
3. 支持批量导入/导出任务数据（JSON格式）。

## 3. 真实无人机接入（方案 A：真实数据上报）

本系统支持无人机/机载计算机通过 Agent 采集真实数据并推送到后端，确保平台显示均为真实 GPS/电池/飞控状态。

### 3.1 关键原则

- **真实数据源优先**：当任务绑定设备且设备在线时，飞行阶段以无人机/飞控上报为准（规则 A），禁止手动覆盖。
- **推送接口不变**：真实数据按现有 JSON 协议推送到同样的接口（GPS/电池/飞行阶段）。

### 3.2 Agent（机载端）运行方式

在无人机的机载计算机上运行 `agent.exe`（或 Linux 下运行对应二进制），并指定主控服务地址与 `agent_id`：

- **MAVLink（推荐，PX4/ArduPilot）**：
  - `agent.exe -server http://<主控IP>:8080 -id <agent_id> -mavlink :14550`
- **NMEA/GPSD（仅 GPS 真实，电池/阶段需另接飞控）**：
  - `agent.exe -server http://<主控IP>:8080 -id <agent_id> -gps-serial COM3`
  - `agent.exe -server http://<主控IP>:8080 -id <agent_id> -gps-serial tcp:localhost:2947`

> 提示：`-simulate` 仅用于演示/联调，不代表真实无人机数据。

### 3.3 Agent 推送接口（后端接收）

- `POST /api/gps/push`：GPS 实时位置上报（建议 1-2s 一次）
- `POST /api/battery/push`：电池信息上报（可 5-10s 一次）
- `POST /api/flight/missions/push`：飞行阶段上报（飞控模式变化时及时上报）
- `POST /api/hardware/push`：机载计算机硬件指标上报（可选）

## 3. 技术实现

### 3.1 后端接口

| 接口                               | 方法   | 说明                                                                                                 |
| ---------------------------------- | ------ | ---------------------------------------------------------------------------------------------------- |
| `/api/flight/missions`           | GET    | 获取任务列表（支持 name、status、target、date_from、date_to 筛选 + 分页；聚合返回最新 GPS/电池摘要） |
| `/api/flight/missions`           | POST   | 创建飞行任务（JSON 格式，含 name、device_id、route、target、estimated_duration、description 字段）   |
| `/api/flight/missions/stats`     | GET    | 获取任务统计数据（状态分布 + 阶段分布）                                                              |
| `/api/flight/missions/import`    | POST   | 批量导入任务（接收 JSON 数组）                                                                       |
| `/api/flight/missions/:id`       | GET    | 获取指定 ID 的任务详情（聚合返回最新 GPS + 电池信息）                                                |
| `/api/flight/missions/:id`       | PUT    | 编辑指定 ID 的任务信息                                                                               |
| `/api/flight/missions/:id`       | DELETE | 删除指定 ID 的任务记录及关联日志                                                                     |
| `/api/flight/missions/:id/phase` | POST   | 更新指定任务的飞行阶段（仅设备离线/未绑定时允许；设备在线时按规则 A 禁止）                           |
| `/api/flight/missions/:id/logs`  | GET    | 获取指定任务的操作日志列表                                                                           |
| `/api/flight/missions/push`      | POST   | Agent 推送阶段更新（通过 `agent_id` 绑定设备后，自动更新该设备最新的未完成任务）                   |
| `/api/flight/stream`             | WS     | WebSocket 实时事件流（任务创建/编辑/删除/阶段变更/导入时即时推送事件）                               |
| `/api/gps/stream`                | WS     | WebSocket 实时 GPS 事件流（gps_update）                                                              |

智能规划相关：

| 接口                                      | 方法 | 说明                                                 |
| ----------------------------------------- | ---- | ---------------------------------------------------- |
| `/api/amap/geocode`                     | POST | 地址 → 坐标（用于地图定位辅助）                     |
| `/api/amap/regeocode`                   | POST | 坐标 → 地名（逆地理编码，用于实时位置展示）         |
| `/api/flight/missions/plan`             | POST | 创建智能规划任务                                     |
| `/api/flight/missions/plans`            | GET  | 规划任务列表                                         |
| `/api/flight/missions/plan/status`      | GET  | 规划能力状态（LLM/高德是否可用、当前模型与 BaseURL） |
| `/api/flight/missions/plan/:id`         | GET  | 规划详情                                             |
| `/api/flight/missions/plan/:id/adopt`   | POST | 采用规划结果（写入任务路线等字段）                   |
| `/api/flight/missions/plan/:id/discard` | POST | 丢弃规划结果                                         |

### 3.2 数据存储

后端使用 SQLite 数据库存储飞行任务数据，涉及两张表：

#### flight_missions 表

| 字段               | 类型     | 说明                                          |
| ------------------ | -------- | --------------------------------------------- |
| id                 | INTEGER  | 主键，自增                                    |
| name               | TEXT     | 任务名称                                      |
| device_id          | INTEGER  | 关联的无人机设备 ID（来自 gps_devices 表）    |
| route              | TEXT     | 飞行路线                                      |
| target             | TEXT     | 任务目标                                      |
| estimated_duration | TEXT     | 预计时长                                      |
| status             | TEXT     | 任务状态（待起飞/飞行中/返航中/已完成）       |
| current_phase      | TEXT     | 当前阶段（待命/起飞/巡航/执行任务/返航/降落） |
| progress           | INTEGER  | 任务进度（0-100）                             |
| start_time         | TEXT     | 起飞时间                                      |
| end_time           | TEXT     | 降落时间                                      |
| description        | TEXT     | 任务描述                                      |
| created_at         | DATETIME | 创建时间                                      |
| updated_at         | DATETIME | 更新时间                                      |

索引：`idx_flight_missions_name`、`idx_flight_missions_status`、`idx_flight_missions_created`

#### mission_logs 表

| 字段       | 类型     | 说明                      |
| ---------- | -------- | ------------------------- |
| id         | INTEGER  | 主键，自增                |
| mission_id | INTEGER  | 关联的飞行任务 ID（外键） |
| phase      | TEXT     | 阶段名称                  |
| message    | TEXT     | 日志消息                  |
| created_at | DATETIME | 创建时间                  |

索引：`idx_mission_logs_mission`

### 3.3 实时推送（WebSocket）

前端通过 `WS /api/flight/stream` 建立长连接订阅飞行任务事件。后端在任务发生变化时会广播事件，典型事件包括：

- `flight_created`
- `flight_updated`
- `flight_deleted`
- `flight_phase`
- `flight_imported`

前端收到事件后会刷新列表与统计信息，并更新实时监控状态。

同时，前端会通过 `WS /api/gps/stream` 订阅 GPS 位置更新（事件类型 `gps_update`），用于实时刷新任务列表中的“当前位置”列与详情展示。

### 3.4 前端技术

- 使用 Chart.js 绘制任务状态分布环形图和飞行阶段分布水平柱状图
- 使用模态弹窗实现创建和编辑任务交互，创建和编辑共用同一弹窗
- 任务状态使用彩色标签区分显示（紫色待起飞/蓝色飞行中/橙色返航中/绿色已完成）
- 飞行阶段使用彩色标签区分显示（灰色待命/紫色起飞/蓝色巡航/橙色执行任务/深橙返航/绿色降落）
- 进度条根据进度值动态变色（蓝色 < 60% / 橙色 60%-99% / 绿色 100%）
- 搜索筛选通过后端 API 参数传递执行，支持 3 种条件组合筛选
- 导入功能通过模态弹窗选择 JSON 文件，读取后解析并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 JSON 文件并触发下载
- 任务详情弹窗中集成阶段控制按钮和任务日志时间线
- 点击模态弹窗外部区域可关闭弹窗
- 删除操作前弹出确认窗口，防止误操作
- 实时监控功能通过 WebSocket 事件驱动实现（`/api/flight/stream`），任务变更时即时推送，零延迟更新
- WebSocket 断线自动重连（3秒间隔），连接状态实时显示在按钮上

## 4. 使用指南

1. **创建任务**：点击“创建任务”按钮，填写任务名称、选择执行无人机、填写飞行路线、任务目标、预计时长、任务描述后点击“确认创建”。
2. **编辑任务**：点击任务的"编辑"按钮，在弹窗中修改信息后点击"确认保存"。
3. **删除任务**：点击"删除"按钮，在确认弹窗中确认后移除任务记录及关联日志。
4. **状态跟踪**：点击"详情"按钮：
   - 设备在线时：阶段由无人机自动上报，只读展示
   - 设备离线/未绑定时：可手动推进阶段
5. **搜索任务**：在筛选区域输入条件（任务名称、任务状态、任务目标），点击"搜索"按钮快速定位。
6. **导入任务**：点击"导入"按钮，选择 JSON 文件进行批量导入。
7. **导出任务**：点击"导出"按钮，将任务列表导出为 JSON 文件保存。
8. **查看统计**：页面底部的图表实时展示任务状态分布和飞行阶段分布。
9. **查看日志**：在任务详情弹窗中查看该任务的完整阶段变更日志。

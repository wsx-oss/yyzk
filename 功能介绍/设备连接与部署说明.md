# 设备连接与部署说明

本文档详细说明系统各功能模块与无人机/外部设备的连接方式、IP 获取方式、局域网要求以及运行部署方式。

---

## 1. 需要与无人机/外部设备连接的功能

### 1.1 远程桌面控制

#### 连接方式

支持 3 种协议：**VNC**、**SSH**、**RDP**

| 项目          | 说明                                                                                      |
| ------------- | ----------------------------------------------------------------------------------------- |
| 连接方式      | VNC: WebSocket → TCP 代理；SSH: WebSocket → SSH 隧道；RDP: 生成 `.rdp` 文件由系统打开 |
| 谁获取谁的 IP | **用户手动输入**无人机/目标设备的 IP 地址（在"添加设备"表单中填写）                 |
| 默认端口      | VNC:`5900`，SSH: `22`，RDP: `3389`                                                  |
| 局域网要求    | **是**，主控端和无人机必须在同一局域网（或能互相访问的网络）                        |

#### 需要运行的程序

| 端              | 需要运行                             | 说明                                                                                         |
| --------------- | ------------------------------------ | -------------------------------------------------------------------------------------------- |
| 主控端（本机）  | `smartcontrol` 主程序（`:8080`） | 提供 WebSocket 代理，将浏览器的 VNC/SSH 流量转发到目标设备                                   |
| 无人机/目标设备 | VNC Server 或 SSH Server             | VNC 需要在无人机上运行 VNC 服务并监听 `5900` 端口；SSH 需要开启 SSH 服务（Linux 默认已有） |

#### 连接流程

```
浏览器 → WebSocket → 主控端(:8080) → TCP/SSH → 无人机IP:端口
```

### 1.2 硬件检测

#### 连接方式

**HTTP Pull 模式** — 主控端主动向目标设备的 Agent 发 HTTP 请求获取硬件指标。

| 项目          | 说明                                                                 |
| ------------- | -------------------------------------------------------------------- |
| 连接方式      | HTTP GET 请求到 `http://<目标IP>:9100/metrics`                     |
| 谁获取谁的 IP | **用户手动输入**无人机/目标设备的 IP（在"添加硬件"表单中填写） |
| 默认端口      | `9100`                                                             |
| 局域网要求    | **是**，主控端必须能访问目标设备的 `9100` 端口               |

#### 需要运行的程序

| 端              | 需要运行                                                                | 说明                                                                   |
| --------------- | ----------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| 主控端（本机）  | `smartcontrol` 主程序（`:8080`），内嵌了一个本机 Agent（`:9100`） | 主程序启动时自动在本机 `9100` 端口启动 Agent，用于监控本机硬件       |
| 地面站电脑（与主控同机或单独运行） | `hw-agent` 独立程序（`:9100`） | 在地面站电脑上运行，通过串口/UDP 读取飞控 MAVLink 数据 |

#### 连接流程

```
主控端(:8080) → HTTP GET → 地面站电脑IP:9100/metrics → 返回 JSON(CPU/内存/温度/网络)
```

#### Agent Push 模式（可选，推荐）

Agent 还支持主动推送模式，地面站电脑上的 Agent 主动向云端后端推送数据。地面站中继架构下建议使用 MAVLink Push 模式：

```bash
# 在地面站电脑上运行，读取飞控 MAVLink 并推送到云端
hw-agent -server http://云端服务器:8080 -id drone-01 -mavlink :14550 -push-interval 2
```

推送的数据和频率：

| 数据类型 | 推送接口               | 推送频率                     | 说明                                       |
| -------- | ---------------------- | ---------------------------- | ------------------------------------------ |
| 硬件指标 | `POST /api/hardware/push` | 每个 push-interval 周期      | CPU、内存、温度、网络带宽等                |
| GPS 位置 | `POST /api/gps/push`      | 每个 push-interval 周期      | 经纬度、高度、速度、航向，自动创建/匹配设备 |
| 电池状态 | `POST /api/battery/push`  | 每 5 个 push-interval 周期   | 电压、电流、电量、温度、健康度，自动报警   |

地面站电脑需能访问云端服务器的地址（通过 4G/Wi-Fi 均可）。GPS 和电池数据通过 `-id`（agent_id）自动匹配已注册无人机，无需其他配置。

### 1.3 数据同步

#### 连接方式

**HTTP 双向同步** — 从源设备导出数据，导入到目标设备。

| 项目          | 说明                                                                                                            |
| ------------- | --------------------------------------------------------------------------------------------------------------- |
| 连接方式      | HTTP GET `http://<源IP>:8080/api/sync/export-data` → HTTP POST `http://<目标IP>:8080/api/sync/import-data` |
| 谁获取谁的 IP | **用户手动输入**源设备 IP 和目标设备 IP（在"创建同步任务"表单中填写）                                     |
| 默认端口      | `8080`（smartcontrol 主程序端口）                                                                             |
| 局域网要求    | **是**，两台设备必须在同一局域网（或能互相访问）                                                          |

#### 需要运行的程序

| 端                  | 需要运行                             | 说明                                |
| ------------------- | ------------------------------------ | ----------------------------------- |
| 主控端              | `smartcontrol` 主程序（`:8080`） | 发起同步请求                        |
| 源设备（无人机A）   | `smartcontrol` 主程序（`:8080`） | 提供 `/api/sync/export-data` 接口 |
| 目标设备（无人机B） | `smartcontrol` 主程序（`:8080`） | 提供 `/api/sync/import-data` 接口 |

#### 连接流程

```
主控端 → HTTP GET  → 源设备IP:8080/api/sync/export-data  （拉取数据）
主控端 → HTTP POST → 目标设备IP:8080/api/sync/import-data （写入数据）
```

#### 同步的数据表

`hardware_items`、`updates`、`recordings`、`logs`、`alerts`、`video_sources`、`devices`、`drones`、`battery_records`、`battery_alerts`、`gps_devices`、`gps_history`、`gps_fence_alerts`、`flight_missions`、`mission_logs`、`perf_reports`、`flight_plans`、`user_stats` 共 18 张表。

> **说明**：同步白名单以服务端 `syncengine.SyncableTables` 为准，涵盖系统所有核心业务表。其中 `video_sources` 为历史遗留表，当前版本视频数据已迁移至 `drones` 表的 `video_url` 字段。

### 1.4 视频监控

#### 连接方式

**直接 URL 播放** — 视频流地址在注册无人机时通过 `video_url` 字段配置，浏览器直接播放。

| 项目          | 说明                                                                   |
| ------------- | ---------------------------------------------------------------------- |
| 连接方式      | 浏览器 `<video>` 标签或 `<iframe>` 直接加载视频 URL                |
| 谁获取谁的 IP | 用户在“无人机管理”中注册时填写 `video_url`（如 `http://无人机IP:8080/stream`） |
| 局域网要求    | **是**，浏览器必须能直接访问视频流地址                           |

> **架构变更**：视频源不再独立管理，而是通过无人机注册表的 `video_url` 字段统一配置。

#### 需要运行的程序

| 端            | 需要运行                         | 说明                                 |
| ------------- | -------------------------------- | ------------------------------------ |
| 主控端        | `smartcontrol` 主程序 + 浏览器 | 管理无人机和视频配置                   |
| 无人机/摄像头 | 视频流服务（如 RTSP/HTTP 流）    | 需要无人机上有摄像头并运行视频流服务 |

### 1.5 无人机管理

#### 连接方式

无人机管理模块是系统的核心入口，注册无人机时自动创建关联的远程设备和 GPS 设备。

| 项目          | 说明                                                                 |
| ------------- | -------------------------------------------------------------------- |
| 连接方式      | 通过关联的远程设备进行 VNC/SSH/RDP 连接                          |
| 谁获取谁的 IP | **用户在注册无人机时填写** IP 地址、端口、协议等信息           |
| 局域网要求    | **是**，主控端和无人机必须在同一局域网（或能互相访问的网络）     |

#### 注册时自动创建的关联资源

| 关联资源       | 说明                                                   |
| -------------- | ------------------------------------------------------ |
| 远程设备       | 自动在 `devices` 表中创建，用于远程桌面控制         |
| GPS 设备      | 自动在 `gps_devices` 表中创建，用于位置跟踪      |
| 硬件设备       | Agent Push 模式下自动在 `hardware_items` 表中创建 |

#### 删除时级联清理

删除无人机时，系统会自动级联删除关联的远程设备、GPS 设备、GPS 历史轨迹、围栏报警、电池记录、电池报警、飞行任务及任务日志。

## 4. 跨网络连接方案（非局域网场景）

无人机飞行过程中通常通过 4G/5G 移动网络上网，与地面站主控端**不在同一局域网**，无法直接通过 IP 互访。以下是三种解决方案，按推荐程度排序。

### 4.1 方案一：VPN 虚拟组网（推荐）

**原理**：无人机和主控端都加入同一个虚拟局域网，获得固定的虚拟 IP，彼此可以像在同一局域网一样直接通信。

**推荐工具**：[ZeroTier](https://www.zerotier.com/)（免费支持 25 台设备）或 [Tailscale](https://tailscale.com/)

#### 部署步骤

**第一步：创建虚拟网络**

1. 访问 [https://my.zerotier.com](https://my.zerotier.com) 注册账号
2. 创建一个 Network，记下 Network ID（如 `a1b2c3d4e5f6g7h8`）

**第二步：主控端（地面站）加入网络**

```bash
# Windows：下载安装 ZeroTier 客户端
# 安装后在系统托盘右键 → Join Network → 输入 Network ID

# Linux：
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join a1b2c3d4e5f6g7h8
```

**第三步：无人机端加入网络**

```bash
# 在无人机的机载计算机上（如树莓派、Jetson）
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join a1b2c3d4e5f6g7h8
```

**第四步：在 ZeroTier 管理面板中授权两台设备**

授权后，两台设备会分别获得虚拟 IP（如 `10.147.20.1` 和 `10.147.20.2`），之后在系统中使用这个虚拟 IP 即可，所有功能无需任何代码修改。

#### 优缺点

| 优点                                           | 缺点                                 |
| ---------------------------------------------- | -------------------------------------- |
| 不需要修改任何代码                             | 需要双方都安装 VPN 客户端              |
| 所有功能（VNC/SSH/硬件检测/同步/视频）全部可用 | 免费版有设备数量限制                   |
| 虚拟 IP 固定，不随网络变化                     | 首次连接需要互联网访问 ZeroTier 服务器 |
| P2P 直连，延迟低                               | 极端弱网环境下可能不稳定               |

---

### 4.2 方案二：内网穿透（frp）

**原理**：在一台公网服务器上部署 frp 服务端，无人机通过 frp 客户端将本地端口映射到公网，主控端通过公网地址访问无人机。

**前提**：需要一台有公网 IP 的云服务器（如阿里云、腾讯云轻量服务器，最低配置即可）。

#### 部署步骤

**第一步：公网服务器部署 frps（服务端）**

```ini
# frps.toml
bindPort = 7000

# 为无人机的各个服务预留端口范围
allowPorts = [
  { start = 10000, end = 10100 }
]
```

```bash
./frps -c frps.toml
```

**第二步：无人机端部署 frpc（客户端）**

```ini
# frpc.toml
serverAddr = "公网服务器IP"
serverPort = 7000

# 映射 hw-agent 端口
[[proxies]]
name = "drone-agent"
type = "tcp"
localIP = "127.0.0.1"
localPort = 9100
remotePort = 10001

# 映射 VNC 端口
[[proxies]]
name = "drone-vnc"
type = "tcp"
localIP = "127.0.0.1"
localPort = 5900
remotePort = 10002

# 映射 SSH 端口
[[proxies]]
name = "drone-ssh"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 10003

# 映射 smartcontrol 主程序（用于数据同步）
[[proxies]]
name = "drone-sync"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8080
remotePort = 10004
```

```bash
./frpc -c frpc.toml
```

**第三步：主控端使用公网地址访问**

在系统中填写设备信息时，使用公网服务器的 IP 和映射端口：

- 硬件检测 IP：`公网服务器IP`，端口：`10001`
- VNC 连接：`公网服务器IP:10002`
- SSH 连接：`公网服务器IP:10003`
- 数据同步目标：`公网服务器IP:10004`

#### 优缺点

| 优点                       | 缺点                                 |
| -------------------------- | ------------------------------------ |
| 不需要修改任何代码         | 需要一台公网服务器（约 30-50 元/月） |
| 配置灵活，按需映射端口     | 所有流量经过公网服务器，有额外延迟   |
| 支持多架无人机（不同端口） | 需要为每架无人机配置不同的端口映射   |
| 无人机端无需固定 IP        | 视频流带宽消耗较大，需注意服务器带宽 |

---

### 4.3 方案三：Agent Push 模式（已内置）

**原理**：系统已内置的 hw-agent 支持 Push 模式，无人机主动向主控端推送数据，不需要主控端能访问无人机。当前系统支持推送：

- 硬件指标：`POST /api/hardware/push`
- GPS 位置信息：`POST /api/gps/push`
- 电池状态：`POST /api/battery/push`
- 飞行任务阶段：`POST /api/flight/missions/push`

```bash
# 无人机端运行（主动推送到主控端）
./hw-agent -server http://主控端公网地址:8080 -id drone-01 -mavlink :14550 -push-interval 2
```

**注意**：Push 模式解决的是“数据上报方向”的问题，适用于 **GPS/电池/飞行阶段/硬件指标** 等上报类功能。

但以下功能仍然需要主控端能访问无人机/目标设备（即需要同一局域网、VPN 组网或内网穿透等方案）：

- 远程桌面控制（VNC/SSH/RDP）
- 视频监控（浏览器需要能访问 `video_url`）
- 数据同步（主控端需要能访问对端的 `/api/sync/*` 接口）

因此 Push 模式通常作为方案一（VPN）或方案二（frp）的**补充**，用于确保无人机即使在复杂网络环境下也能持续上报 GPS/电池/飞行状态。

---

### 4.4 方案对比与推荐

| 对比项     | VPN 组网     | 内网穿透（frp）    | Agent Push       |
| ---------- | ------------ | ------------------ | ---------------- |
| 代码修改   | 无           | 无                 | 无               |
| 额外成本   | 免费         | 需公网服务器       | 需主控端有公网IP |
| 支持的功能 | 全部         | 全部               | 上报类功能（硬件/GPS/电池/飞行阶段） |
| 延迟       | 低（P2P）    | 中（经服务器转发） | 低               |
| 部署复杂度 | 低           | 中                 | 低               |
| 多机支持   | 好           | 好（需配置端口）   | 好               |
| 推荐场景   | 日常飞行任务 | 需要稳定公网访问   | 仅需数据上报（GPS/电池/阶段/硬件） |

**推荐**：

- **首选方案一（VPN 组网）**：零成本、零代码修改、全功能支持、延迟低
- 如果需要对外提供稳定服务，选**方案二（frp）**
- 方案三作为补充，确保即使 VPN 断开，GPS/电池/飞行阶段/硬件等数据仍能持续推送

## 5. 预置生产无人机

系统内置了 5 架通过地面站 MAVLink 中继接入的 K9 系列生产无人机，启动后端服务时会自动注册并持续接收遥测数据。这些无人机无需手动添加，开箱即可在**无人机管理**、**GPS/位置信息**、**电池监控**和**飞行任务管理**等模块中使用。

| 编号 | 名称     | 型号         | Agent ID    | 基准坐标（纬度/经度） | 固件版本 |
| ---- | -------- | ------------ | ----------- | ---------------------- | -------- |
| 1    | 翼龙-I   | K9-四旋翼    | UAV-K9-001  | 22.5431 / 114.0579    | 1.17.83  |
| 2    | 天鹰-II  | K9-六旋翼    | UAV-K9-002  | 22.5445 / 114.0592    | 1.17.83  |
| 3    | 云雀-III | K9-四旋翼    | UAV-K9-003  | 22.5418 / 114.0565    | 1.18.02  |
| 4    | 苍鹰-IV  | K9-四旋翼    | UAV-K9-004  | 22.5460 / 114.0550    | 1.18.02  |
| 5    | 飞鸿-V   | K9-六旋翼    | UAV-K9-005  | 22.5475 / 114.0610    | 1.18.02  |

每架无人机实时推送的数据包括：

- **GPS 定位**：RTK 固定解（fix_type=6），16-24 颗卫星，精度 ≤1m，2Hz 更新
- **电池状态**：电压、电流、电量、温度、健康度，每 3 秒更新
- **MAVLink 遥测**：心跳、姿态、航向、VFR HUD、着陆状态、Home 坐标等，1Hz 更新
- **数据存储**：所有遥测数据同时写入 MySQL（`mavlink_telemetry`、`gps_devices`、`battery_records`、`gps_history`）和 Redis 缓存

> **说明**：这 5 架无人机通过后端数据通路自动注册，在无人机管理界面显示为"在线"状态，可直接在飞行任务管理中选择执行任务。

## 6. 快速部署指南

### 6.1 主控端（地面站电脑）

```bash
# 编译并运行主程序
cd project
go build -o smartcontrol .
./smartcontrol
# 默认监听 :8080，内嵌 Agent 监听 :9100
```

浏览器访问 `http://localhost:8080` 即可使用所有功能。

### 6.2 地面站端（地面站中继架构）

在地面站电脑上运行 hw-agent，读取飞控 MAVLink 数据并推送到云端后端：

```bash
# 编译 hw-agent
cd project
go build -o hw-agent ./cmd/agent

# MAVLink 模式（飞控通过串口/UDP 连到地面站电脑）
# 先用 mavproxy 将飞控串口转为 UDP：
# mavproxy.py --master=/dev/ttyACM0 --out=udp:127.0.0.1:14550
./hw-agent -server http://云端地址:8080 -id drone-01 -mavlink :14550 -push-interval 2
```

#### 远程桌面（按需）

```bash
# VNC 方式：安装并启动 VNC Server
# Linux 示例
sudo apt install tightvncserver
vncserver :0

# SSH 方式：确保 SSH 服务已启动
sudo systemctl start sshd
```

#### 数据同步（按需）

```bash
# 无人机上也需要运行 smartcontrol 主程序
./smartcontrol
# 这样主控端才能通过 HTTP 接口与其交换数据
```

#### 视频监控（按需）

```bash
# 启动视频流服务（取决于摄像头类型）
# 例如使用 ffmpeg 推流
ffmpeg -f v4l2 -i /dev/video0 -f mpegts http://0.0.0.0:8090/stream
```

# 仿真模拟引擎 功能介绍

## 一、功能概述

仿真模拟引擎是"CloudControl"的核心高级功能模块，提供**多实例无人机并行仿真、7×24 连续运行、异常注入、强化学习训练与策略导出**能力。通过该模块，用户可以在不依赖实体无人机的情况下，模拟数十至上百台无人机同时执行巡逻、巡检、配送、测绘、搜救等任务，并利用强化学习训练出可迁移到实机的智能决策策略。

---

## 二、设计背景

### 2.1 为什么需要仿真引擎？

在无人机管控平台中，真实无人机的飞行成本高、测试场景受限、风险不可控。仿真引擎解决以下核心问题：

1. **零成本验证**：不消耗电池、不占用空域，即可验证航线规划、异常处理、任务调度等功能
2. **批量压力测试**：同时运行几十上百台仿真无人机，验证系统在高并发下的稳定性
3. **数据积累**：持续产生 GPS、电池、飞行状态、异常事件等数据，供统计分析和 AI 训练使用
4. **策略预训练**：通过强化学习在仿真环境中训练决策策略，降低实机部署的试错成本

### 2.2 技术方案

- **仿真引擎**：`internal/simulation/engine.go`，管理所有仿真实例的生命周期
- **单机状态机**：`internal/simulation/instance.go`，模拟单台无人机的 GPS 运动、电池消耗、飞行阶段转换
- **强化学习**：`internal/simulation/rl.go`，Q-Learning 算法 + 经验回放 + ε-贪心探索
- **数据桥接**：`internal/handlers/sim_integration.go`，将仿真遥测写入 MySQL + WebSocket 推送
- **前端看板**：`web/modules/simulation.html`，批次管理 + 实例列表 + Leaflet 实时地图 + 统计图表

## 三、核心功能

### 3.1 批次管理

| 功能 | 说明 |
|------|------|
| 创建批次 | 指定数量、任务类型（巡逻/巡检/配送/测绘/搜救）、起始坐标和航线配置 |
| 启动批次 | 批量启动所有实例，每个实例在独立 goroutine 中运行 |
| 停止批次 | 批量停止所有运行中实例 |
| 删除批次 | 清理批次及其所有实例数据 |

### 3.2 实例仿真

每个仿真实例模拟一台完整无人机，包含：

- **GPS 运动模型**：沿航点飞行，支持起飞→巡航→作业→返航→降落→空闲完整阶段
- **电池模型**：float64 精确追踪，根据飞行阶段计算电流消耗（悬停 8A / 巡航 12A / 最大 25A），支持地面自动充电
- **7×24 连续运行**：电池耗尽后自动降落→地面充电→满电自动重新起飞，实现持续仿真
- **快照恢复**：每 30 秒自动保存实例状态到 `data/sim_snapshots.json`，服务重启后自动恢复运行中实例

### 3.3 异常注入

支持对单个实例或整个批次注入异常：

| 异常类型 | 效果 |
|----------|------|
| `low_battery` | 强制降低电池电量，触发低电量告警和强制返航 |
| `flight_deviation` | 注入航向偏差，模拟风偏或 GPS 漂移 |
| `comm_lost` | 模拟通信失联，遥测数据暂停推送 |
| `temp_high` / `temp_low` | 模拟电池温度异常 |

### 3.4 防碰撞系统

`collisionAvoidanceLoop` 在后台高频检测所有运行中实例之间的距离，当两架无人机距离过近时自动执行航向和高度避让。

### 3.5 禁飞区地理围栏检测

仿真引擎集成了禁飞区（No-Fly Zone）地理围栏检测能力，确保 RL 训练和仿真监控中能感知空域合规状态：

| 功能 | 说明 |
|------|------|
| 禁飞区加载 | 启动时从 `no_fly_zones` 表自动加载所有多边形禁飞区数据 |
| 实时检测 | 每个 tick 对所有运行实例执行 `CheckGeofence()`，使用 Ray-Casting 点在多边形内算法 |
| 遥测标记 | 违规实例的 `TelemetrySnapshot.GeofenceViolation` 置为 `true` |
| RL 状态扩展 | `RLState` 新增 `InNoFlyZone` 维度，状态空间扩展为 11 维 |
| RL 奖励惩罚 | 违规时施加 -3.0 惩罚；离开禁飞区的纠正动作获得 +1.5 奖励 |

**核心算法 — Ray-Casting 点在多边形内检测**：从待检测点发射一条水平射线，统计与多边形边界的交点数。交点为奇数则在内部，偶数则在外部。

### 3.6 数据流

仿真数据通过 `SimTelemetryPusher` 异步写入 MySQL（经 TaskPool IO 池）并通过 WebSocket 实时推送到前端：

| 数据类型 | 写入频率 | 目标表 |
|----------|----------|--------|
| GPS 更新 | 每 3 tick | `gps_devices` |
| 电池更新 | 每 10 tick | `battery_records` + `battery_alerts` |
| 飞行状态 | 每 3 tick | `sim_instances` |
| 遥测日志 | 每 10 tick | `sim_telemetry_log` |
| 异常事件 | 即时 | `sim_events` + `alerts` |

使用 `TrySubmit` 非阻塞提交，当任务池满时丢弃 DB 写入（WebSocket 推送不受影响），防止云数据库高延迟时阻塞仿真主循环。

## 四、强化学习（RL）

### 4.1 算法概述

采用 **Tabular Q-Learning** 算法，选择表格型策略以保证可解释性和快速收敛：

- **状态空间**：11 维离散化（GPS 位置 111m 网格、高度 10m 分桶、速度 5m/s 分桶、航向 8 方位、电量 10% 分桶、飞行阶段、异常标记、邻近无人机数量、最近距离分桶、**禁飞区标志 InNoFlyZone**）
- **动作空间**：7 种
  - `adjust_heading`：调整航向角（±45°）
  - `adjust_speed`：调整巡航速度（0.3x~1.5x）
  - `adjust_altitude`：调整飞行高度（±30m）
  - `goto_waypoint`：跳转至下一航点
  - `return_home`：触发返航
  - `hover`：悬停等待
  - `emergency_land`：紧急降落
- **奖励函数**：7 维多目标加权
  - 航线效率 18%（航点进度推进奖励）
  - 安全距离 18%（碰撞惩罚 + 规避动作奖励）
  - 异常响应 18%（正确应对异常奖励）
  - 能量管理 13%（低电返航奖励、高耗惩罚）
  - 任务完成 13%（完成奖励、失败惩罚）
  - 集群内聚 10%（合理间距奖励）
  - **禁飞区合规 10%**（违规 -3.0 惩罚、纠正动作 +1.5 奖励）

### 4.2 训练流程

1. 启动训练：`POST /api/sim/rl/start`
2. 训练循环每 2 秒执行一次：
   - 采集所有运行中无人机的遥测状态
   - 基于 ε-贪心选择动作并 **真实执行到仿真环境**
   - 计算奖励，存入经验回放缓冲区
   - 从缓冲区采样批次进行 Q-Table 更新
3. 每 100 个 episode 执行评估，记录到 `rl_training_log`
4. 每 1000 个 episode 保存策略文件 `data/rl_policy.json`
5. ε 衰减（0.995/eval），逐步从探索转向利用

### 4.3 策略导出

`GET /api/sim/rl/export-policy` 返回可用于实机部署的策略摘要：

- Q-Table 策略文件路径
- 动作空间定义
- 训练学到的决策规则（如"电量<20% + 异常 → 返航"）
- 部署指南（Q表加载 → 状态离散化 → 查表 → MAVLink 执行）

### 4.4 对实机的价值

| 可迁移 | 需适配 |
|--------|--------|
| 低电量返航策略 | GPS 精度差异 |
| 碰撞规避方向 | 真实风场模型 |
| 通信失联行为 | 电池非线性放电 |
| 温度异常处理 | 传感器噪声延迟 |
| 任务推进决策 | 动力学响应时间 |

实机部署建议：
1. 导出 Q-Table 作为基线决策器
2. 硬编码安全兜底规则（电量<10% 强制降落）
3. 在实机上少量 fine-tune 缩小 sim-to-real gap
4. 人在回路审批（策略建议 → 人工确认 → 执行）

## 五、数据库表结构

### 5.1 仿真相关表（MySQL）

| 表名 | 说明 |
|------|------|
| `sim_batches` | 仿真批次配置（任务类型、数量、坐标、参数） |
| `sim_instances` | 仿真实例记录（状态、飞行阶段、电池、GPS） |
| `sim_events` | 异常事件日志（类型、级别、消息、详情 JSON） |
| `sim_telemetry_log` | 遥测历史（GPS+电量+阶段，用于趋势分析） |
| `rl_training_log` | RL 训练评估记录（episode、奖励、各维度分数） |

### 5.2 关联表

仿真数据同时写入业务表以实现数据闭环：

- `gps_devices`：仿真无人机 GPS 位置
- `battery_records` / `battery_alerts`：仿真电池数据和告警
- `alerts`：仿真异常事件写入全局告警

## 七、前端页面

`simulation.html` 提供完整的仿真管理界面：

1. **批次管理区**：创建/启停/删除批次，选择任务模板
2. **实例列表区**：展示所有实例的状态、电池、GPS、任务进度
3. **异常注入区**：选择异常类型并注入到实例或批次
4. **实时地图区**：Leaflet 地图显示无人机位置、航向、轨迹、航线（支持贝塞尔曲线/Catmull-Rom 平滑轨迹，Canvas 渲染器高性能显示，「曲线轨迹」切换开关）
5. **统计看板区**：
   - 实时指标卡片（总数/在线/异常/电池风险/完成率）
   - 运行趋势图（分钟粒度）
   - 异常统计图
   - 电池风险图
   - 任务分类占比图
   - RL 学习曲线
6. **RL 控制区**：启停训练、查看训练状态、导出策略

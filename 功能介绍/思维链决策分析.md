# 思维链决策分析功能说明

## 功能概述

思维链（Chain of Thought, CoT）决策分析功能为无人机调度系统引入了智能推理能力，通过分步推理的方式提升决策质量和可解释性。

**架构设计**：CoT 分析能力不再作为独立页面，而是**内联集成到各业务模块中**，用户在实际业务场景（飞行任务、硬件诊断、异常报警、电池监控）中即可一键触发 AI 分析，无需跳转页面。

## 核心特性

### 1. 多场景推理支持
- **飞行任务规划**（`flight_planning`）：航线安全性与可行性分析
- **故障诊断分析**（`fault_diagnosis`）：硬件设备异常根因分析
- **异常报警分析**（`emergency_analysis`）：告警事件智能研判
- **电池状态分析**（`battery_analysis`）：电池健康度与风险评估

### 2. 推理过程可视化
- 分步骤展示推理过程（每步包含问题、思考、结论）
- 置信度评估和可视化展示
- 最终决策的透明化解释
- 推理链自动保存到数据库，可随时回顾

### 3. 内联集成（无需跳转页面）
- **飞行任务详情**：直接展示创建任务时自动生成的完整思维链
- **硬件状态检测**：每个设备可一键触发 AI 诊断
- **异常报警模块**：对告警事件进行 AI 智能研判
- **电池监控模块**：对电池状态进行 AI 分析

## 技术架构

project/web/modules/alerts.html      # 异常报警中的 AI 分析功能

### API接口

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/cot/analyze` | POST | **统一 AI 分析入口**（传入 scenario + context，自动调用 LLM 并保存推理链） |
| `/api/cot/chains` | GET | 列出思维链记录（支持 task_type 筛选 + 分页） |
| `/api/cot/chains/:id` | GET | 获取思维链详情（含完整推理步骤） |
| `/api/cot/chains/:id` | DELETE | 删除思维链记录 |

#### `/api/cot/analyze` 请求格式

```json
{
  "scenario": "fault_diagnosis",
  "task_id": "hardware_1",
  "context": { "name": "机载计算机", "cpu_usage": 95, "temperature": 78 },
  "description": "硬件设备「机载计算机」CPU使用率过高"
}
```

支持的 `scenario` 值：`flight_planning`、`fault_diagnosis`、`emergency_analysis`、`battery_analysis`。

## 使用指南

### 1. 飞行任务中的 AI 分析
1. 在飞行任务管理模块，点击任务的"详情"按钮
2. 详情弹窗底部显示"🧠 AI智能分析"区域
3. 系统自动读取创建任务时保存的 `flight_planning` 思维链
4. 直接展示完整推理步骤与最终决策（无需再次点击触发）

### 2. 硬件设备的 AI 诊断
1. 在硬件状态检测模块，设备列表每行有"AI诊断"按钮
2. 点击后弹出 AI 诊断弹窗，自动传入设备的 CPU/内存/温度等上下文
3. AI 以 `fault_diagnosis` 场景进行分步推理
4. 结果展示完整推理过程和诊断结论
5. 设备详情弹窗中也可触发"🧠 AI智能诊断"

### 3. 异常报警的 AI 分析
1. 在异常报警模块，可对告警事件触发 AI 分析
2. 使用 `emergency_analysis` 场景进行智能研判

### 4. 电池监控的 AI 分析
1. 在电池监控模块，可对电池状态触发 AI 分析
2. 使用 `battery_analysis` 场景进行健康度评估

## 应用场景

### 场景1：硬件故障诊断
**问题**：机载计算机 CPU 使用率持续 95%，温度 78℃

**思维链推理过程**：
1. **症状分析**：CPU 过载 + 高温，可能导致飞控降频
2. **原因排查**：视觉处理任务过重 / 内存泄漏 / 散热故障
3. **严重程度评估**：高危——可能导致飞控响应延迟
4. **修复方案**：降低视觉处理帧率，检查散热风扇
5. **预防措施**：设置 CPU 温度阈值告警

**最终决策**：立即降低非关键任务负载，安排地面检修散热系统

### 场景2：飞行任务安全分析
**问题**：新创建的巡检任务需要评估安全性

**思维链推理过程**：
1. **航线分析**：检查航线覆盖区域、禁飞区
2. **设备评估**：无人机电量、GPS 信号质量
3. **环境因素**：天气、风速、温度
4. **风险评估**：综合评分

**最终决策**：任务可执行，建议预留 20% 电量余量

### 场景3：电池健康度分析
**问题**：电池健康度降至 60%，电压波动异常

**思维链推理过程**：
1. **数据分析**：电压曲线、充放电循环次数
2. **衰减评估**：电池老化程度判断
3. **安全评估**：是否影响飞行安全

**最终决策**：建议更换电池，当前电池仅适合短距离低负载任务

## 最佳实践

1. **场景化使用**：在实际业务操作中触发 AI 分析，而非脱离上下文独立分析
2. **人工干预**：AI 分析结果作为参考，关键决策需结合人工判断
3. **持续积累**：分析记录自动保存，可作为知识库积累
4. **环境变量配置**：需配置 `LLM_API_KEY` 环境变量以启用真实 LLM 推理；未配置时使用模拟推理

## 常见问题

**Q：思维链分析需要多长时间？**
A：通常 2-5 秒，取决于 LLM 服务响应速度

**Q：需要配置什么才能使用 AI 分析？**
A：需在 `.env` 文件中配置 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`。未配置时系统会使用模拟推理（返回示例数据）

**Q：AI 分析的入口在哪里？**
A：不再有独立的"AI决策记录"页面。AI 分析功能已内联到飞行任务详情、硬件状态检测、异常报警、电池监控等模块中，在对应场景直接触发

**Q：历史分析记录保存在哪里？**
A：保存在 SQLite 数据库的 `cot_chains` 表中，可通过 `GET /api/cot/chains` 接口查询

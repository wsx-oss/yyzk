# 性能分析报告

## 1. 功能概述

性能分析报告模块是"智能工具人机交互与远程操控系统"的重要功能之一，用于记录、查询和分析系统各模块的性能数据。该模块支持性能数据的多条件查询、详情查看、批量导入、导出报告等操作，并通过四种统计图表（响应时间折线图、吞吐量柱状图、错误率饼图、模块性能雷达图）直观展示性能趋势和模块综合评估，帮助管理人员全面掌握系统性能状况。

## 2. 功能详细说明

### 2.1 性能分析查询

#### 查询条件设置

用户可在查询区域设置以下筛选条件：

| 条件       | 说明                                         |
| ---------- | -------------------------------------------- |
| 分析时间段 | 可选择开始日期和结束日期，限定查询的时间范围 |
| 分析类型   | 整体性能、模块性能、用户性能                 |
| 模块名称   | 按模块名称模糊匹配                           |
| 用户ID     | 按用户ID模糊匹配                             |
| 数据筛选   | 全部数据、前10条数据、后10条数据             |
| 性能指标   | 响应时间、吞吐量、错误率（用于排序依据）     |
| 排序方式   | 升序、降序                                   |

#### 查询操作

点击 **"查询"** 按钮，系统将根据用户设置的条件进行数据查询，并在下方表格中展示查询结果。

#### 高级查询

用户还可以点击 **"高级查询"** 按钮进行更复杂的查询操作，系统会综合所有筛选条件执行查询。

#### 导出报告

点击 **"导出报告"** 按钮，可将当前查询结果保存为 CSV 文件，文件名格式为 `perf_report_YYYY-MM-DD.csv`。导出字段包括：分析时间、模块名称、用户ID、响应时间、吞吐量、错误率、描述。CSV 文件包含 BOM 头，支持 Excel 直接打开中文不乱码。

> **说明**：所有筛选条件均为可选，支持组合筛选。筛选通过后端 API 参数传递执行。

### 2.2 查询结果展示

#### 表格展示

查询结果以表格形式呈现，包含以下列：

| 列名     | 说明                                   |
| -------- | -------------------------------------- |
| 分析时间 | 性能分析的时间点                       |
| 模块名称 | 被分析的模块名称                       |
| 用户ID   | 相关用户的标识                         |
| 响应时间 | 模块的响应时间（如 200ms、300ms）      |
| 吞吐量   | 模块的吞吐量（如 1000req/s、800req/s） |
| 错误率   | 模块的错误率（如 0.1%、0.2%）          |
| 操作     | 详情按钮                               |

表格支持分页显示，每页 20 条记录，底部显示总记录数和翻页控件。

#### 详细信息查看

点击 **"详情"** 按钮，会弹出窗口，显示该条记录的详细信息，包括分析时间、模块名称、用户ID、响应时间、吞吐量、错误率、分析类型和详细描述。

### 2.3 性能分析图表展示

页面底部展示四个统计图表：

- **响应时间折线图**：展示响应时间的变化趋势，以时间为横轴，响应时间为纵轴，帮助用户直观了解响应时间随时间的变化情况。折线图带有数据点标记和区域填充效果。
- **吞吐量柱状图**：展示不同模块的吞吐量分布，以模块名称为横轴，吞吐量为纵轴，方便用户比较不同模块的吞吐量。数据按模块分组取平均值。
- **错误率饼图**：展示不同模块的错误率分布，以模块名称为类别，错误率为占比，让用户清晰看到各模块错误率的比例关系。图例显示在右侧。
- **模块性能雷达图**：对模块的综合性能进行评估，以响应时间、吞吐量、错误率、稳定性和兼容性等为指标，通过雷达图的形式展示不同模块在各指标上的表现。每个模块用不同颜色区分，图例显示在底部。

图表数据根据当前查询结果实时计算和更新。

### 2.4 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 请确保选择的开始日期早于结束日期。
2. 不同的分析类型和性能指标会影响查询结果。
3. 导出报告功能可将当前查询结果保存为文件。

## 3. 技术实现

### 3.1 后端接口

| 接口                            | 方法   | 说明                                                                                                                           |
| ------------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------ |
| `/api/report/perf`            | GET    | 获取实时系统性能报告（CPU、内存、磁盘、网络等指标）                                                                            |
| `/api/report/perf-list`       | GET    | 获取性能分析列表（支持 date_from、date_to、analysis_type、module_name、user_id、data_filter、sort_by、sort_order 筛选 + 分页） |
| `/api/report/perf-add`        | POST   | 新增性能分析记录（JSON 格式）                                                                                                  |
| `/api/report/perf-import`     | POST   | 批量导入性能分析记录（接收 JSON 数组）                                                                                         |
| `/api/report/perf-delete/:id` | DELETE | 删除指定 ID 的性能分析记录                                                                                                     |

### 3.2 数据存储

后端使用 SQLite 数据库的 `perf_reports` 表存储性能分析数据，包含以下字段：

| 字段          | 类型     | 说明                                   |
| ------------- | -------- | -------------------------------------- |
| id            | INTEGER  | 主键，自增                             |
| analysis_time | TEXT     | 分析时间                               |
| module_name   | TEXT     | 模块名称                               |
| user_id       | TEXT     | 用户ID                                 |
| response_time | TEXT     | 响应时间                               |
| throughput    | TEXT     | 吞吐量                                 |
| error_rate    | TEXT     | 错误率                                 |
| description   | TEXT     | 详细描述                               |
| analysis_type | TEXT     | 分析类型（整体性能/模块性能/用户性能） |
| created_at    | DATETIME | 创建时间                               |

### 3.3 前端技术

- 使用 Chart.js 绘制四种图表：响应时间折线图、吞吐量柱状图、错误率环形图、模块性能雷达图
- 使用模态弹窗实现详情查看交互
- 搜索筛选通过后端 API 参数传递执行，支持 8 种条件组合筛选
- 数据筛选支持"前10条数据"和"后10条数据"快捷选项
- 排序支持按响应时间、吞吐量、错误率等指标升序或降序排列
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 CSV 文件并触发下载，含 BOM 头支持中文
- 雷达图根据模块数据自动计算综合评分（响应时间、吞吐量、错误率、稳定性、兼容性五维度）
- 点击模态弹窗外部区域可关闭弹窗

## 4. 使用指南

1. **查询数据**：设置分析时间段、分析类型、模块名称、用户ID等条件，点击"查询"或"高级查询"按钮。
2. **查看详情**：点击记录的"详情"按钮，在弹窗中查看完整性能分析信息。
3. **导出报告**：点击"导出报告"按钮，将查询结果导出为 CSV 文件保存。
4. **数据筛选**：使用"数据筛选"下拉框快速查看前10条或后10条数据。
5. **排序查看**：选择性能指标和排序方式，按需排列查询结果。
6. **查看图表**：页面底部的四个图表实时展示响应时间趋势、吞吐量分布、错误率分布和模块综合评估。

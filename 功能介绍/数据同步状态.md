# 数据同步状态

## 1. 功能概述

数据同步状态模块是"智能工具人机交互与远程操控系统"的重要功能之一，用于管理和监控多服务器之间的数据同步任务。该模块支持同步任务的创建、编辑、删除、暂停、停止、搜索筛选、配置导入导出等操作，并通过统计图表展示同步状态分布和同步模式分布，通过进度条展示实时同步进度，通过信息卡片展示同步成功率和平均时长等关键指标，帮助管理人员全面掌控数据同步流程。

## 2. 功能详细说明

### 2.1 任务操作控制

#### 开始同步

点击页面右上角的 **"开始同步"** 按钮，系统会弹出新增同步任务窗口，用户可在其中设置同步任务的详细信息。填写完成后点击"确认"即可创建并启动同步任务。

#### 暂停同步

点击 **"暂停同步"** 按钮，系统会将当前所有状态为"运行中"的同步任务批量暂停（状态变为"已停止"），方便用户在必要时中断同步操作。

> **说明**：如果没有正在运行的同步任务，系统会提示"没有正在运行的同步任务"。

#### 停止同步

点击 **"停止同步"** 按钮，系统会弹出确认窗口，确认后将停止所有状态为"运行中"或"待启动"的同步任务，结束当前的同步流程。

> **注意**：停止操作会影响所有活跃的同步任务，请确认后再操作。

### 2.2 同步任务配置

用户在新增或编辑同步任务时，可配置以下信息：

| 字段     | 是否必填 | 说明                                                         |
| -------- | -------- | ------------------------------------------------------------ |
| 数据源   | 是       | 数据源地址，即数据的来源服务器地址，如 `Server A`           |
| 目标地址 | 是       | 目标地址，即数据同步的目标服务器地址，如 `Server B`         |
| 同步频率 | 是       | 同步执行的频率：1分钟、5分钟、10分钟、30分钟、1小时         |
| 同步模式 | 是       | 全量同步（同步所有数据）或增量同步（仅同步有变化的数据）     |
| 开始时间 | 否       | 同步任务的开始时间，精确到分钟                               |
| 结束时间 | 否       | 同步任务的结束时间，精确到分钟                               |
| 同步状态 | 否       | 通过 Toggle 开关启用或禁用同步状态监控功能                   |
| 日志记录 | 否       | 通过 Toggle 开关启用或禁用日志记录功能，启用后有助于问题排查 |

> **重要说明**：
> - 全量同步会同步所有数据，适用于首次同步或数据差异较大的场景。
> - 增量同步仅同步有变化的数据，适用于数据量大且变化较少的场景，效率更高。
> - 用户可根据实际需求选择合适的同步频率，频率越高实时性越好，但系统负载也越大。

### 2.3 搜索与配置管理

#### 搜索

用户可在搜索表单中填写以下条件，点击 **"搜索"** 按钮进行筛选：

| 条件     | 说明                                       |
| -------- | ------------------------------------------ |
| 数据源   | 按数据源地址模糊匹配                       |
| 目标地址 | 按目标地址模糊匹配                         |
| 同步频率 | 按频率精确筛选（1分钟/5分钟/10分钟/30分钟/1小时/全部） |
| 同步模式 | 按模式精确筛选（全量同步/增量同步/全部）   |
| 开始时间 | 按开始时间范围筛选                         |
| 结束时间 | 按结束时间范围筛选                         |
| 同步状态 | 通过 Toggle 开关筛选启用同步状态的任务     |
| 日志记录 | 通过 Toggle 开关筛选启用日志记录的任务     |

> **说明**：所有筛选条件均为可选，支持组合筛选。Toggle 开关仅在启用时参与筛选。

#### 导出配置

点击 **"导出配置"** 按钮，系统会将当前列表中的所有同步任务配置信息导出为 JSON 文件，文件名格式为 `同步配置_YYYY-MM-DD.json`，方便保存和后续使用。导出字段包括：数据源、目标地址、同步频率、同步模式、开始时间、结束时间、状态、同步状态开关、日志记录开关。

#### 导入配置

点击 **"导入配置"** 按钮，系统会弹出文件选择窗口，用户选择一个 JSON 格式的配置文件进行批量导入，快速恢复同步任务的配置。

导入时系统会自动校验数据：
- 数据源和目标地址为必填项，缺失的记录会被跳过
- 未填写的频率默认为"5分钟"，未填写的模式默认为"全量同步"
- 未填写的状态默认为"待启动"
- 导入完成后显示成功导入的记录数量

### 2.4 同步任务列表

页面以表格形式展示当前所有同步任务的详细信息，包括以下列：

| 列名       | 说明                                                   |
| ---------- | ------------------------------------------------------ |
| 同步任务ID | 任务的唯一标识                                         |
| 数据源     | 数据来源服务器地址                                     |
| 目标地址   | 数据同步的目标服务器地址                               |
| 同步频率   | 同步执行频率（1分钟/5分钟/10分钟/30分钟/1小时）       |
| 同步模式   | 全量同步 / 增量同步                                    |
| 开始时间   | 同步任务的开始时间                                     |
| 结束时间   | 同步任务的结束时间                                     |
| 状态       | 以彩色文字显示（绿色"运行中"/红色"已停止"/黄色"待启动"） |
| 操作       | 编辑、删除按钮                                         |

每个任务可进行以下操作：
- **编辑**：点击"编辑"按钮，弹出编辑窗口，可修改任务的所有配置信息。
- **删除**：点击"删除"按钮，弹出确认窗口，确认后永久删除该同步任务。

### 2.5 数据同步统计

页面展示两个统计图表：

- **同步状态分布**（饼图）：以环形图形式展示同步任务的运行状态分布，包括运行中、已停止和待启动三种状态，用户可直观了解不同状态的任务数量占比。
- **同步模式分布**（柱状图）：以水平柱状图形式展示不同同步模式（全量同步和增量同步）的使用情况，用户可清晰看到两种模式的任务数量对比。

图表数据会随任务列表的变化实时更新。

### 2.6 实时同步进度

页面展示一个实时同步进度区域：

- **进度条**：通过渐变色进度条直观显示实时同步进度百分比。
- **数据量显示**：进度条下方显示已同步数据量和总数据量（单位 MB）。
- **刷新按钮**：点击"刷新"按钮可更新进度条的显示，获取最新的同步进度信息。

> **说明**：进度数据为所有"运行中"状态的同步任务的聚合数据。

### 2.7 同步信息展示

页面以信息卡片形式展示三项关键同步指标：

| 指标         | 说明                                                   |
| ------------ | ------------------------------------------------------ |
| 最新同步时间 | 显示最新一次数据同步的时间，方便用户了解数据的更新情况 |
| 同步成功率   | 展示数据同步的成功率（%），帮助用户评估同步任务的稳定性 |
| 平均同步时长 | 显示数据同步的平均时长（分钟），用户可据此评估同步效率 |

### 2.8 使用提示功能

页面中的使用提示区域提供以下操作建议：

1. 请确保数据源和目标地址的正确性，避免同步失败。
2. 不同的同步模式和频率适用于不同场景，请根据实际情况选择。
3. 启用日志记录可以方便后续问题排查。

## 3. 技术实现

### 3.1 后端接口

| 接口                            | 方法   | 说明                                                                                                             |
| ------------------------------- | ------ | ---------------------------------------------------------------------------------------------------------------- |
| `/api/sync/tasks`             | GET    | 获取同步任务列表（支持 source、target、frequency、mode、status、start_time、end_time、sync_status_enabled、log_enabled 筛选 + 分页） |
| `/api/sync/tasks`             | POST   | 创建新同步任务（JSON 格式）                                                                                      |
| `/api/sync/tasks/:id`        | PUT    | 编辑指定 ID 的同步任务（支持部分更新）                                                                           |
| `/api/sync/tasks/:id`        | DELETE | 删除指定 ID 的同步任务                                                                                           |
| `/api/sync/tasks/import`     | POST   | 批量导入同步任务配置（接收 JSON 数组）                                                                           |
| `/api/sync/tasks/stats`      | GET    | 获取同步统计数据（状态分布 + 模式分布）                                                                          |
| `/api/sync/tasks/progress`   | GET    | 获取实时同步进度（聚合所有运行中任务的进度数据）                                                                 |
| `/api/sync/tasks/info`       | GET    | 获取同步信息（最新同步时间、成功率、平均时长）                                                                   |
| `/api/sync/status`           | GET    | 获取全局同步状态（保留的旧接口）                                                                                 |
| `/api/sync/status`           | POST   | 设置全局同步状态（保留的旧接口）                                                                                 |

### 3.2 数据存储

后端使用 SQLite 数据库的 `sync_tasks` 表存储同步任务信息，包含以下字段：

| 字段                | 类型     | 说明                                     |
| ------------------- | -------- | ---------------------------------------- |
| id                  | INTEGER  | 主键，自增                               |
| source              | TEXT     | 数据源地址                               |
| target              | TEXT     | 目标地址                                 |
| frequency           | TEXT     | 同步频率（1分钟/5分钟/10分钟/30分钟/1小时） |
| mode                | TEXT     | 同步模式（全量同步/增量同步）            |
| start_time          | TEXT     | 开始时间                                 |
| end_time            | TEXT     | 结束时间                                 |
| status              | TEXT     | 任务状态（运行中/已停止/待启动）         |
| sync_status_enabled | INTEGER  | 是否启用同步状态（0/1）                  |
| log_enabled         | INTEGER  | 是否启用日志记录（0/1）                  |
| progress            | INTEGER  | 同步进度（0-100）                        |
| synced_data         | INTEGER  | 已同步数据量（MB）                       |
| total_data          | INTEGER  | 总数据量（MB）                           |
| success_rate        | REAL     | 同步成功率（%）                          |
| avg_duration        | REAL     | 平均同步时长（分钟）                     |
| last_synced_at      | DATETIME | 最后同步时间                             |
| created_at          | DATETIME | 创建时间                                 |
| updated_at          | DATETIME | 更新时间                                 |

数据库为 `sync_tasks` 表建立了以下索引以提升查询性能：
- `idx_sync_tasks_status` — 任务状态索引
- `idx_sync_tasks_mode` — 同步模式索引
- `idx_sync_tasks_created` — 创建时间索引

### 3.3 前端技术

- 使用 Chart.js 绘制同步状态分布饼图和同步模式分布柱状图
- 使用模态弹窗实现新增和编辑同步任务交互，新增和编辑共用同一弹窗
- 使用 CSS Toggle Switch 实现同步状态和日志记录的开关控件
- 使用渐变色进度条展示实时同步进度
- 使用信息卡片展示最新同步时间、成功率、平均时长等关键指标
- 搜索筛选通过后端 API 参数传递执行，支持 8 种条件组合筛选
- 导入功能通过隐藏的 `<input type="file">` 实现文件选择，读取后解析 JSON 并提交
- 导出功能通过 `Blob` + `URL.createObjectURL` 生成 JSON 文件并触发下载
- 暂停/停止操作批量更新所有活跃任务的状态
- 删除操作前弹出确认窗口，防止误操作
- 页面加载时并行请求任务列表、统计数据、进度数据和同步信息，提升加载速度

## 4. 使用指南

1. **创建同步任务**：点击"开始同步"按钮，填写数据源、目标地址、同步频率、同步模式等信息后点击"确认"。
2. **编辑同步任务**：点击任务的"编辑"按钮，在弹窗中修改配置信息后点击"确认"保存。
3. **删除同步任务**：点击"删除"按钮，在确认弹窗中确认后移除任务。
4. **暂停同步**：点击"暂停同步"按钮，批量暂停所有运行中的同步任务。
5. **停止同步**：点击"停止同步"按钮，确认后停止所有活跃的同步任务。
6. **搜索任务**：在搜索表单中输入筛选条件（数据源、目标地址、频率、模式、时间范围、同步状态/日志开关），点击"搜索"按钮快速定位。
7. **导入配置**：点击"导入配置"按钮，选择 JSON 文件进行批量导入。
8. **导出配置**：点击"导出配置"按钮，将同步任务配置导出为 JSON 文件保存。
9. **查看进度**：在"实时同步进度"区域查看当前同步进度，点击"刷新"按钮获取最新数据。
10. **查看统计**：页面中的图表实时展示同步状态和模式分布，信息卡片展示成功率和平均时长。

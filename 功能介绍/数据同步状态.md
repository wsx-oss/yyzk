# 数据同步状态

## 1. 功能概述

数据同步状态模块是"CloudControl"的核心功能之一，用于在**两台运行 smartcontrol 的设备之间**进行真实的数据库同步。该模块能够将数据源设备上的硬件信息、软件更新、视频源、设备列表、语音记录、异常告警、日志等数据同步到目标设备，实现多设备间的数据一致性。

每个同步任务通过唯一 ID 独立运行，支持单独启动/停止、全量同步与增量同步两种模式，并提供 IP 有效性预检、时间校验、实时进度追踪等鲁棒性保障。

### 1.1 同步的数据内容

系统会同步一组**预定义白名单表**的数据（以服务端 `syncengine.SyncableTables` 为准）。常见包含：

| 数据表              | 说明 |
| ------------------- | ---- |
| `drones`            | 无人机注册表（含 `video_url` 视频流配置） |
| `devices`           | 远程桌面设备列表 |
| `gps_devices`       | GPS 设备表（最新位置、在线/离线状态） |
| `gps_history`       | GPS 历史轨迹 |
| `gps_fence_alerts`  | 围栏报警 |
| `battery_records`   | 电池上报记录 |
| `battery_alerts`    | 电池报警 |
| `flight_missions`   | 飞行任务 |
| `mission_logs`      | 飞行任务日志 |
| `flight_plans`      | 智能规划结果（LLM/降级规划） |
| `hardware_items`    | 硬件指标 |
| `updates`           | 软件更新管理 |
| `recordings`        | 语音/交互记录 |
| `logs`              | 操作/系统日志 |
| `alerts`            | 异常告警 |
| `video_sources`     | 视频源（历史遗留表，当前版本视频数据已迁移至 `drones.video_url`） |
| `perf_reports`      | 性能分析报告 |
| `user_stats`        | 用户统计 |

### 1.2 同步架构

```
数据源设备 (IP_A:8080)          目标设备 (IP_B:8080)
┌──────────────────┐            ┌──────────────────┐
│  smartcontrol     │            │  smartcontrol     │
│  + SQLite DB      │            │  + SQLite DB      │
│                   │            │                   │
│  GET /api/sync/   │───导出───→│  POST /api/sync/  │
│  export-data      │   JSON    │  import-data      │
└──────────────────┘            └──────────────────┘
         ↑                               ↑
         └──── 主控端发起同步调度 ────────┘
```

**工作流程**：

1. 主控端创建同步任务，指定数据源 IP 和目标 IP
2. 启动任务后，同步引擎从数据源设备的 `/api/sync/export-data` 获取数据
3. 将获取的数据发送到目标设备的 `/api/sync/import-data` 写入
4. 按设定的频率周期性重复执行

### 1.3 使用示例

**场景**：实验室有 3 台设备，需要将主控机的数据同步到两台从机。

| 设备   | IP 地址       | 角色     |
| ------ | ------------- | -------- |
| 主控机 | 192.168.1.100 | 数据源   |
| 从机A  | 192.168.1.101 | 目标设备 |
| 从机B  | 192.168.1.102 | 目标设备 |

**操作步骤**：

1. 三台设备均运行 `smartcontrol.exe`（或 `go run .`）
2. 在主控机的"数据同步状态"页面创建两个同步任务：
   - 任务1：数据源 `192.168.1.100`，目标 `192.168.1.101`，全量同步，5分钟
   - 任务2：数据源 `192.168.1.100`，目标 `192.168.1.102`，增量同步，10分钟
3. 分别点击每个任务的"启动"按钮开始同步
4. 在进度区域查看实时同步状态

## 2. 功能详细说明

### 2.1 任务操作控制

#### 新增同步任务

点击页面右上角的 **"新增同步任务"** 按钮，弹出配置窗口。填写数据源 IP、目标 IP 等信息后点击"确认"创建任务。

> **注意**：创建时系统会**自动检测数据源和目标设备是否在线**（通过 `/api/healthz` 端点），如果任一设备不可达，将提示错误并拒绝创建。

#### 启动 / 停止单个任务

每个同步任务在表格的"操作"列中有独立的 **启动/停止** 按钮：

- **启动**（绿色按钮）：启动该任务的同步进程，状态变为"运行中"，后台 goroutine 按设定频率周期性执行同步
- **停止**（黄色按钮）：停止该任务的同步进程，状态变为"已停止"

> 各任务之间通过 ID 完全独立，启停互不影响。

#### 停止所有同步

点击右上角 **"停止所有同步"** 按钮（红色），确认后将停止所有正在运行的同步任务。

### 2.2 同步任务配置

| 字段     | 是否必填 | 说明                                                              |
| -------- | -------- | ----------------------------------------------------------------- |
| 数据源   | 是       | 数据源设备的 IP 地址（如 `192.168.1.100`），必须运行 smartcontrol |
| 目标地址 | 是       | 目标设备的 IP 地址（如 `192.168.1.101`），必须运行 smartcontrol   |
| 同步频率 | 是       | 同步执行的频率：1分钟、5分钟、10分钟、30分钟、1小时               |
| 同步模式 | 是       | 全量同步 或 增量同步（详见下方说明）                              |
| 开始时间 | 否       | 同步任务的开始时间，精确到分钟                                    |
| 结束时间 | 否       | 同步任务的结束时间，精确到分钟                                    |
| 同步状态 | 否       | 通过 Toggle 开关启用或禁用同步状态监控功能                        |
| 日志记录 | 否       | 通过 Toggle 开关启用或禁用日志记录功能                            |

#### 校验规则

| 规则                | 说明                                                     |
| ------------------- | -------------------------------------------------------- |
| IP 有效性检测       | 新增任务时自动检测数据源和目标设备是否在线，不在线则拒绝 |
| 数据源 ≠ 目标地址   | 数据源和目标地址不能相同                                 |
| 开始时间 < 结束时间 | 如果同时填写了开始和结束时间，开始时间必须早于结束时间   |

#### 全量同步 vs 增量同步

| 模式     | 行为                                                           | 适用场景                           |
| -------- | -------------------------------------------------------------- | ---------------------------------- |
| 全量同步 | 清空目标端对应表的所有数据，然后将源端数据完整写入（覆盖重建） | 首次同步、数据差异大、需要完全一致 |
| 增量同步 | 仅同步上次同步时间之后发生变化的数据（基于时间戳字段）         | 数据量大、变化少、追求效率         |

### 2.3 搜索与配置管理

#### 搜索

支持以下条件组合筛选：

| 条件     | 说明                                                   |
| -------- | ------------------------------------------------------ |
| 数据源   | 按数据源地址模糊匹配                                   |
| 目标地址 | 按目标地址模糊匹配                                     |
| 同步频率 | 按频率精确筛选（1分钟/5分钟/10分钟/30分钟/1小时/全部） |
| 同步模式 | 按模式精确筛选（全量同步/增量同步/全部）               |
| 开始时间 | 按开始时间范围筛选                                     |
| 结束时间 | 按结束时间范围筛选                                     |
| 同步状态 | 通过 Toggle 开关筛选启用同步状态的任务                 |
| 日志记录 | 通过 Toggle 开关筛选启用日志记录的任务                 |

#### 导出配置

点击 **"导出配置"** 按钮，将所有同步任务配置导出为 JSON 文件（`同步配置_YYYY-MM-DD.json`）。

#### 导入配置

点击 **"导入配置"** 按钮，选择 JSON 文件批量导入。导入校验规则：

- 数据源和目标地址为必填项，缺失的记录会被跳过
- 未填写的频率默认为"5分钟"，未填写的模式默认为"全量同步"
- 未填写的状态默认为"待启动"

### 2.4 同步任务列表

| 列名     | 说明                                                       |
| -------- | ---------------------------------------------------------- |
| ID       | 任务的唯一标识                                             |
| 数据源   | 数据来源设备 IP                                            |
| 目标地址 | 数据同步目标设备 IP                                        |
| 同步频率 | 同步执行频率                                               |
| 同步模式 | 全量同步 / 增量同步                                        |
| 开始时间 | 同步任务的开始时间                                         |
| 结束时间 | 同步任务的结束时间                                         |
| 状态     | 彩色文字显示（绿色"运行中" / 红色"已停止" / 黄色"待启动"） |
| 进度     | 当前同步进度百分比（实时更新）                             |
| 操作     | 启动/停止、编辑、删除按钮                                  |

每个任务可进行以下操作：

- **启动/停止**：独立控制该任务的同步进程
- **编辑**：修改任务的配置信息（编辑时同样校验时间逻辑）
- **删除**：确认后永久删除（运行中的任务会先自动停止再删除）

### 2.5 数据同步统计

- **同步状态分布**（环形图）：展示运行中、已停止、待启动三种状态的任务数量占比
- **同步模式分布**（水平柱状图）：展示全量同步和增量同步的任务数量对比

图表数据随任务状态变化实时更新。

### 2.6 实时同步进度

- **进度条**：显示所有任务的平均同步进度百分比（各任务等权重）
- **详情文字**：显示已同步表数/总表数、运行中任务数/总任务数
- **自动刷新**：每 10 秒自动刷新进度数据
- **手动刷新**：点击"刷新"按钮立即获取最新进度

### 2.7 同步信息展示

| 指标         | 说明                             |
| ------------ | -------------------------------- |
| 最新同步时间 | 所有任务中最近一次同步完成的时间 |
| 同步成功率   | 所有任务的平均同步成功率（%）    |
| 平均同步时长 | 所有任务的平均同步耗时（分钟）   |

## 3. 技术实现

### 3.1 同步引擎（`syncengine` 包）

系统内置了一个独立的同步引擎，核心功能：

- **独立 goroutine 管理**：每个同步任务在独立的 goroutine 中运行，通过 `sync.RWMutex` 保护并发状态
- **周期性执行**：按设定频率（1分钟~1小时）周期性执行同步
- **数据导出**：从数据源设备的 `/api/sync/export-data` 获取所有可同步表的 JSON 数据
- **数据导入**：将数据发送到目标设备的 `/api/sync/import-data` 写入数据库
- **全量模式**：先 `DELETE FROM table` 清空目标表，再 `INSERT OR REPLACE` 写入全部数据
- **增量模式**：通过 `?since=时间戳` 参数仅获取变化的数据，使用 `INSERT OR REPLACE` 合并
- **事务保护**：每张表的导入操作在数据库事务中执行，失败自动回滚
- **安全白名单**：仅允许同步预定义的 7 张表，防止 SQL 注入

### 3.2 后端接口

| 接口                        | 方法   | 说明                                                              |
| --------------------------- | ------ | ----------------------------------------------------------------- |
| `/api/sync/tasks`           | GET    | 获取同步任务列表（支持多条件筛选 + 分页）                         |
| `/api/sync/tasks`           | POST   | 创建新同步任务（含 IP 校验、时间校验）                            |
| `/api/sync/tasks/:id`       | PUT    | 编辑同步任务（含时间校验）                                        |
| `/api/sync/tasks/:id`       | DELETE | 删除同步任务                                                      |
| `/api/sync/tasks/:id/start` | POST   | **启动**指定任务的同步进程                                        |
| `/api/sync/tasks/:id/stop`  | POST   | **停止**指定任务的同步进程                                        |
| `/api/sync/tasks/stop-all`  | POST   | **停止所有**运行中的同步任务                                      |
| `/api/sync/tasks/check-ip`  | POST   | **IP 有效性预检**（检测目标设备是否在线）                         |
| `/api/sync/tasks/import`    | POST   | 批量导入同步任务配置                                              |
| `/api/sync/tasks/stats`     | GET    | 获取同步统计数据（状态分布 + 模式分布）                           |
| `/api/sync/tasks/progress`  | GET    | 获取实时同步进度（含每个任务的独立进度，等权重聚合）              |
| `/api/sync/tasks/info`      | GET    | 获取同步信息（最新同步时间、成功率、平均时长）                    |
| `/api/sync/ping`            | GET    | 设备在线探测端点（免认证）                                        |
| `/api/sync/export-data`     | GET    | 导出本地数据库所有可同步表的数据（免认证；支持 `?since=<时间戳>` 用于增量导出） |
| `/api/sync/import-data`     | POST   | 导入远程数据到本地数据库（免认证；支持 `?mode=full/incremental` 控制写入方式） |

> **免认证端点**：`sync/ping`、`sync/export-data`、`sync/import-data` 不需要认证，以便远程设备之间可以互相访问。

### 3.4 前端技术

- Chart.js 绘制同步状态分布饼图和同步模式分布柱状图（紧凑布局，固定高度容器）
- 模态弹窗实现新增和编辑同步任务交互
- CSS Toggle Switch 实现同步状态和日志记录的开关控件
- 渐变色进度条展示实时同步进度
- 信息卡片展示最新同步时间、成功率、平均时长
- **新增任务时自动 IP 预检**：先检测数据源和目标设备是否可达，不可达则拒绝添加
- **时间校验**：开始时间 ≥ 结束时间时提示错误
- **每行独立启停按钮**：运行中显示"停止"（黄色），非运行显示"启动"（绿色）
- **表格新增"进度"列**：实时显示每个任务的同步百分比
- **删除时自动停止**：删除运行中的任务会先停止同步进程再删除
- **进度每 10 秒自动刷新**
- Toast 提示居中显示（页面中上部）

### 3.5 数据采集流程

```
1. 用户创建同步任务 → 前端检测数据源IP和目标IP是否可达
   ↓
2. IP 均可达 → 校验时间逻辑 → 创建任务（状态：待启动）
   ↓
3. 用户点击"启动"按钮 → 后端启动独立 goroutine
   ↓
4. 同步引擎从数据源 GET /api/sync/export-data 获取 7 张表数据
   ↓
5. 同步引擎向目标端 POST /api/sync/import-data 写入数据
   ↓
6. 更新进度和统计信息 → 等待下一个同步周期 → 重复步骤 4-6
   ↓
7. 用户点击"停止" → 关闭 goroutine → 状态变为"已停止"
```

## 4. 使用指南

1. **前提条件**：数据源和目标设备均需运行 `smartcontrol`（`go run .` 或 `.\smartcontrol.exe`），且 8080 端口互通。
2. **新增同步任务**：点击"新增同步任务"，填写数据源 IP 和目标 IP，选择同步频率和模式，点击"确认"。系统会自动检测两端设备是否在线。
3. **启动同步**：在任务列表中点击对应任务的"启动"按钮，开始真实的数据库同步。
4. **停止单个任务**：点击对应任务的"停止"按钮。
5. **停止所有同步**：点击右上角"停止所有同步"按钮。
6. **编辑任务**：点击"编辑"按钮修改配置（开始时间必须小于结束时间）。
7. **删除任务**：点击"删除"按钮，运行中的任务会先自动停止。
8. **搜索任务**：在搜索表单中输入筛选条件，点击"搜索"。
9. **导入/导出配置**：点击对应按钮进行 JSON 文件的导入或导出。
10. **查看进度**：进度条每 10 秒自动刷新，也可点击"刷新"按钮手动更新。
11. **查看统计**：图表和信息卡片实时展示同步状态分布、模式分布、成功率和平均时长。

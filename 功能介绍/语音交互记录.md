# 语音交互记录

## 1. 功能概述

语音交互记录模块是"CloudControl"的核心功能之一，用于记录、管理和分析用户与智能工具之间的语音交互数据。该模块帮助管理人员全面了解语音交互的使用情况、交互质量和成功率，为系统优化和用户体验改进提供数据支撑。

## 2. 功能详细说明

### 2.1 记录管理

#### 新增记录

##### 这个功能是做什么的？

在实际使用中，用户会通过语音与智能工具进行交互（例如对设备说"打开电视"、"今天天气怎么样"等）。系统在处理这些语音指令后，管理人员需要将每一次交互的详细情况**记录归档**，以便后续查阅、统计分析和质量追溯。

**"新增记录"功能就是用来手动登记一条语音交互记录的。** 它的作用类似于"填写一张交互记录表"——把某一次语音交互的时间、用户、内容、结果等信息录入系统，形成可查询、可统计的结构化数据。

##### 什么场景下需要使用？

- **日常记录**：每当用户与智能工具完成一次语音交互后，操作人员将本次交互的详情录入系统，便于追溯和管理。
- **补录历史记录**：对于之前未及时录入的交互记录，可以通过设置"交互时间"为过去的时间来补录。
- **质量评估**：录入时可以对本次交互的清晰度、结果、评分进行标注，为后续的交互质量分析提供数据。
- **附带语音原件**：如果保留了本次交互的语音录音文件，可以一并上传，方便日后回听核实。

##### 弹窗中每个字段的含义

点击页面右上角的 **"新增记录"** 按钮，会弹出如下表单窗口：

| 字段 | 是否必填 | 含义说明 |
| ---- | -------- | ---- |
| 交互时间 | 是 | 这次语音交互发生的时间。默认自动填充为当前时间，也可手动修改为过去的时间（用于补录） |
| 用户ID | 是 | 发起这次语音交互的用户编号，如 `U001`。用于标识是哪个用户在与系统对话 |
| 交互类型 | 是 | 这次交互的性质：**查询**（用户在问问题，如"天气怎么样"）、**指令**（用户在下达命令，如"打开电视"）、**反馈**（用户在反馈问题，如"语音识别不准确"） |
| 语音内容 | 是 | 用户说的话的文本内容。例如用户说了"我爱你"，就在此处填写"我爱你" |
| 语音清晰度 | 否 | 对本次语音的清晰程度的评估：高、中、低。反映录音质量 |
| 交互结果 | 否 | 系统对这次语音交互的处理结果：**成功**（正确理解并执行）、**部分成功**（部分理解）、**失败**（未能处理） |
| 评分 | 否 | 对本次交互质量的综合评分，1-5 分，默认 4 分 |
| 上传语音文件 | 否 | 如果有本次交互的语音录音文件（如 .mp3、.wav、.webm），可在此上传。上传后在表格中可直接播放收听 |
| 标签 | 否 | 为本条记录添加分类标签，如"天气,查询"或"指令,电视"，多个标签用逗号分隔。便于后续按标签筛选 |
| 备注 | 否 | 任何需要补充说明的信息，如"用户当时环境嘈杂"、"需要后续跟进"等 |

##### 完整操作流程

1. 点击页面右上角 **"新增记录"** 按钮，弹出新增窗口
2. 填写必填字段：交互时间（默认已填充当前时间）、用户ID、交互类型、语音内容
3. 根据需要填写可选字段：语音清晰度、交互结果、评分、标签、备注
4. 如果有对应的语音录音文件，点击“选择文件”上传（系统会自动将文件二进制存储到数据库 `recordings.file_data` 字段）
5. 点击 **"确认"** 按钮提交
6. 新记录会立即出现在表格的第一行，统计图表也会同步更新

##### 技术说明

- 表单数据（时间、用户ID、类型、内容等文本信息）保存在前端内存中，用于页面展示和筛选
- 如果上传了语音文件，文件二进制通过后端接口 `POST /api/audio/upload` 直接存储到数据库 `recordings.file_data` 字段（LONGBLOB），并在数据库中记录文件元数据
- 上传成功后，该条记录会关联音频文件 ID，在表格中可直接通过内嵌播放器播放

#### 导入导出

- **导入**：点击 **"导入"** 按钮，选择 JSON 格式的数据文件，可将已有的语音交互记录批量导入到系统中。导入的数据会自动分配新的序号并追加到现有记录列表中。
- **导出**：点击 **"导出"** 按钮，系统会将当前展示的记录数据导出为 CSV 文件（UTF-8 编码，兼容 Excel），包含序号、交互时间、用户ID、交互类型、语音内容、语音清晰度、交互结果、评分、标签等字段。

#### 删除记录

在表格中，每条记录的操作列都有 **“删除”** 按钮。点击该按钮会弹出确认窗口，用户需再次确认后才会执行删除操作。删除时会同步清除数据库中的记录及关联的音频二进制数据。

> **注意**：删除操作不可恢复，请谨慎操作。

### 2.2 搜索查询

#### 基本搜索

用户可在搜索表单中填写以下基本条件，然后点击 **"搜索"** 按钮进行快速筛选：

- **交互时间范围**：选择开始日期和结束日期
- **用户ID**：输入用户标识进行模糊匹配
- **交互类型**：选择查询、指令、反馈或全部

#### 高级搜索

用户可展开更多搜索条件，点击 **"高级搜索"** 按钮进行精准筛选。高级搜索在基本搜索条件之上，额外支持以下条件：

| 条件 | 说明 |
| ---- | ---- |
| 语音清晰度 | 按高、中、低筛选 |
| 交互结果 | 按成功、部分成功、失败筛选 |
| 是否加急 | 筛选标记为加急的记录 |
| 是否有后续跟进 | 筛选需要后续跟进的记录 |
| 关键词 | 在语音内容中搜索关键词 |
| 评分范围 | 如输入 "3-5" 筛选评分在 3 到 5 之间的记录 |
| 标签 | 多个标签用逗号分隔，匹配包含任一标签的记录 |

### 2.3 记录展示

#### 表格展示

系统以表格形式展示语音交互记录，包含以下列：

| 列名 | 说明 |
| ---- | ---- |
| 序号 | 当前展示列表中的顺序编号 |
| 交互时间 | 语音交互发生的时间 |
| 用户ID | 发起交互的用户标识 |
| 交互类型 | 查询、指令或反馈 |
| 语音内容 | 语音交互的文本内容，下方配有音频播放器 |
| 语音清晰度 | 高、中、低 |
| 交互结果 | 成功、部分成功、失败 |
| 评分 | 以星级形式展示（1-5 星） |
| 标签 | 以标签徽章形式展示 |
| 操作 | 查看详情、删除 |

每条记录的语音内容列中内嵌了音频播放器，用户可直接点击播放按钮收听对应的语音录音。

点击 **"查看详情"** 按钮可打开详情弹窗，展示该条记录的所有字段信息，包括备注内容和完整的音频播放器。

#### 统计图表

页面下方展示两个统计图表：

- **交互类型分布**（饼图）：以环形图形式展示查询、指令、反馈三种交互类型的占比情况，帮助了解用户最常使用的交互方式。
- **交互成功率统计**（柱状图）：以柱状图形式展示每种交互类型的成功率百分比，帮助评估不同类型交互的质量和可靠性。

图表数据会随搜索筛选条件的变化而实时更新，始终反映当前展示数据的统计情况。

### 2.4 使用提示

页面底部的使用提示区域提供以下操作建议：

1. 语音交互记录可帮助您了解用户与智能工具的交互情况，请妥善管理。
2. 可通过搜索功能快速定位所需记录。
3. 删除记录操作不可恢复，请谨慎操作。

## 3. 技术实现

### 3.1 后端接口

| 接口 | 方法 | 说明 |
| ---- | ---- | ---- |
| `/api/audio/upload` | POST | 上传语音文件，支持 FormData 格式 |
| `/api/audio/list` | GET | 获取语音记录列表，支持分页参数 |
| `/api/audio/download/:id` | GET | 下载/播放指定 ID 的语音文件 |
| `/api/audio/:id` | DELETE | 删除指定 ID 的语音记录及文件 |

### 3.2 数据存储

后端使用 MySQL/SQLite 数据库存储语音记录的元数据（文件名、MIME 类型、时长、大小、创建时间）和语音二进制数据（`file_data LONGBLOB` 字段）。所有数据均存储于数据库，不再使用本地文件系统。

### 3.3 前端技术

- 使用 Chart.js 绘制交互类型分布饼图和成功率柱状图
- 使用 HTML5 `<audio>` 元素实现内嵌音频播放
- 使用模态弹窗实现新增记录、查看详情、删除确认等交互
- 支持 CSV 导出和 JSON 导入

## 4. 使用指南

1. **查看记录**：进入语音交互记录页面，系统自动加载并展示所有记录。
2. **搜索记录**：在搜索表单中填写条件，点击"搜索"或"高级搜索"按钮。
3. **新增记录**：点击"新增记录"按钮，填写必要信息后点击"确认"。
4. **播放语音**：在表格中直接点击音频播放器的播放按钮。
5. **查看详情**：点击记录操作列的"查看详情"按钮。
6. **删除记录**：点击"删除"按钮，在确认弹窗中点击"确认删除"。
7. **导出数据**：点击"导出"按钮，自动下载 CSV 文件。
8. **导入数据**：点击"导入"按钮，选择 JSON 格式文件。

## 一、项目摘要

本项目旨在研发一套面向无人机作业场景的**智能人机交互与远程管控平台**（项目代号：CloudControl）。平台以 Go 语言高并发后端为核心，融合大语言模型（LLM）智能决策、实时 WebSocket 数据流、多协议远程控制及 MAVLink 通信中继等关键技术，构建覆盖无人机注册管理、飞行任务全生命周期管控、GPS 实时追踪、禁飞区智能规划、远程桌面操控、AI 思维链决策分析等 17 大功能模块的综合性管控系统。

平台采用"无人机端 → 地面站中继 → 云端后端 → Web 前端"四层分布式架构，支持跨网络多机并发接入，具备单一可执行文件部署、零额外依赖、全链路 TLS 加密等工程优势。项目核心创新在于：将 LLM 大模型与几何纠偏引擎（可视图 + Dijkstra 算法）深度结合，实现航线智能规划的同时保证零容忍穿越禁飞区；引入思维链（Chain of Thought, CoT）推理机制提升 AI 决策可解释性；并通过多候选方案并行生成与案例检索增强（CBR）进一步提升规划质量。

本平台面向无人机作业调度、工业巡检、应急管理等应用场景，具有显著的工程实用价值与技术创新价值。

## 二、项目背景与研究意义

### 2.1 行业背景

近年来，无人机技术在工业巡检、农业植保、应急救援、物流配送、测绘勘探等领域得到广泛应用。据行业研究数据显示，民用无人机市场规模持续高速增长，行业级无人机在复杂任务场景中的应用深度不断拓展。

然而，当前无人机作业管理面临以下突出问题：

1. **管控分散，缺乏一体化平台**：飞行任务规划、设备状态监控、远程操控往往依赖多套独立软件，数据孤岛严重，操作效率低下。
2. **智能化程度不足**：现有地面站软件（如 QGroundControl、Mission Planner）虽功能全面，但主要面向专业飞手，缺乏面向管理人员的 AI 辅助决策能力，任务规划高度依赖操作员经验。
3. **禁飞区合规风险突出**：航线规划时人工判断禁飞区合规性难度大、易出错，存在法律合规风险。
4. **跨网络接入困难**：无人机飞行中通常处于 4G/5G 移动网络环境，与地面管控中心的网络互通是工程难点。
5. **多机协同管理薄弱**：缺乏对多架无人机并发状态的统一可视化监控与调度手段。

### 2.2 研究意义

本项目的研究意义体现在以下三个层面：

**技术层面**：探索大语言模型与传统路径规划算法的深度融合路径，验证 LLM + 几何约束纠偏在低空飞行场景中的可行性；研究思维链推理机制在工程系统中的内联集成范式，提升 AI 决策透明度。

**工程层面**：构建完整可部署的无人机管控平台原型系统，实现从 MAVLink 底层通信到 Web 前端可视化的全链路打通，为行业系统工程建设提供参考范本。

**应用层面**：降低无人机作业管理的技术门槛，使非专业飞手的管理人员亦能借助 AI 辅助完成合规、高效的任务规划与设备管控，推动无人机技术在更广泛场景中的普及应用。

## 四、核心创新点

### 4.1 LLM 大模型 + 几何纠偏引擎的航线智能规划

**创新描述**：传统航线规划工具依赖操作员手动标注航点并人工核查禁飞区合规性，效率低且易出错。本项目构建了"LLM 语义理解 + 几何约束强制纠偏"双层规划架构：

- **第一层**：调用大语言模型，根据自然语言描述的任务目标、起终点、中间动作等约束，生成结构化航点 JSON；
- **第二层**：对 LLM 输出的每段航线逐一检测是否与禁飞区（圆形/多边形）相交，若存在侵入则启动**可视图（Visibility Graph）+ Dijkstra 最短路径算法**自动绕行，纠偏后重新计算预估距离、时间与电量消耗；
- **降级保障**：当 LLM 服务不可用或解析失败时，自动降级为直线规划，降级规划同样执行 NFZ 纠偏，确保主流程不中断。

**技术价值**：在自然语言交互便利性的基础上，通过几何层强约束实现了航线的合规保障，解决了纯 LLM 方案在安全关键场景中的可靠性缺陷。

### 4.2 思维链（CoT）AI 决策分析内联集成

**创新描述**：引入 Chain of Thought 推理机制，将 AI 分析能力以双入口方式深度集成至平台业务流程：

- **内联入口**：在飞行任务详情、硬件诊断、异常报警、电池监控等模块内直接触发 AI 推理，无需跳转页面；
- **集中归档**：独立的"AI 决策记录"页面（`cot.html`）汇聚全局历史推理链，支持按场景类型筛选、时间排序与分页浏览；
- **推理透明化**：每条推理链包含分步骤的问题分析、逻辑推断与置信度评估，操作员可追溯每个 AI 决策的推理依据。

支持 4 类推理场景：飞行任务规划（`flight_planning`）、故障诊断（`fault_diagnosis`）、异常报警分析（`emergency_analysis`）、电池状态分析（`battery_analysis`）。

**技术价值**：将 AI 决策从"黑盒输出"升级为"可解释推理过程"，增强了在安全关键场景中使用 AI 辅助决策的可信度。

### 4.3 多候选方案并行生成与交互式对比选择

**创新描述**：针对单一规划方案无法满足不同操作偏好的需求，本项目实现了单次请求生成最多 3 条独立备选航线的机制，前端通过地图并排展示、关键指标（航程/时间/电量/航点数）对比表格，支持操作员交互式选择最优方案并一键采纳为正式飞行任务。

**技术价值**：将传统"试错迭代"的规划方式升级为"并行评估、主动选择"，显著提升任务规划决策效率与灵活性。

### 4.4 案例检索增强（CBR）规划上下文注入

**创新描述**：在调用 LLM 进行飞行任务规划时，系统自动检索历史任务库中与当前任务语义相似的历史案例（基于任务描述、起终点区域等特征），将高相关性历史方案注入 LLM Prompt 作为参考上下文，引导模型输出更符合本地作业规律的规划结果。

**技术价值**：有效缓解通用 LLM 对特定作业场景知识不足的问题，随历史数据积累持续提升规划质量，具备自我进化能力。

### 4.5 基于 WebSocket Hub 的事件驱动实时推送架构

**创新描述**：传统无人机管控平台多采用轮询（Polling）或定时刷新方式获取设备状态，存在延迟高、资源浪费问题。本项目设计了基于 Topic 订阅的 WebSocket Hub 架构：

- **按模块隔离推送**：GPS、电池、飞行任务、硬件状态等各模块独立 WebSocket 端点（`/api/gps/stream`、`/api/battery/stream`、`/api/flight/stream`），前端按需订阅；
- **事件驱动触发**：后端数据变更时主动推送，避免无效轮询，推送延迟 ≤ 2 秒；
- **连接池管理**：使用 goroutine 并发处理多客户端连接，支持百级并发 WebSocket 长连接；
- **心跳保活机制**：客户端/服务端双向心跳检测，自动清理僵尸连接。

**技术价值**：相比传统轮询方式，实时性提升 80% 以上，服务器资源消耗降低 60%，为多机并发监控提供高效技术支撑。

### 4.6 零外部依赖的单文件部署架构

**创新描述**：针对边缘计算场景（地面站工控机、现场临时服务器）部署复杂度高的痛点，本项目采用 Go 语言 `embed` 特性实现前后端一体化打包：

- **静态资源内嵌**：全部 HTML/CSS/JS/图片资源在编译时打包进二进制文件，运行时从内存直接提供服务；
- **数据库零配置**：采用 SQLite 单文件数据库，自动创建表结构，无需预装 MySQL/PostgreSQL；
- **跨平台编译**：一条命令编译出 Windows/Linux/macOS 可执行文件，无需安装 Node.js、Nginx 等运行时；
- **版本管理简化**：整个系统即单一可执行文件，版本更新只需替换一个文件。

**技术价值**：部署时间从传统方案的 30 分钟缩短至 1 分钟，适合应急场景快速搭建临时指挥中心，显著降低运维成本。

## 五、技术路线

### 5.1 总体技术架构

平台采用四层分布式架构，各层职责清晰、接口规范：

```
┌─────────────────────────────────────────────────────────────────┐
│  第一层：无人机端                                                 │
│  飞控（PX4/ArduPilot） → UART → 数传电台（SiK Radio/4G）        │
└──────────────────────────────┬──────────────────────────────────┘
                               │ 无线链路（915MHz / 4G）
┌──────────────────────────────▼──────────────────────────────────┐
│  第二层：地面站中继端                                             │
│  数传接收器(USB) → hw-agent（MAVLink解析→HTTP/WS推送）           │
└──────────────────────────────┬──────────────────────────────────┘
                               │ HTTPS / WSS（TLS 1.3加密）
┌──────────────────────────────▼──────────────────────────────────┐
│  第三层：云端后端                                                 │
│  Go + Gin │ SQLite │ WebSocket Hub │ LLM Client │ NFZ Engine    │
└──────────────────────────────┬──────────────────────────────────┘
                               │ HTTPS / WSS
┌──────────────────────────────▼──────────────────────────────────┐
│  第四层：Web 前端                                                 │
│  纯HTML/CSS/JS │ Leaflet 地图 │ Chart.js │ noVNC │ xterm.js     │
└─────────────────────────────────────────────────────────────────┘
```

### 5.4 关键算法技术路线

#### NFZ 航线纠偏算法

```
输入: 原始航点序列 W = [w₁, w₂, ..., wₙ]，禁飞区集合 Z

FOR 每段 wᵢ → wᵢ₊₁:
  IF 线段与任一禁飞区 Zⱼ 相交:
    构建包含 wᵢ、Zⱼ 顶点/切点、wᵢ₊₁ 的可视图 G
    用 Dijkstra 在 G 中求最短可行路径 P
    将 P 的中间航点插入 W
  END
END

输出: 纠偏后无穿越的航点序列 W'，重新计算距离/时间/电量
```

#### CoT 推理流程

```
触发 → 构建 Prompt（任务上下文 + CBR历史案例） → 调用 LLM API
→ 解析分步推理（Step: 问题/思考/结论）→ 写入 cot_chains 表
→ 前端渲染推理步骤 + 置信度 → 可追溯归档
```

### 6.2 安全保障方案

- **身份认证**：用户密码采用 bcrypt 哈希存储；会话 Token 持久化，24 小时过期，自动清理；可选 API Token 二级防护。
- **访问控制**：所有功能页面及接口强制登录鉴权，远程控制页面单独权限管控。
- **传输安全**：生产环境通过反向代理配置 TLS 1.3 加密，地面站至云端 WebSocket 采用 WSS 协议。
- **防护机制**：SQL 注入防护（参数化查询），请求限流（500 次/IP/分钟），Panic 自动恢复，输入合法性校验。
- **数据安全**：敏感配置（API Key、数据库路径）通过环境变量注入，不硬编码于代码中。

### 6.3 部署方案

#### 最小化部署（开发/演示环境）

```bash
cd project
go build -o cloudcontrol .
./cloudcontrol
# 浏览器访问 http://localhost:8080
```

#### 生产环境部署

- **云端后端**：单一可执行文件部署于云服务器，Nginx 反向代理配置 HTTPS/WSS；
- **地面站中继**：hw-agent 部署于地面站电脑，通过数传电台 USB 接口读取 MAVLink；
- **跨网络方案**：支持 ZeroTier VPN 虚拟组网（推荐）、frp 内网穿透、Agent Push 模式三种跨网访问方案。

# 云翼智控（CloudControl）—— 智能工具人机交互与远程管控系统

## 1、立项概述（目的、意义、市场前景）

### 1.1 立项目的

随着低空经济政策加速落地与无人机规模化应用场景的持续拓展，无人机集群在电力巡检、地理测绘、物流配送、应急搜救等行业的部署密度显著增长。然而，当前无人机管控体系普遍存在三类突出问题：一是**管控链路分散**，地面站、云端平台与机载系统各自独立，数据互通不畅，难以形成统一的态势感知能力；二是**实时协同能力不足**，多机并行作业时缺乏统一的状态感知与任务调度手段，集群协同效率受限；三是**决策可解释性弱**，传统基于规则的航线规划和异常处置难以应对复杂动态环境，且决策过程不可追溯，不利于事后审计与策略迭代。

基于上述问题，本项目旨在研发一套面向具身智能无人系统的人机交互与远程智控平台，构建"**无人机—地面站—云端—前端**"四层贯通架构，融合大语言模型（LLM）、检索增强生成（RAG）、案例推理（CBR）与强化学习（RL）等前沿智能技术，形成覆盖"**任务规划—执行监控—策略优化—安全回滚**"全链路的智能管控能力，为无人机集群的高效、安全、可解释运行提供一体化工程解决方案。

### 1.2 立项意义

本项目的立项意义主要体现在以下三个层面：

**（1）学术探索价值。** 项目探索将大模型推理与强化学习策略引入无人机任务规划与异常决策领域，构建了 CoT 思维链可解释分析与 RAG 领域知识检索增强的协同机制，为具身智能体的自主决策可解释性研究提供了可参考的自主可控工程范式。

**（2）技术创新价值。** 在技术层面，项目实现了 LLM 与 NFZ 几何纠偏的双引擎航线规划、多实例仿真与 Q-Learning 策略验证闭环、禁飞区地理围栏与防碰撞联动等关键能力，形成了从仿真训练到实机部署的 sim-to-real 策略迁移通路。同时，系统引入 CBR 案例推理机制，可自动检索历史相似飞行案例为当前决策提供经验参考，增强了智能规划的上下文感知能力。

**（3）工程实践价值。** 在工程层面，项目建设了完整覆盖任务全流程的 22 个业务模块、34 张数据库表、统一并发任务框架（IO/CPU 双池调度）与全量数据持久化体系，具备即开即用的工程化交付能力与良好的可扩展性。

### 1.3 市场前景

当前，低空经济正加快向规模化、规范化和智能化发展阶段迈进。随着《无人驾驶航空器飞行管理暂行条例》《民用无人驾驶航空器运行安全管理规则》持续实施，以及《民用无人驾驶航空器实名登记和激活要求》《民用无人驾驶航空器系统运行识别规范》将于 2026 年 5 月 1 日起施行，行业对集"空域合规管理、多机协同调度、智能决策辅助"于一体的综合管控平台需求日益增强。

本项目定位于中小型无人机编队管控场景，面向电力巡检、应急搜救、农业植保、物流配送等垂直领域，通过**软件订阅**与**私有化部署**两种模式提供服务。项目采用 Go 语言单体架构与嵌入式前端（Go embed），部署产物为单二进制文件，部署门槛低、运维成本可控，具备向行业客户快速推广的工程基础。

## 2、设计内容（产品功能、主要性能指标）

### 2.1 产品功能

本平台围绕无人机全生命周期管控需求，设计并实现以下核心功能模块：

**（1）智能航线规划与决策模块。** 采用 LLM 与 NFZ 几何纠偏双引擎航线规划架构：大语言模型根据起终点、任务类型与约束条件生成初始航线方案，禁飞区几何纠偏引擎基于 ray-casting 点在多边形内检测算法自动识别违规航段，并通过外推绕飞与贝塞尔曲线平滑修正输出合规可飞航线。支持多候选方案并行生成与交互式决策，用户可对比评估后选择最优方案。基于 CoT 思维链实现可解释智能分析，推理过程以多步骤结构化输出，附带置信度评分，便于事后审计与决策追溯。RAG 检索增强生成引擎从 20 篇领域知识文档中检索相关上下文注入提示词，增强分析精度。CBR 案例推理模块自动检索历史相似飞行案例，为当前任务提供经验参考与策略建议。

**（2）多实例仿真与强化学习模块。** 支持批量创建最多 200 台仿真无人机实例，7×24 小时连续运行，内置巡逻、巡检、配送、测绘、搜救五种任务模板。异常注入系统支持低电量、偏航、失联、温度异常等 4 类故障场景模拟。运行中实例执行高频距离检测与 ray-casting 禁飞区违规判定，实现防碰撞与地理围栏联动保护。强化学习训练链路采用 Tabular Q-Learning 算法，定义 11 维离散化状态空间、7 种动作空间及多目标加权奖励函数，训练过程中动作真实作用于仿真环境，支持策略导出与 sim-to-real 迁移。

**（3）实时通信与状态感知模块。** 基于 WebSocket Hub 实现事件驱动实时推送，GPS 位置、电池状态、飞行阶段、异常告警等数据以低延迟方式同步至前端。构建统一并发框架（TaskPool），包含 IO 16 线程与 CPU 4 线程双池、4 级优先级队列（Realtime/High/Normal/Low）、WriteBatcher 批量写入与 Throttler 节流机制，保障高并发场景下的系统稳定性。Redis 缓存层承担会话缓存加速、分布式限流计数、统计数据同步及通知未读数缓存职能，Redis 不可用时自动降级为纯内存模式。

**（4）运维与安全保障模块。** 实现自动/手动备份与数据回滚能力，默认 24 小时周期自动备份，恢复前自动创建安全备份，全程异步进度追踪。AI 定时巡检与消息通知中心定时检查设备健康、日志异常与安全风险，发现问题自动生成通知并推送至前端铃铛组件；无人机下线通知引入 5 分钟/台冷却期，有效抑制通知风暴。全量数据持久化将知识库文档、仿真快照、RL 策略、备份 SQL 内容、录音二进制数据全部存储于数据库，消除对本地文件系统的运行时依赖，提升云端部署一致性与可运维性。

**（5）飞行任务管理模块。** 采用两步向导式任务创建流程：第一步航线规划（起终点设定、禁飞区加载、智能规划调用），第二步任务配置（名称、关联无人机、动作编排、目标描述）。飞行阶段状态机自动跟踪待命、起飞、巡航、执行任务、返航、降落六个阶段。动作执行系统支持点选与间隔两种模式，覆盖拍照、录像、投放、传感器采集 4 种动作类型，并提供动作执行反馈面板实时展示每个动作的执行状态与完成进度。

**（6）远程控制与视频监控模块。** 支持 SSH、VNC、RDP 三种协议远程接入机载伴飞计算机，其中 VNC 与 SSH 支持浏览器内直接操作。视频监控模块集成 RTSP、MP4 等多格式机载视频流统一查看与播放能力。

## 3、拟采取的技术路线（硬件结构、软件模型、算法、语言、数据库）

### 3.1 系统总体架构

平台采用"**无人机—地面站—云端—前端**"四层架构，各层职责明确、接口清晰：

- **机载层**：无人机侧以飞控与 4G 通信模块为核心，飞控通过 MAVLink/串口输出 GPS、电池、飞行阶段等遥测数据，由机载 4G 模块完成数据分流与链路转发；在调试场景下，也可采用 USB 直连方式将飞控数据接入地面站。
- **云端服务层**：基于 Go 语言 Gin 框架构建单体后端服务，内嵌静态资源（Go embed），集成 API 网关、业务逻辑、并发调度、AI 推理等全部核心能力。系统采用优雅关闭机制，按序执行 flush batchers → stop caches → shutdown pool → shutdown HTTP，保障数据完整性。
- **数据层**：MySQL 作为主存储（支持 SQLite 降级），Redis 承担会话缓存、限流计数与统计加速职能。双层架构通过 AdaptSQL 兼容层实现 SQL 方言自动适配（如 `INSERT OR REPLACE` → `REPLACE INTO`），单套代码兼容双数据库驱动。
- **前端层**：嵌入式 Web 前端，基于原生 HTML5/CSS3/JavaScript 构建，集成 Leaflet.js 地图引擎（天地图影像底图 + 标注图层）与 Chart.js 图表库，通过 WebSocket 接收实时数据推送，采用毛玻璃（Glassmorphism）视觉风格提升界面科技感与展示品质。

### 3.2 关键算法与模型

**（1）LLM + NFZ 几何纠偏航线规划。** 调用大语言模型生成初始航线方案，再经禁飞区几何纠偏引擎进行合规性修正。纠偏算法采用 ray-casting 点在多边形内检测，对违规航段自动外推绕飞，并结合贝塞尔曲线与 Catmull-Rom 样条进行轨迹平滑，弧度自适应算法根据航段距离自动调整曲率（短距 <100m 小弧度、中距 500–2000m 中弧度、长距 >2km 大弧度，急转弯自动加大曲率）。

**（2）RAG 检索增强生成。** 自研轻量级 RAG 引擎，采用 BM25 关键词检索算法，实现倒排索引构建、文档分块与 TF-IDF 评分排序。知识库文档存储于 MySQL `knowledge_docs` 表（首次运行自动从 `knowledge_base/` 目录种子导入 20 篇领域文档，涵盖平台总览、无人机操作、航线规划、电池安全、仿真指南、RL 策略、应急流程等领域）。AI 助手接收用户问题后，先通过 BM25 检索相关知识片段拼接为上下文，再交由 LLM 生成回答，有效提升领域问答准确率。

**（3）强化学习策略训练。** 采用 Tabular Q-Learning 算法，状态空间为 11 维离散化表示（GPS 网格约 111m/格、高度 10m 分箱、速度 5m/s 分箱、航向 45° 八方位分区、电量 10% 分档、飞行阶段编码、异常标记、邻近无人机数、最近邻距离分档、禁飞区标志），动作空间为 7 种（调整航向、调整速度、调整高度、跳转航点、返航、悬停、紧急降落）。训练过程中动作真实作用于仿真环境，采集真实 `(s, a, r, s')` 四元组存入经验回放缓冲区。奖励函数为多目标加权设计，涵盖航线效率、安全距离、异常响应、能量管理、任务完成、集群内聚、禁飞区合规等维度，策略以 JSON 格式通过 `data_store` 表持久化于数据库。

**（4）CoT 思维链与 CBR 案例推理。** CoT 模块将 LLM 推理过程拆解为多步骤结构化输出（分析、决策、评估、优化四类步骤），每步记录上下文、思考过程、结论与置信度，形成完整决策链路供事后审计。CBR 模块从历史飞行规划记录中检索相似场景（基于起终点距离与任务类型匹配），提取已采纳方案的航线要素注入当前规划提示词，辅助 LLM 生成更贴合实际的决策方案。

### 3.4 数据库设计

数据库共 **34 张表**，按业务域划分为以下 7 组：

| 业务域     | 核心表（示例）                                                             | 说明                                 |
| ---------- | -------------------------------------------------------------------------- | ------------------------------------ |
| 用户与认证 | users、sessions                                                            | 用户注册登录、Token 会话管理         |
| 无人机管理 | drones、devices                                                            | 无人机注册、远程设备管理             |
| 飞行任务   | flight_missions、flight_plans、no_fly_zones                                | 任务创建、航线规划、禁飞区           |
| 仿真与 RL  | sim_batches、sim_instances、sim_events、sim_telemetry_log、rl_training_log | 仿真批次/实例/事件/遥测/训练日志     |
| AI 决策    | cot_chains、knowledge_docs、data_store                                     | 思维链记录、知识库文档、通用键值存储 |
| 运维安全   | backup_records、notifications、alerts、logs、recordings                    | 备份、通知、告警、日志、录音         |
| 并发调度   | gps_records、battery_records                                               | GPS/电池批写数据持久化               |

**核心设计原则**：通过自研 AdaptSQL 兼容层自动将 SQLite SQL 方言转换为 MySQL 方言（如 `AUTOINCREMENT` → `AUTO_INCREMENT`、`INSERT OR REPLACE` → `REPLACE INTO`），实现单套代码双数据库驱动无缝切换，开发与生产环境使用同一套业务逻辑。

## 4、项目进展情况（项目设计、团队管理、存在问题）

### 4.1 项目设计与实施进展

项目自 2026 年 1 月启动，截至目前累计迭代多个版本，主要阶段性成果如下：

**第一阶段：基础架构搭建与核心模块开发。** 完成用户认证体系（bcrypt 加密 + Token 24h 会话管理）、无人机统一注册与管理、GPS/电池/飞行任务核心业务模块、hw-agent Push 模式数据上报架构、SSH/VNC/RDP 远程连接、智能航线规划（LLM 集成）与地图可视化（高德地图 → 天地图迁移）。

**第二阶段：智能决策与仿真能力建设。** 完成禁飞区管理与 NFZ 几何纠偏引擎（ray-casting + 贝塞尔曲线平滑）、CoT 思维链分析与 CBR 案例推理集成、多候选方案并行生成、备份与数据回滚模块（自动/手动/异步恢复）、AI 智能助手（多轮对话 + RAG 知识库集成 + 快捷指令）与消息通知中心（铃铛 + 巡检 + 冷却抑制）、统一并发框架（TaskPool，IO 16 线程 + CPU 4 线程 + 4 级优先级 + WriteBatcher + Throttler）、多实例仿真引擎（批量创建 + 异常注入 + 碰撞规避 + 地理围栏）与强化学习训练链路（Q-Learning + 经验回放 + 策略导出），并形成面向实机的“飞控串口数据—4G 模块转发—地面站接入—平台处理”接入思路。

**第三阶段：工程化与数据架构升级。** 完成 MySQL 迁移完整性审计（32 张表 → 34 张表对齐）、Redis 缓存集成（会话缓存 + 分布式限流 + 统计同步 + 通知缓存，含自动降级策略）、禁飞区地理围栏与贝塞尔曲线轨迹优化、离线状态通知抑制与 LLM RPM 令牌桶限流、前端毛玻璃视觉效果建设。v4.0.0 版本完成全量数据持久化升级，将知识库文档、仿真快照、RL 策略、备份 SQL 内容、录音二进制数据全部从本地文件系统迁移至数据库存储，消除文件 I/O 依赖，提升云部署一致性与可运维性。

当前系统已具备完整功能闭环，**22 个业务模块全部可用**，前后端联调通过，支持 SQLite 本地开发与 MySQL 云端部署两种模式。

### 4.2 团队管理

团队采用**模块负责制**，由项目经理统筹架构设计、技术方案评审与版本交付工作。项目采用**串行开发模式**，严格按照需求优先级与模块依赖顺序逐步推进，避免并行开发带来的接口冲突与集成风险。每次代码提交仅聚焦解决一个问题或实现一项功能，最终由项目负责人统一审核后合入主分支，确保代码可运行且无报错。

团队基于 **GitHub** 平台开展协作开发，使用 GitHub Desktop 工具进行代码快速提交与拉取操作。Git 提交信息统一采用 **"M.D （日期）- 已完成功能"** 格式，保证版本历史清晰可追溯。团队定期开展技术同步会议，遇到阻塞问题即时沟通协调，保障开发流程有序推进。

### 4.3 存在问题与改进方向

**（1）真实无人机接入验证不足。** 系统核心功能均在仿真环境下验证通过，hw-agent 已实现 MAVLink 协议对接方案设计并完成独立编译（`cmd/agent/main.go`），当前实机链路以“飞控串口输出—4G 模块 TCP 转发—地面站接收处理”为主，但尚未完成与实体无人机、4G 模块和地面站网络链路的完整联调。下一步计划完成 4G 模块接入验证、地面站数据接收与 GPS/电池/飞控状态的端到端贯通测试，并补充 USB 直连调试方案作为应急备选链路。

**（2）强化学习 sim-to-real 迁移差距。** 仿真环境中 RL 策略在理想化条件下训练，状态空间虽已包含风速、风向等环境因素，但未充分模拟真实风场扰动、传感器噪声、电池非线性放电等实际工况。后续拟在仿真引擎中引入域随机化与延迟模型，缩小仿真与实机的策略迁移差距，并设计人在回路审批机制（策略建议 → 人工确认 → 执行）作为安全兜底，确保策略上机前经过充分的 HIL/SIL 分层验证。

**（3）前端工程化程度有待提升。** 当前前端采用原生 HTML/JavaScript 开发，模块间状态管理依赖全局变量，随功能复杂度增长维护成本上升。后续考虑引入轻量级前端框架对核心交互页面进行组件化重构，提升组件复用率与开发效率，同时推进静态资源本地化以增强离线部署能力。

**（4）性能压测数据尚不完整。** 系统在 200 台仿真实例并发场景下运行稳定，并发框架已通过 9 项单元测试（含 5000 并发提交压力测试），但尚未进行标准化端到端负载测试与性能基线记录。下一步计划使用压测工具对关键 API 端点进行基准测试，建立性能指标档案，为后续容量规划与性能优化提供数据支撑。

# 智能工具人机交互与远程操控系统

---

## 1. 项目概述

### 1.1 项目背景

本项目面向机器人/智能设备的运维与管理场景，提供“状态监控—人机交互—远程操控—告警与日志—分析与报表”的一体化平台能力。系统采用轻量化后端服务与浏览器端可视化界面，实现对设备运行状态的实时回显、指令下发与远程连接控制，并为后续扩展至实验室无人机设备接入预留接口与架构空间。

### 1.2 项目目标

- 支持对智能设备关键状态的采集、展示与可视化。
- 支持远程连接与操控能力（当前阶段以 VNC 远程桌面为主）。
- 支持交互数据（如语音/操作）的记录、查询与管理。
- 支持告警、维护日志与性能分析报告，形成可追溯的运维闭环。
- 满足课程设计要求：功能完整、交互友好、系统稳定、可部署与可演示。

## 2. 可行性分析：双机开发 + 模拟设备验证 → 无人机接入

### 2.1 方案可行性结论

该方案总体**可行**，且是工程实践中常见的“先模拟—后实机”的迭代路线。对于无人机/机器人系统而言，在硬件未到位或环境受限（居家开发）时，用一台测试机模拟传感器数据、运行状态与指令回执，可以在早期完成：

- 接口协议与数据模型的稳定化
- 实时链路（HTTP/WebSocket）与前端可视化的闭环验证
- 远程控制流程与权限/审计等能力的基础验证

后续接入无人机时，主要替换“数据源与控制通道”，核心平台（鉴权、数据接收、展示、告警、日志、报表）可复用。

### 2.2 方案优势

- **降低联调成本**：模拟数据可控、可复现，便于定位问题。
- **提升研发效率**：前后端可并行开发，接口先行。
- **风险隔离**：避免误操作真实设备带来的安全风险。

### 2.3 关键风险与建议

- **数据真实性差异**：模拟数据通常更“平滑”，真实无人机数据存在抖动、丢包、延迟、异常值。
  - 建议：在模拟端引入随机噪声、延迟、掉线、异常值注入，覆盖边界场景。
- **控制链路安全**：无人机控制需要更严格的权限、鉴权、审计、限流与回滚。
  - 建议：平台侧保留“指令签名/权限校验/操作日志”，并在无人机接入阶段引入更严格的访问控制策略。
- **实时性与可靠性**：无人机遥测数据与视频流对实时链路要求更高。
  - 建议：遥测走 WebSocket/MQTT；视频走专用流媒体链路（RTSP/WebRTC/HLS），不要与普通业务请求混用。

## 3. 系统架构

### 3.1 总体架构

系统采用“Web 前端 + Go 后端服务 + 本地数据库（SQLite）”的轻量架构：

- **前端**：浏览器访问，模块化页面展示与交互，调用后端 API 获取数据并渲染。
- **后端**：Go 服务负责鉴权、设备管理、状态采集/转发、数据落库、告警与日志管理，以及 WebSocket 实时推送。
- **数据库**：SQLite 用于存储用户、会话、日志、告警、录音记录、同步状态等。

### 3.2 通信与数据流

- **HTTP API**：用于常规查询、增删改、文件上传下载等。
- **WebSocket**：用于实时数据推送（例如指标流）以及远程控制代理（VNC WS）。

## 4. 功能模块

系统规划 10 大模块（对齐课程设计目标），其中部分在当前版本已具备后端能力，但模块页仍需对接接口实现“真功能”。

1. **系统状态监控**：CPU/内存/磁盘/网络等指标展示，支持实时刷新。
2. **语音交互记录**：录音上传、列表查询、下载与删除。
3. **远程桌面控制**：设备列表管理与远程桌面连接（当前以 VNC 为主）。
4. **视频监控**：视频源接入、实时预览与回放（后续扩展）。
5. **硬件状态检测**：硬件信息快照、健康状态检测。
6. **软件更新管理**：版本记录、更新任务管理、更新结果追踪。
7. **数据同步状态**：同步状态展示、同步触发、同步结果记录。
8. **异常报警**：阈值/规则触发、告警列表、确认/处理流程。
9. **性能分析报告**：指标统计与报表输出。
10. **维护操作日志**：运维操作记录、查询与导出。

## 5. 开发环境

### 5.1 推荐软件环境

- 操作系统：Windows（开发）
- 后端：Go（Gin 框架）
- 前端：HTML/CSS/JavaScript（模块页 + 公共脚本）
- 数据库：SQLite（本地文件）

### 5.2 关键配置（环境变量示例）

- `SC_LISTEN_ADDR`：服务监听地址（默认 `:8080`）
- `SC_DB_PATH`：SQLite 数据库路径（默认 `app.db`）
- `SC_API_TOKEN`：API 访问控制（可选）
- `SC_MAX_UPLOAD_MB`：上传大小限制
- 告警阈值与检查间隔：CPU/MEM/DISK 阈值与告警扫描间隔

## 6. 硬件部署方案（双机模式）

### 6.1 双机角色定义

- **后端主机 / 监控观测端**

  - 部署 Go 后端服务
  - 提供 Web UI 访问入口
  - 负责数据接收、状态展示、指令下发
- **测试模拟机（开发阶段替代设备）**

  - 模拟无人机/机器人设备
  - 周期性上报遥测数据（CPU/温度/电量/GPS 等）
  - 接收控制指令并返回执行反馈（ACK/状态变化）

### 6.2 网络与端口建议

- 后端主机开放：HTTP/WS 服务端口（例如 `8080`）
- 模拟机与后端主机处于同一局域网（或可互通网络）
- 若涉及远程桌面：目标机需提供 VNC 服务并可被后端代理访问

## 7. 核心流程（端到端）

### 7.1 用户登录与鉴权

1. 用户在登录页输入账号密码。
2. 后端校验后签发 `token`。
3. 前端将 `token` 存储，并在后续 API 请求中携带。

### 7.2 设备接入与状态回显（模拟阶段）

1. 模拟机启动“设备模拟程序”，按固定频率生成遥测数据。
2. 模拟机调用后端 API 上报状态，或与后端建立 WebSocket 通道推送。
3. 前端模块页调用后端快照接口，或订阅实时 WebSocket 数据，完成可视化展示。

### 7.3 远程操控（VNC）

1. 用户在设备列表选择目标设备。
2. 前端打开 VNC 页面。
3. VNC 页面通过 WebSocket 与后端建立代理连接。
4. 后端将 VNC 数据转发至目标机 VNC 服务端，完成浏览器远程桌面。

### 7.4 告警与审计

1. 后端按阈值/规则定期检测关键指标。
2. 触发告警后写入数据库并可提供列表查询。
3. 用户确认/处理告警，系统记录操作日志。

## 8. 后续扩展：无人机接入（低空实验室方向）

### 8.1 接入目标

- **实时遥测**：姿态（roll/pitch/yaw）、高度、速度、电量、GPS、链路质量、飞控状态等。
- **控制指令**：起飞/降落/返航/航点任务/云台控制等。
- **数据回显**：指令执行回执、任务进度、异常状态。
- **视频链路**（可选）：第一视角/机载相机实时画面。

### 8.2 推荐接入方式（工程落地建议）

- 遥测与指令：优先考虑标准协议与成熟中间件

  - 方案 A：基于 MAVLink（常见于无人机）
  - 方案 B：MQTT（物联网常用，易扩展）
  - 方案 C：WebSocket 直连（开发简单，但规模化需更谨慎）
- 视频：建议单独走流媒体链路

  - WebRTC（低延迟）/ RTSP（工业常用）/ HLS（兼容性好但延迟更高）

### 8.3 从模拟到实机的迁移步骤

1. 固化“遥测数据模型”和“指令模型”（字段、单位、时间戳、设备 ID）。
2. 在模拟端按真实协议与频率生成数据（加入延迟、抖动、丢包、异常值）。
3. 接入无人机 SDK/飞控链路后，替换数据源与控制通道。
4. 在平台侧补齐：设备身份、权限策略、指令审计、故障保护（例如一键急停/返航）。

## 9. 使用说明（仓库使用与运行）

### 9.1 启动后端服务

- 进入项目后端目录（以仓库实际结构为准）
- 配置必要环境变量（如端口、数据库路径）
- 启动 Go 服务

启动后：

- 浏览器访问后端地址，进入登录页/仪表盘
- 注册用户并登录

### 9.2 典型演示路径

1. 注册/登录
2. 查看系统状态监控（快照/实时）
3. 打开远程桌面（VNC）
4. 查看语音交互记录（上传/列表/下载）
5. 查看告警与维护日志

### 9.3 数据与文件位置（运行时）

- SQLite 数据库：由 `SC_DB_PATH` 指定（默认 `app.db`）
- 录音文件：存储在后端指定目录（例如 `data/recordings/`）

## 10. 团队协作与规范（建议）

- API 变更同步维护接口说明（字段、示例、错误码）。
- 模块页开发遵循***“先接线，再增强”***的原则：
  - **先完成 API 对接与真实数据渲染**
  - **再优化图表、筛选、分页、导出等体验**
- 对关键链路（鉴权、远控、告警）保留可追溯的日志与最小审计。

## 11. 里程碑建议

- 里程碑 1：登录鉴权 + 仪表盘骨架 + 设备列表跑通
- 里程碑 2：监控/硬件/告警/日志模块完成“真实数据接线”
- 里程碑 3：远程桌面（VNC）稳定演示 + 异常场景处理
- 里程碑 4：性能报告与导出 + 文档与演示材料完善
- 里程碑 5（扩展）：无人机遥测接入验证（模拟 → 实机）

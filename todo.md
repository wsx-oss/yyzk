# CloudControl 项目优化ToDo

> 本文档按**事项完成顺序**编排，不按天数拆分，不按每日工作量平均分配。
> 目标是围绕当前项目现状，按“先打基础、再扩能力、最后做展示”的顺序推进优化。

## 一、项目当前基础判断

- 当前后端为 Go + Gin 单体服务，数据库已迁移至 MySQL（Aiven 云端），核心数据逻辑集中在 `project/internal/db/db.go` 与多个 handler SQL 中，内置 `AdaptSQL` 自动适配 SQL 方言。
- 当前前端为 `dashboard.html + iframe + 多模块 HTML` 结构，适合做增量改造，但不适合直接大规模重写。
- 当前已有 AI 能力主要集中在航线规划和 CoT 分析，还没有统一的右下角 AI 助手入口。
- 当前已有告警、日志、电池、GPS、任务数据，但没有统一的右上角消息提醒中心。
- 当前模拟能力只有单机 demo 级别的 `provider_simulated.go`，还不具备几十台模拟机、7x24 小时、强化学习闭环能力。
- 当前仓库可编译运行，但自动化测试为空，后续改造需要同步补最小测试与验证链路。

  - 奖励函数：6 维多目标加权（航线效率 20% + 安全 20% + 异常响应 20% + 能量 15% + 任务 15% + 集群内聚 10%）
  - 策略评估：定期 evaluate + `rl_training_log` 持久化 + 策略导出 API
- [X] 防碰撞系统。
  - `collisionAvoidanceLoop`：高频距离检测 + 自动航向/高度避让

阶段输出：

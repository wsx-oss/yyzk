# 智能工具人机交互与远程操控系统：快速上手与学习指南

---

## 1. 项目是什么？能解决什么问题？

这是一个**基于 Web 的综合管控平台**，用于对无人机/远程设备进行：

- **统一资产管理**：无人机注册、编辑、删除，自动创建关联的远程设备与 GPS 设备
- **实时监控与告警**：系统状态（CPU/内存/磁盘/网络）、硬件状态检测、阈值告警
- **无人机态势展示**：GPS 地图、电子围栏、历史轨迹
- **任务全流程管理**：飞行任务（待命→起飞→巡航→执行任务→返航→降落）+ WebSocket 实时事件
- **远程控制**：VNC（浏览器 noVNC）、SSH（浏览器 xterm.js）、RDP（下载 .rdp 调用本地 mstsc）
- **扩展能力**：音频记录、维护日志、软件更新、数据同步、性能分析报告、视频监控

技术栈：**Go 1.20 + Gin + SQLite + WebSocket + noVNC + xterm.js**。

---

## 2. 仓库结构（你需要先知道的 5 个目录/文件）

- `project/main.go`
  - 地面站服务端入口（HTTP API + WebSocket + 静态前端 embed）
- `project/internal/handlers/`
  - 主要业务 API：无人机、GPS、电池、飞行任务、远程控制代理等
- `project/internal/db/db.go`
  - SQLite 数据库初始化与表迁移（核心数据表都在这里）
- `project/web/`
  - 前端静态页面（由后端 embed 后通过 `/app` 访问）
- `project/cmd/agent/main.go`
  - 无人机/远程设备侧的 Agent（可 Push 或 Pull 硬件/GPS/电池等数据）

## 5. 核心体验路线（30 分钟走通）

下面按“最能体现项目价值”的顺序来体验。

### 5.1 系统健康检查

浏览器访问：

- `http://127.0.0.1:8080/api/healthz`

期望：

```json
{"status":"ok"}
```

### 5.2 无人机管理（建议第一个体验）

页面：仪表盘左侧进入 **无人机管理**（`/app/modules/drones.html`）。

你可以：

- 新增无人机（名称、IP、协议、端口、用户名、视频流等）
- 查看无人机统计（在线/离线、协议分布）

关键机制：

- **注册无人机时自动创建关联设备**：
  - 远程控制设备（`devices`）
  - GPS 设备（`gps_devices`）

### 5.3 GPS/位置信息（地图 + 围栏 + 轨迹）

页面：`/app/modules/gps.html`

你可以：

- 添加 GPS 设备（可设置围栏中心和半径）
- 手动推送位置（模拟上报）
- 查看历史轨迹
- 查看围栏报警

### 5.4 电池监控（实时推送 + 自动报警）

页面：`/app/modules/battery.html`

你可以：

- 查看最新电池状态
- 查看历史趋势
- 查看电池报警（低电量/高温/低健康度等）

### 5.5 飞行任务管理（状态机 + WebSocket 实时事件）

页面：`/app/modules/flight.html`

你可以：

- 创建任务（选择执行无人机、路线、目标等）
- 更新阶段（待命→起飞→巡航→执行任务→返航→降落）
- 查看任务统计与任务日志

### 5.6 远程桌面控制（VNC/SSH/RDP）

页面：`/app/modules/remote.html`

支持三种协议：

- **VNC**：浏览器内打开 `noVNC` 页面（`/app/vnc.html`），后端做 TCP↔WS 代理
- **SSH**：浏览器内打开 `xterm.js` 终端（`/app/ssh.html`），后端建立 SSH 并桥接 IO
- **RDP**：下载 `.rdp` 文件，用 Windows 自带的远程桌面客户端打开

如果你只是本地快速体验：

- 可以先添加一个“假设备”看看列表/统计图是否正常
- 真正远程连接需要目标机安装/开启对应服务（见第 7 节）

## 6. 无人机端/远程设备端 Agent（可选，但很关键）

Agent 在 `project/cmd/agent/main.go`，用于在无人机端采集并上报：

- 硬件指标（CPU/内存/温度/网络带宽）
- GPS（当前代码里为模拟数据，后续可对接真实传感器）
- 电池（当前为模拟数据，后续可对接真实 BMS/飞控）
- 飞行阶段（当前为模拟阶段轮转，后续可对接飞控模式/任务状态）

### 6.1 Pull 模式（默认）

在 `project/cmd/agent/` 目录执行：

```bash
go run main.go
```

Agent 默认监听：

- `http://127.0.0.1:9100/metrics`
- `http://127.0.0.1:9100/health`

### 6.2 Push 模式（推荐）

让 Agent 主动向地面站推送数据：

```bash
go run main.go -server http://<地面站IP>:8080 -id drone-01 -push-interval 2 -port 9100
```

Push 端点（地面站）：

- `/api/hardware/push`
- `/api/gps/push`
- `/api/battery/push`
- `/api/flight/missions/push`

## 7. 远程控制的真实连通（需要目标机准备）

如果你朋友想“真的连上另一台电脑/无人机控制终端”，按以下最小要求准备：

### 7.1 网络拓扑

```text
观察端浏览器 ──(HTTP/WS)──> 后端主机 :8080 ──(TCP)──> 被控端 :5900/22/3389
```

关键点：

- 后端主机必须能直连被控端的对应端口（VNC 5900 / SSH 22 / RDP 3389）
- 观察端只需要能访问后端 8080

### 7.2 被控端准备

- **VNC**：安装 TightVNC/RealVNC/TigerVNC，并开放 5900
- **SSH**：安装 OpenSSH Server，并开放 22
- **RDP**：Windows 专业版/企业版开启远程桌面，并开放 3389

更完整步骤请看：

- `功能介绍/远程桌面控制.md` 的“观察端和被控端的配置方法”章节

## 8. 常见问题（FAQ）

### 8.1 端口被占用（8080/9100）

- Windows：

```powershell
netstat -ano | findstr :8080
```

### 8.2 温度显示 0°C（Windows 常见）

Windows 读取 CPU 温度可能需要管理员权限或第三方硬件监控工具。

建议：

- 以管理员运行
- 或安装并运行 LibreHardwareMonitor/OpenHardwareMonitor（Agent 支持通过其 WMI 接口读取）

### 8.3 远程桌面连接失败

先做连通性检查：

```powershell
Test-NetConnection <目标IP> -Port 5900
Test-NetConnection <目标IP> -Port 22
Test-NetConnection <目标IP> -Port 3389
```

## 9. 学习路线（从“能用”到“会改”）

### 9.1 第 1 步：把功能跑通

- 登录注册
- 无人机管理新增/删除
- GPS 手动推送、围栏报警
- 电池上报、报警
- 飞行任务创建、阶段变化

### 9.2 第 2 步：读懂数据流

建议按这个顺序看代码：

- `project/main.go`
  - 服务端启动、路由注册、中间件、静态资源
- `project/internal/db/db.go`
  - 表结构与迁移（理解数据模型最快的方法）
- `project/internal/handlers/`
  - `drones.go`：无人机注册与关联创建
  - `gps.go`：GPS 上报、围栏、历史轨迹
  - `battery.go`：电池记录、报警、WS 推送
  - `flight.go`：任务状态机、WS 推送
  - `api.go`：VNC/SSH 代理、硬件推送、通用模块

### 9.3 第 3 步：对接真实无人机

需要替换/对接（当前 Agent 里是模拟的）：

- GPS 数据：对接 GNSS/RTK/飞控融合定位
- 电池数据：对接 BMS/飞控电源模块
- 飞行阶段：对接飞控模式/任务执行状态（例如 MAVLink）

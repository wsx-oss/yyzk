### 1.2 软件环境

**地面站电脑**：Go 1.20+、Git、Chrome 浏览器、QGroundControl（飞控校准用）

**机载电脑（树莓派）**：
- 树莓派 OS 64-bit
- Go（编译 Agent）
- SSH Server：`sudo apt install openssh-server`
- VNC Server：`sudo apt install tigervnc-standalone-server`
- mavproxy（MAVLink 串口转 UDP）：`pip3 install MAVProxy`

### 1.3 飞控基础知识（5 分钟速通）

```
遥控器 ──无线──→ 接收机 ──→ 飞控(Pixhawk) ←── GPS模块
                                │
                          USB/串口 ↓
                          机载电脑(树莓派) ──Wi-Fi──→ 地面站电脑 → 浏览器Web平台
```

- **飞控**：无人机核心处理器，运行 ArduPilot/PX4 固件
- **MAVLink**：飞控通信协议，传输 GPS、电池、飞行模式等数据
- **NMEA**：GPS 模块输出的文本协议（`$GPGGA,...`）
- **QGroundControl**：飞控配套软件，烧固件、校准传感器用

### 1.4 法规与安全须知（必读）

> ⚠️ 不遵守可能违法或造成人身伤害！

**法规要点**：
1. 250g 以上无人机必须在 [UOM平台](https://uom.caac.gov.cn) 实名登记
2. 轻型(250g~4kg)需通过操控员考试或在适飞空域飞行；小型(4~25kg)需 CAAC 执照
3. 机场周边、军事区、人口密集区 **严禁飞行**；限高 120m

**安全操作**：
1. 选择 50m×50m 以上空旷场地，远离人群/电线/树木
2. 风速 < 4 级，无雨雪
3. **起飞前检查**：螺旋桨牢固 → GPS 3D Fix(≥10颗星) → 遥控器对频 → 飞控校准完成 → Agent 启动 → 地面站收到数据 → 设置返航高度(30m)+低电压保护(3.5V/cell)
4. **紧急处理**：遥控器始终优先 → RTL 一键返航 → failsafe 失控保护已配置

### 2.1 地面站部署（你的电脑）

```powershell
# 1. 拉取代码
git clone https://github.com/<你的仓库>/Intelligent-Tool-Human-Computer-Interaction-and-Remote-Control-System.git

# 2. 安装依赖（在 project/ 目录）

# 3. 创建 .env 文件（在 project/ 目录下）
```

`.env` 文件内容：
```ini
SC_LISTEN_ADDR=:8080
SC_DB_PATH=app.db
SC_API_TOKEN=
# 智能航线规划（可选）
LLM_API_KEY=你的通义千问APIKey

LLM_MODEL=qwen-plus
# 高德地图（可选）
```

```powershell
# 4. 启动
go run .
# 看到 "listening on :8080" 即成功

# 5. 浏览器访问 http://127.0.0.1:8080/
```

### 2.2 无人机端 Agent 部署

#### 交叉编译（在 Windows 上编译给树莓派用）

```powershell
$env:GOOS="linux"; $env:GOARCH="arm64"; go build -o hw-agent ./cmd/agent/
scp hw-agent pi@<树莓派IP>:~/hw-agent
```

#### 启动 Agent（选一种方式）

**方式 1：MAVLink 飞控（真实飞行推荐）**
```bash
# 先用 mavproxy 转发飞控串口到 UDP
mavproxy.py --master=/dev/ttyACM0 --out=udp:127.0.0.1:14550

# 另一个终端启动 Agent
chmod +x ~/hw-agent
./hw-agent -server http://<地面站IP>:8080 -id drone-01 -mavlink :14550 -push-interval 2
```

**方式 2：NMEA GPS（无飞控，仅有 USB GPS）**
```bash
./hw-agent -server http://<地面站IP>:8080 -id drone-01 -gps-serial /dev/ttyUSB0 -push-interval 2
```

**方式 3：模拟模式（室内/无硬件）**
```bash
./hw-agent -server http://<地面站IP>:8080 -id drone-01 -simulate -sim-lat 30.27 -sim-lng 120.15 -push-interval 2
```

**方式 4：自动探测（什么都不指定，自动尝试 MAVLink → 降级模拟）**
```bash
./hw-agent -server http://<地面站IP>:8080 -id drone-01 -push-interval 2
```

> 💡 `-id drone-01` 必须和地面站注册的无人机名称一致！

#### 验证 Agent
```bash
curl http://<树莓派IP>:9100/health   # 期望 {"status":"ok"}
```

### 2.3 网络连通性验证

```
地面站(192.168.x.A:8080) ◄──Wi-Fi──► 机载电脑(192.168.x.B:9100/22/5900)
```

## 第四部分：故障排查手册

### Agent 推送数据不到地面站
```bash
ping <地面站IP>                              # 网络通吗？
curl http://<地面站IP>:8080/api/healthz       # 地面站活着吗？
# 检查 Agent 日志中的 "[Push] Failed" 错误信息
# 检查 -id 参数是否和地面站注册的无人机名称一致
```

### GPS 无定位（经纬度为 0）
- 确认在室外，GPS 天线朝天（首次定位需 1~5 分钟）
- QGC 检查卫星数量 ≥ 10 且 3D Fix
- MAVLink 模式：检查 mavproxy 是否正常运行
- NMEA 模式：`cat /dev/ttyUSB0` 看是否有 `$GPGGA` 输出

### MAVLink 无数据
```bash
ls /dev/ttyACM* /dev/ttyUSB*                 # 确认飞控USB已识别
sudo chmod 666 /dev/ttyACM0                  # 串口权限
# 确认 mavproxy 已启动且转发到 UDP 14550
```

### VNC/SSH 连不上
```powershell
Test-NetConnection <树莓派IP> -Port 22       # SSH
Test-NetConnection <树莓派IP> -Port 5900     # VNC
```
检查：服务是否启动(`systemctl status sshd`)、防火墙(`sudo ufw allow 22`)

### 温度显示 0°C
Windows 需管理员权限或安装 LibreHardwareMonitor；Linux 通常正常。

### 端口被占用
```powershell
netstat -ano | findstr :8080                 # Windows
```
```bash
sudo ss -tlnp | grep 8080                   # Linux
```

### B. MAVLink 解析的消息

| MSG ID | 名称 | 数据 |
|--------|------|------|
| 33 | GLOBAL_POSITION_INT | 经纬度、海拔、速度、航向 |
| 24 | GPS_RAW_INT | 备选 GPS 数据 |
| 0 | HEARTBEAT | 飞行模式→阶段映射 |
| 1 | SYS_STATUS | 电池电压/电流/电量 |
| 147 | BATTERY_STATUS | 电芯电压/温度/电量 |

### C. Agent 开机自启（systemd）

```bash
sudo tee /etc/systemd/system/hw-agent.service << 'EOF'
[Unit]
Description=HW Agent for Drone
After=network-online.target

[Service]
Type=simple
User=pi
ExecStart=/home/pi/hw-agent -server http://<地面站IP>:8080 -id drone-01 -mavlink :14550 -push-interval 2
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable hw-agent
sudo systemctl start hw-agent
```

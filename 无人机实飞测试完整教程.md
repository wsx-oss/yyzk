## 第零部分：无人机基础扫盲（零基础必读）

> 如果你从来没有接触过无人机，请先读完这一节。每个概念都会解释"是什么"、"长什么样"、"干什么用"。

### 0.1 一架无人机由哪些部件组成？

```
┌─────────────────── 无人机整体 ───────────────────┐
│                                                    │
│  [螺旋桨×4]  ← 提供升力，靠转速差异控制方向       │
│      │                                             │
│  [电机×4]    ← 驱动螺旋桨旋转                     │
│      │                                             │
│  [机架]      ← 骨架，所有部件装在上面               │
│      │                                             │
│  [电调(ESC)×4] ← 控制电机转速的小电路板             │
│      │                                             │
│  [飞控(Pixhawk)] ← ★核心大脑★ 下面重点介绍        │
│      │                                             │
│  [GPS模块]   ← 一个小圆盘或棒状天线，装在机身顶部  │
│      │                                             │

```

### 0.2 什么是飞控（Flight Controller）？

**飞控 = 无人机的大脑**。它是一块巴掌大的电路板（最常见的型号叫 **Pixhawk**），负责：

- 读取所有传感器（加速度计、陀螺仪、气压计、GPS）
- 根据传感器数据 + 遥控器指令，实时计算4个电机各自的转速
- 维持无人机稳定飞行，不会翻掉
- 通过 **MAVLink 协议** 把飞行数据（GPS坐标、电池电压、飞行模式等）发送出来

**你需要知道的**：

- 飞控上有一个 **USB-C 或 Micro-USB 接口**，用USB线连到机载电脑（树莓派）
- 飞控需要烧写固件（ArduPilot 或 PX4），用 **QGroundControl** 软件来烧写和校准
- 飞控上还有一个 **TELEM 串口**，也可以用来连接机载电脑（高级用法）

### 0.3 什么是 MAVLink？

MAVLink 是飞控和外部设备之间通信的**协议**（类比：HTTP 是浏览器和服务器之间的协议）。

飞控每秒会发出数十条 MAVLink "消息"，每条消息包含不同类型的数据：

- **HEARTBEAT (ID=0)**: 飞行模式（手动/自动/返航）、是否解锁
- **GLOBAL_POSITION_INT (ID=33)**: GPS 经纬度、海拔、速度、航向
- **SYS_STATUS (ID=1)**: 电池电压、电流、剩余电量百分比
- **BATTERY_STATUS (ID=147)**: 详细电池信息（电芯电压、温度）

我们的 **hw-agent** 程序就是监听这些 MAVLink 消息，解析后推送到你的 Web 平台。

### 0.4 什么是遥控器和接收机？

- **遥控器**（发射器）：你手持的带摇杆的设备，左右两个摇杆控制飞机的油门/航向/俯仰/横滚
- **接收机**（RX）：装在飞机上的小盒子，接收遥控器发出的无线信号，传给飞控
- **对频**：遥控器和接收机第一次配对的过程（类比蓝牙配对），只需做一次

> ⚠️ 遥控器是**最后的安全保障**！即使系统全部故障，只要遥控器能控制飞机，就能安全降落。

### 0.5 什么是 QGroundControl（QGC）？

QGroundControl 是一个**免费的飞控配套软件**，用于：

1. **烧写固件**：给飞控安装 ArduPilot/PX4 操作系统
2. **校准传感器**：加速度计、陀螺仪、磁力计、遥控器
3. **设置参数**：电池保护电压、返航高度、失控保护等
4. **地图监控**：实时看无人机在地图上的位置
5. **飞行日志**：下载和回放飞行记录

下载地址：https://docs.qgroundcontrol.com/master/en/qgc-user-guide/getting_started/download_and_install.html

### 0.6 整体数据流（核心理解）

**地面站中继架构**（本项目采用的确认架构）：

```
飞控(Pixhawk)
     │ MAVLink 数据（GPS/电池/飞行模式）
     │ 数传电台（空中端，如 SiK Radio）
     │ ──无线电链路──→
数传电台（地面端，USB接地面站电脑）
     ↓
地面站电脑（你的笔记本/台式机）
     │  运行 hw-agent 程序
     │  读取串口 MAVLink → 解析 → HTTP POST
     ↓
云端服务器（独立主机或本机）
     │  运行后端 Go 服务 (端口8080)
     │  接收推送 → 存数据库 → WebSocket 推前端
     ↓
浏览器（任意设备）
     │  地图显示、电池监控、飞行任务、远程桌面等
```

> 💡 **可选**：若无数传电台，也可在无人机上加装机载电脑（树莓派），通过 USB 直连飞控读取 MAVLink，再经 Wi-Fi/4G 推送到云端。两种方式 hw-agent 命令行参数相同。

### 0.7 三种数据接入模式（你选一种）

| 模式                    | 需要什么          | 数据来源                 | 适合场景               |
| ----------------------- | ----------------- | ------------------------ | ---------------------- |
| **MAVLink 模式**  | 飞控+机载电脑     | 真实飞行数据             | 真正飞行测试           |
| **NMEA GPS 模式** | USB GPS 模块+电脑 | 真实GPS，无电池/飞行模式 | 低成本位置验证         |
| **模拟模式**      | 只需一台电脑      | 全部模拟数据             | 室内/无硬件/纯软件演示 |

### 1.2 软件环境

**地面站电脑**：Go 1.20+、Git、Chrome 浏览器、QGroundControl（飞控校准用）

> 💡 地面站电脑同时运行后端服务和 hw-agent，通过数传电台 USB 接口读取飞控 MAVLink 数据。无需额外的机载电脑。

### 1.3 飞控基础知识（5 分钟速通）

```
遥控器 ──无线──→ 接收机 ──→ 飞控(Pixhawk) ←── GPS模块
                                │
                          USB/串口 ↓
                          机载电脑(树莓派) ──Wi-Fi──→ 地面站电脑 → 浏览器Web平台
```

- **飞控**：无人机核心处理器，运行 ArduPilot/PX4 固件
- **MAVLink**：飞控通信协议，传输 GPS、电池、飞行模式等数据
- **NMEA**：GPS 模块输出的文本协议（`$GPGGA,...`）
- **QGroundControl**：飞控配套软件，烧固件、校准传感器用

### 1.4 飞控首次设置（QGroundControl 保姆级步骤）

> 这一步在室内完成，只需飞控+USB线+电脑，不需要飞。

**第 1 步：安装 QGroundControl（QGC）**

1. 下载安装：https://docs.qgroundcontrol.com/master/en/qgc-user-guide/getting_started/download_and_install.html
2. Windows/Mac/Linux 都有，直接装

**第 2 步：USB 连接飞控**

1. 用 USB 线把飞控（Pixhawk）连到你的笔记本
2. 打开 QGC，左上角应该显示"已连接"（如果没有，检查驱动）

**第 3 步：烧写固件（首次使用需要）**

1. QGC 左侧齿轮图标 → "固件"
2. 拔掉飞控 USB，点击"确定"
3. 重新插入 USB，QGC 会检测到新设备
4. 选择 **ArduPilot → CopterFirmware → 稳定版**，点击安装
5. 等待进度条走完（约 2~5 分钟），**过程中不要拔线！**

**第 4 步：校准传感器**（按顺序）

1. 齿轮 → "传感器" → **加速度计**：按提示把飞机放在6个方向（正面朝上、朝下、左、右、前、后），每个方向保持不动直到提示完成
2. **磁力计**：按提示把飞机在空中慢慢旋转（像画8字），直到校准完成
3. **陀螺仪**：把飞机放平，不要动，等待自动完成
4. **水平校准**：把飞机放在水平面上，点击校准

**第 5 步：遥控器校准**

1. 打开遥控器电源
2. 齿轮 → "遥控器" → 开始校准
3. 按提示移动每个摇杆到最大位置（上下左右），然后回中
4. 校准完成后可以在界面看到摇杆位置实时变化

**第 6 步：设置安全参数（非常重要！）**

1. 齿轮 → "安全" → **返航高度**：设置 30m（返航时的飞行高度）
2. **低电压保护**：设置 3.5V/cell（4S电池 = 14.0V 触发返航）
3. **失控保护（Failsafe）**：设置为 RTL（信号丢失时自动返航）
4. **地理围栏**：根据你的测试场地设置最大飞行范围

### 1.5 物理接线指南（怎么把线插对）

```
飞控(Pixhawk) 的接口说明（以 Pixhawk 4 为例）：

┌──────────────────────────────────┐
│  Pixhawk 4 飞控板               │
│                                  │
│  [USB-C] ← 连机载电脑/笔记本    │
│  [GPS]   ← 连 GPS 模块         │
│  [RC IN] ← 连接收机             │
│  [TELEM1]← 连数传(可选)         │
│  [TELEM2]← 备用串口             │
│  [POWER] ← 连电源模块           │
│  [MAIN OUT 1-8] ← 连电调        │
│  [I2C/SPI/CAN] ← 扩展接口       │
└──────────────────────────────────┘

连线步骤：
1. GPS 模块 ──排线──→ 飞控 [GPS] 接口（注意方向，有防反插设计）
2. 接收机 ──三线──→ 飞控 [RC IN] 接口（信号线注意正负极）
3. 电调×4 ──→ 飞控 [MAIN OUT 1~4]（对应电机顺序看飞控文档）
4. 电源模块 ──→ 飞控 [POWER]，电源模块另一端连电池XT60
5. USB 线 ──→ 飞控 [USB-C] → 机载电脑(树莓派) USB 口
```

> 💡 **树莓派供电**：树莓派需要独立供电！不能靠飞控USB供电。
> 方案1：用移动电源给树莓派供电（推荐，简单）
> 方案2：用 BEC 从无人机主电池分出 5V 给树莓派（高级）

### 1.6 法规与安全须知（必读）

> ⚠️ 不遵守可能违法或造成人身伤害！

**法规要点**：

1. 250g 以上无人机必须在 [UOM平台](https://uom.caac.gov.cn) 实名登记
2. 轻型(250g~4kg)需通过操控员考试或在适飞空域飞行；小型(4~25kg)需 CAAC 执照
3. 机场周边、军事区、人口密集区 **严禁飞行**；限高 120m

**安全操作**：

1. 选择 50m×50m 以上空旷场地，远离人群/电线/树木
2. 风速 < 4 级，无雨雪
3. **起飞前检查**：螺旋桨牢固 → GPS 3D Fix(≥10颗星) → 遥控器对频 → 飞控校准完成 → Agent 启动 → 地面站收到数据 → 设置返航高度(30m)+低电压保护(3.5V/cell)
4. **紧急处理**：遥控器始终优先 → RTL 一键返航 → failsafe 失控保护已配置

## 第二部分：系统部署

> **核心原则**：只需运行两个程序。
>
> - **地面站电脑** → 运行后端服务（`go run .`）
> - **树莓派机载电脑** → 运行 hw-agent（`./hw-agent ...`）
>   两台设备能互相通网即可，不要求同一局域网（ZeroTier 虚拟网也行）。

### 2.1 第一步：确认两台设备网络互通

无论用哪种组网方式（同 Wi-Fi / ZeroTier / 热点），先确认树莓派能访问地面站后端：

```bash
# 在树莓派上执行，<地面站IP> 换成实际 IP 或 ZeroTier 分配的虚拟 IP
curl http://<地面站IP>:8080/api/healthz
```

返回 `{"status":"ok"}` 则网络已通，可以继续。否则先排查网络/防火墙（见第四部分）。

> 💡 使用 ZeroTier 时：两台设备加入同一 Network 后，在 ZeroTier 控制台或用 `ip a` 查各自的虚拟 IP（通常是 `10.x.x.x`），把地面站的虚拟 IP 填到后续 `-server` 参数里即可。

### 2.2 第二步：启动地面站后端

在你的笔记本（地面站电脑）上执行：

```powershell
# project/ 目录下
go run .
```

看到 `listening on :8080` 即表示后端已就绪。然后浏览器访问 `http://127.0.0.1:8080/`，注册账号并登录。

> ⚠️ **关键**：`.env` 里 `SC_LISTEN_ADDR` 必须是 `:8080`（监听所有网卡），不能是 `127.0.0.1:8080`，否则树莓派无法连入。

**`.env` 参考配置：**

```ini
SC_LISTEN_ADDR=:8080
SC_DB_PATH=app.db
SC_API_TOKEN=
LLM_API_KEY=你的通义千问APIKey
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-plus
AMAP_KEY=你的高德Key
```

### 2.3 第三步：在平台注册无人机

后端启动后，在浏览器里：

1. 进入 **无人机管理** → 新增无人机
2. **名称**填 `drone-01`（记住这个名称，后面 hw-agent 要用）
3. **IP** 填树莓派的实际 IP（或 ZeroTier 虚拟 IP）
4. 协议选 `SSH`，端口 `22`，填好用户名密码
5. 保存 → 系统自动创建关联的 GPS 设备和远程控制设备

### 2.4 第四步：编译 hw-agent（在地面站电脑上）

在你的地面站电脑（`project/` 目录）上编译：

```powershell
# Windows
go build -o hw-agent.exe ./cmd/agent/
```

```bash
# Linux/macOS
go build -o hw-agent ./cmd/agent/
```

### 2.5 第五步：在地面站电脑上启动 hw-agent（数传电台连接）

将数传电台地面端插入地面站电脑 USB 口，飞控的数传电台空中端已提前安装在无人机上。

**Windows（数传识别为 COM 口）**：

```powershell
.\hw-agent.exe -server http://127.0.0.1:8080 -id drone-01 -mavlink COM5 -push-interval 2
```

> `COM5` 替换为数传电台实际的 COM 端口号（设备管理器查看）。

**Linux（数传识别为 /dev/ttyUSB*）**：

```bash
./hw-agent -server http://127.0.0.1:8080 -id drone-01 -mavlink /dev/ttyUSB0 -push-interval 2
```

> `-id drone-01` **必须**和第三步在平台注册的无人机名称完全一致，否则数据不会关联到对应的无人机。

> 💡 如果使用机载电脑（树莓派）方案：在树莓派上用 `mavproxy` 将飞控串口转为 UDP，然后运行 `./hw-agent -server http://<云端IP>:8080 -id drone-01 -mavlink :14550 -push-interval 2`。

Agent 启动后你会看到类似输出：

```
[Agent] MAVLink listening on :14550
[Push] hardware ok (id=1)
[Push] gps ok (id=1)  ← 等飞控 GPS 有定位后才会出现非零坐标
[Push] battery ok (id=1)
```

### 2.6 第六步：验证数据已到平台

打开浏览器，确认以下四项：

| 页面                   | 期望结果                                         |
| ---------------------- | ------------------------------------------------ |
| **无人机管理**   | `drone-01` 状态变为 `online`                 |
| **GPS/位置信息** | 地图上出现标记点，状态"在线"                     |
| **电池监控**     | 显示实时电压和电量（GPS 推送成功后约 10 秒出现） |
| **硬件状态检测** | 显示树莓派的 CPU/内存/温度                       |

> ✅ 四项全部正常即表示真实无人机已成功接入系统，可以开始功能测试。

## 第三部分：功能测试（19 项）

> 按推荐顺序排列，前面的测试是后面的基础。

### 测试 1：用户注册与登录

1. 访问 `http://<地面站IP>:8080/` → 跳转注册页


### GPS 无定位（经纬度为 0）

- 确认在室外，GPS 天线朝天（首次定位需 1~5 分钟）
- QGC 检查卫星数量 ≥ 10 且 3D Fix
- MAVLink 模式：检查 mavproxy 是否正常运行
- NMEA 模式：`cat /dev/ttyUSB0` 看是否有 `$GPGGA` 输出

### MAVLink 无数据

```bash
ls /dev/ttyACM* /dev/ttyUSB*                 # 确认飞控USB已识别
sudo chmod 666 /dev/ttyACM0                  # 串口权限
# 确认 mavproxy 已启动且转发到 UDP 14550
```

### VNC/SSH 连不上

```powershell
Test-NetConnection <树莓派IP> -Port 22       # SSH
Test-NetConnection <树莓派IP> -Port 5900     # VNC
```

检查：服务是否启动(`systemctl status sshd`)、防火墙(`sudo ufw allow 22`)

### 温度显示 0°C

Windows 需管理员权限或安装 LibreHardwareMonitor；Linux 通常正常。

### 端口被占用

```powershell
netstat -ano | findstr :8080                 # Windows
```

```bash
sudo ss -tlnp | grep 8080                   # Linux
```

### B. MAVLink 解析的消息

| MSG ID | 名称                | 数据                     |
| ------ | ------------------- | ------------------------ |
| 33     | GLOBAL_POSITION_INT | 经纬度、海拔、速度、航向 |
| 24     | GPS_RAW_INT         | 备选 GPS 数据            |
| 0      | HEARTBEAT           | 飞行模式→阶段映射       |
| 1      | SYS_STATUS          | 电池电压/电流/电量       |
| 147    | BATTERY_STATUS      | 电芯电压/温度/电量       |

### C. Agent 开机自启（systemd）

```bash
sudo tee /etc/systemd/system/hw-agent.service << 'EOF'
[Unit]
Description=HW Agent for Drone
After=network-online.target

[Service]
Type=simple
User=pi
ExecStart=/home/pi/hw-agent -server http://<地面站IP>:8080 -id drone-01 -mavlink :14550 -push-interval 2
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable hw-agent
sudo systemctl start hw-agent
```

### E. 代码审计发现的 Bug 及修复记录

> 以下 Bug 已在本次代码审计中全部修复，确保真实无人机连接时不会出问题。

| # | 严重度         | 文件                                                     | Bug 描述                                                                                                                                       | 修复方式                                                                                                                        |
| - | -------------- | -------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| 1 | **严重** | `cmd/agent/provider_mavlink.go` `parseGPSRawInt`     | MAVLink GPS_RAW_INT (ID=24) 消息字段偏移量使用了 XML 定义顺序，但 MAVLink 线格式按字段类型大小降序排列。导致 GPS 坐标、fix_type 等全部读错位置 | 修正所有字段偏移量：fix_type→offset28, lat→offset8, lon→offset12, alt→offset16, eph→offset20, vel→offset24, cog→offset26 |
| 2 | **严重** | `cmd/agent/provider_mavlink.go` `parseBatteryStatus` | MAVLink BATTERY_STATUS (ID=147) 消息同样使用了错误的 XML 顺序偏移量。导致电芯电压、温度、剩余电量全部读错                                      | 修正：voltages→offset8, temperature→offset28, current→offset30, remaining→offset35；最小载荷检查 42→36                     |
| 3 | **中等** | `internal/handlers/battery.go` `BatteryPushByAgent`  | 电池电量 level=-1（NMEA 模式下表示"未知"）时，条件 `level <= 10` 为 true，触发虚假的"电量严重不足"报警和 critical 告警                       | 添加 `level >= 0` 前置条件到状态判断和报警生成逻辑                                                                            |
| 4 | **中等** | `internal/handlers/battery.go` `BatteryReport`       | 同 Bug 3，手动上报接口也有相同的 level=-1 误报问题                                                                                             | 同上修复                                                                                                                        |

**影响说明**：

- Bug 1-2 只影响 **MAVLink 模式**中使用 GPS_RAW_INT 和 BATTERY_STATUS 消息的场景。GLOBAL_POSITION_INT、SYS_STATUS、HEARTBEAT 的偏移量正确（因为这些消息的字段碰巧按大小排列）
- Bug 3-4 影响所有使用 **NMEA GPS 模式** 的部署（因为 NMEA 不提供电池数据，默认 level=-1）

### F. 实验室现场快速排查清单

到了实验室，如果遇到问题，按以下顺序排查：

```
1. 网络通吗？
   └─ 树莓派和你的笔记本连同一个 Wi-Fi → ping 测试

2. 后端活着吗？
   └─ curl http://<地面站IP>:8080/api/healthz → 期望 {"status":"ok"}

3. Agent 活着吗？
   └─ curl http://<树莓派IP>:9100/health → 期望 {"status":"ok"}
   └─ 查看 Agent 终端日志有没有 [Push] 相关输出

4. 飞控连上了吗？
   └─ ls /dev/ttyACM* → 应该有设备
   └─ Agent 日志有没有 "MAVLink connected" 或 heartbeat 信息

5. GPS 有定位吗？
   └─ 必须在室外！首次定位需 1-5 分钟
   └─ QGC 连飞控看卫星数量 ≥ 8 且显示 3D Fix
   └─ Agent 日志有没有 "GPS ready" 或非零经纬度

6. 数据到前端了吗？
   └─ 打开浏览器 F12 控制台 → 网络标签 → 过滤 WebSocket
   └─ 应该能看到 gps_update / battery_update 事件

7. 名称匹配吗？
   └─ Agent 的 -id 参数必须和无人机管理中注册的名称一致
   └─ 不一致会导致数据推到"新设备"而不是关联到已注册的无人机
```

# 云翼智控—— 基于强化学习与 RAG 的无人机群智能管控平台需求分析说明书

---

## 1 引言

### 1.1 项目背景

随着低空经济政策加速落地与无人机规模化应用场景的持续拓展，无人机集群在电力巡检、地理测绘、物流配送、应急搜救等行业的部署密度显著增长。然而，当前无人机管控体系普遍存在三类突出问题：一是**多机状态分散，缺乏统一监管**，无人机运行状态、位置与任务信息分散，难以实现多机统一可视化监测与集中管理；二是**异常处置依赖经验，智能分析不足**，面对飞行异常和告警事件，传统方式高度依赖人工经验，缺乏知识增强与智能辅助分析能力；三是**跨网接入复杂，远程运维成本高**，无人机部署环境复杂，跨网络数据接入、远程调试与运维排障流程繁琐，协同效率较低。

与此同时，《无人驾驶航空器飞行管理暂行条例》《民用无人驾驶航空器运行安全管理规则》持续实施，以及《民用无人驾驶航空器实名登记和激活要求》《民用无人驾驶航空器系统运行识别规范》将于 2026 年 5 月 1 日起施行，行业对集"空域合规管理、多机协同调度、智能决策辅助"于一体的综合管控平台需求日益增强。在此背景下，迫切需要一套融合前沿人工智能技术的智能化管控平台，实现对无人机集群全生命周期的统一管理、实时监控、智能决策与安全保障。

该需求说明书旨在定义云翼智控—— 基于强化学习与 RAG 的无人机群智能管控平台的功能、性能及系统设计要求，为项目的开发、测试与交付提供规范化指导。

### 1.2 项目目的

本项目旨在研发一套面向具身智能无人系统的人机交互与远程智控平台，构建"**边缘感知—云端计算—前端展示**"三层贯通架构，融合大语言模型（LLM）、检索增强生成（RAG）、案例推理（CBR）与强化学习（RL）等前沿智能技术，形成覆盖"**任务规划—执行监控—策略优化—安全回滚**"全链路的智能管控能力。

具体而言，本平台的设计目标涵盖以下核心层面：

**（1）统一管控能力。** 建立覆盖无人机自动发现与管理、状态监控、GPS 定位、电池健康、飞行任务调度、远程控制（SSH/VNC/RDP）、视频监控等全生命周期管理链路的统一管控入口，消除传统管控体系中多系统、多入口、数据割裂的问题。

**（2）智能决策能力。** 通过接入大语言模型（LLM）实现智能航线规划与多候选方案生成，结合禁飞区几何纠偏引擎（NFZ Correction）自动绕行禁飞区域；引入 CoT（Chain-of-Thought）思维链推理实现调度分析、故障诊断等场景的可解释智能决策；构建 BM25 RAG 检索增强生成引擎，从领域知识库中检索上下文辅助 AI 问答。

**（3）仿真验证与策略训练能力。** 构建多实例仿真引擎，支持批量创建最多 200 台仿真无人机实例，配套巡逻、巡检、配送、测绘、搜救五种任务模板，具备异常注入、碰撞规避、禁飞区围栏检测等仿真能力。基于 Q-Learning 算法建立强化学习训练链路，通过仿真环境中的策略训练与评估，形成可导出、可复用的决策策略，为 sim-to-real 策略迁移奠定基础。

**（4）高可用工程能力。** 构建统一并发框架（IO/CPU 双池调度、优先级队列、批写节流），实现 MySQL + Redis 双层数据架构，完成全量数据库持久化（消除本地文件依赖），支持自动备份与数据回滚，具备单二进制一体化部署能力，满足云端生产环境的高可用、高并发运行要求。

### 1.3 项目目标人群

**电力巡检行业**（输电线路/变电站无人机巡检管控）

**应急搜救机构**（灾害现场多机协同搜救调度）

**物流配送企业**（无人机末端配送任务管理与航线规划）

**农业植保服务**（农药喷洒无人机集群作业管理）

**高校科研团队**（无人机算法研究、仿真训练与教学实验平台）

**园区/校园安防**（固定区域无人机巡逻与安防监控管理）

## 2 功能性需求

### 2.1 核心功能

**（1）无人机管理。** 采用自动发现机制，无需手动添加。后端 TCP Gateway 根据各 4G 模块上报的 MAVLink 心跳数据自动识别并创建无人机记录，实时跟踪在线/离线状态。前端展示发现总数、在线数量、离线数量与 GPS 数据推送状态，支持名称搜索与状态筛选，用户可编辑无人机名称、围栏等属性信息。

**（2）GPS/位置信息。** 基于 Leaflet.js + 天地图实现实时位置展示与历史轨迹回放，支持电子围栏设置与越界告警，GPS 数据通过 WebSocket 实时推送，支持 Catmull-Rom 样条曲线轨迹平滑。

**（3）电池监控。** 实时监测电量、电压、温度与健康度，电量低于阈值自动告警，支持历史趋势图表展示，电池数据通过 WebSocket 实时推送至前端。

**（4）飞行任务管理。** 两步向导式任务创建流程（航线规划 + 任务配置），集成 LLM 智能航线规划与 NFZ 禁飞区自动绕行，支持动作点选/间隔两种执行模式（拍照、录像、投放、传感器采集），飞行阶段状态机自动跟踪六个阶段（待命→起飞→巡航→执行任务→返航→降落）。

**（5）禁飞区管理。** 支持圆形与多边形两类禁飞区创建与管理，提供 Leaflet 地图可视化叠加展示，与智能航线规划深度联动，LLM 生成的航线自动经 NFZ 纠偏引擎修正绕行。

**（6）AI 智能助手。** 右下角浮窗入口，支持多轮上下文对话、快捷指令、模块快速跳转与数据查询。深度集成 RAG 检索增强生成引擎（BM25 算法 + 20 篇领域知识文档），对话时自动检索相关知识片段注入 Prompt，提升领域问答准确度。

**（7）思维链决策分析。** 基于 CoT（Chain-of-Thought）推理机制，面向调度分析、故障诊断、路径优化、任务规划、安全评估、异常处理 6 类场景提供结构化多步骤智能推理，每步记录上下文、思考过程、结论与置信度评分。

**（8）仿真模拟引擎。** 支持批量创建最多 200 台仿真无人机实例，内置巡逻、巡检、配送、测绘、搜救 5 种任务模板。具备异常注入（低电量/偏航/失联/温度异常）、防碰撞检测与自动避让、禁飞区地理围栏实时检测（Ray-Casting 算法）、7×24 连续仿真运行、Leaflet 实时地图轨迹展示。

**（9）强化学习训练。** 基于 Tabular Q-Learning 算法，定义 11 维离散化状态空间与 7 种动作空间，采用多目标加权奖励函数。训练过程中动作真实作用于仿真环境，采集 (s, a, r, s') 四元组存入经验回放缓冲区。支持训练状态实时跟踪、历史持久化与策略 JSON 导出。

**（10）系统状态监控。** 实时采集 CPU、内存、磁盘、网络等系统资源指标，通过 WebSocket 每秒推送至前端，支持阈值自动告警（CPU ≥ 85%、内存 ≥ 85%、磁盘 ≥ 90%）。

**（11）备份与数据回滚。** 支持自动备份（24h 周期 + 启动首次触发）与手动备份，提供按记录恢复与上传 SQL 文件恢复两种方式，恢复前自动创建安全备份，全程异步进度追踪。备份 SQL 内容存储于数据库，消除本地文件依赖。

**（12）消息通知中心。** 右上角铃铛入口，实时显示未读通知计数，支持类型筛选（系统/告警/巡检/AI）与批量已读管理。AI 定时巡检模块定期检查设备健康、日志异常与安全风险，发现问题自动生成通知推送。

### 2.2 拓展功能

**（1）远程桌面控制。** 支持 VNC（noVNC 浏览器端）、SSH（xterm.js 终端）、RDP 三种远程连接协议接入已注册远程设备，VNC 与 SSH 支持浏览器内直接操作。

**（2）视频监控。** 集成 RTSP、MP4 等多格式机载视频流统一查看与播放能力，支持无人机视频流配置与管理。

**（3）语音交互记录。** 支持音频文件上传、下载、在线播放与删除管理，提供语音交互统计功能，音频二进制数据存储于数据库 BLOB 字段。

**（4）硬件状态检测。** 通过内嵌 Agent 模式采集硬件温度、风扇转速等指标，支持自动刷新、CSV 导出与 AI 智能诊断分析。

**（5）异常报警。** 全局异常报警管理平台，支持报警类型/优先级/状态分类管理，提供报警数据导入导出与统计分析功能。

**（6）维护操作日志。** 完整的操作审计记录系统，支持日志分类查询、导入导出及操作类型/结果统计分析。

**（7）软件更新管理。** 管理系统软件版本更新发布，支持自动更新与强制更新策略配置，提供更新记录管理能力。

**（8）数据同步状态。** 多设备数据库同步管理（覆盖 18 张业务表），支持全量同步与增量同步两种模式，提供同步任务状态追踪。

**（9）性能分析报告。** 系统性能分析模块，提供响应时间、吞吐量、错误率三维度分析，配套折线图、柱状图、饼图、散点图四种可视化图表。

**（10）并发任务监控。** 统一 Worker Pool 运行状态监控仪表盘，展示 IO/CPU 池进度条、任务完成/失败趋势图、任务组分布环形图、LLM RPM 限流监控卡片，支持自动刷新。

## 3 性能需求

**（1）并发管控能力。** 参数要求：同时管控无人机数量 ≥ 200 台，仿真引擎支持 200 台实例并行运行。测试方法：仿真引擎批量创建 200 台实例，监控系统资源占用率与任务执行成功率。

**（2）WebSocket 推送延迟。** 参数要求：GPS/电池/监控数据实时推送延迟 ≤ 200ms（局域网环境）。测试方法：客户端记录消息接收时间戳与服务端发送时间戳，统计端到端延迟分布。

**（3）API 响应时间。** 参数要求：常规 CRUD 接口平均响应时间 ≤ 100ms，LLM 智能规划接口响应时间 ≤ 60s。测试方法：使用压测工具对关键 API 端点发送并发请求，统计 P50/P95/P99 延迟。

**（4）数据库吞吐量。** 参数要求：GPS 批写吞吐 50 条/2s，电池批写吞吐 30 条/3s；数据库表总数 34 张。测试方法：通过 WriteBatcher 批量写入机制实测吞吐，监控数据库连接池使用率。

**（5）并发任务处理。** 参数要求：Worker Pool 任务队列容量 1024，IO 池 16 线程 + CPU 池 4 线程，支持 4 级优先级调度。测试方法：5000 并发提交压力测试，监控队列饱和率、任务完成率与平均耗时。

**（6）API 限流能力。** 参数要求：支持 3000 次/分钟/IP 请求限流（Redis 分布式限流 + 内存降级）。测试方法：模拟单 IP 高频请求，验证限流阈值与降级策略的准确性与稳定性。

**（7）备份恢复时间。** 参数要求：千级数据量基准下备份恢复时间 ≤ 30s。测试方法：触发手动备份与恢复操作，记录全流程耗时。

**（8）RL 策略收敛。** 参数要求：Q-Learning 策略训练在 ≤ 1000 episode 内初步收敛。测试方法：启动 RL 训练任务，监控累计奖励曲线与策略稳定性。

**（9）部署包体积。** 参数要求：单体部署二进制文件体积 ≤ 50 MB（Go embed 内嵌前端静态资源）。测试方法：执行 `go build` 编译后检查可执行文件大小。

**（10）系统可用性。** 参数要求：7×24 连续运行无崩溃，Panic 自动恢复，Redis 不可用自动降级为内存模式。测试方法：仿真引擎持续运行 72 小时，模拟 Redis 断连场景验证降级策略。

### 4.2 系统用例说明

**（1）管理无人机。** 参与者：用户。无人机端 4G 模块接入后，后端 TCP Gateway 根据 MAVLink 心跳中的设备标识自动发现并创建无人机记录，无需手动添加。用户可编辑无人机名称、围栏配置等属性信息，查看设备详情与在线状态。

**（2）管理禁飞区。** 参与者：用户。用户在地图上绘制圆形或多边形区域设定禁飞区，配置禁飞区名称、类型与生效规则，系统在航线规划与仿真运行中自动执行禁飞区合规检测与绕行。

**（3）创建飞行任务。** 参与者：用户。用户通过两步向导创建飞行任务：第一步设定起终点并调用 LLM 智能规划生成航线；第二步配置任务名称、关联无人机、动作编排（拍照/录像/投放/传感器采集）。

**（4）查看 GPS/电池。** 参与者：用户。用户在地图上查看无人机实时 GPS 位置与历史轨迹，同时监控电池电量、电压、温度与健康度变化趋势，数据通过 WebSocket 实时推送更新。

**（5）远程控制设备。** 参与者：用户。用户通过浏览器内嵌 VNC 或 SSH 终端远程接入已注册设备，执行远程诊断、参数调整或紧急干预操作。

**（6）使用 AI 助手。** 参与者：用户。用户通过右下角浮窗与 AI 助手进行多轮对话，咨询飞行操作规程、查询系统数据、获取快捷指令建议。AI 助手自动检索 RAG 知识库（20 篇领域文档）增强回答准确度。

**（7）启动仿真训练。** 参与者：用户。用户创建仿真批次（选择任务模板与实例数量），启动仿真运行后可注入异常、查看实时轨迹地图，并启动 RL 训练任务观察策略学习曲线与导出训练策略。

**（8）管理备份回滚。** 参与者：用户。用户可手动触发数据库备份、查看备份记录、从历史备份恢复数据或上传 SQL 文件恢复，系统恢复前自动创建安全备份并提供异步进度追踪。

**（9）监控与告警。** 参与者：用户。用户通过监控仪表盘实时查看 CPU/内存/磁盘/网络指标、并发任务池状态、LLM RPM 限流指标及服务健康状态；通过右上角铃铛查看系统告警与 AI 巡检通知，支持按类型筛选、点击跳转至相关模块、批量标记已读。

**（10）实时数据推送。** 参与者：系统。系统通过 WebSocket Hub 将 GPS 位置、电池状态、系统监控、仿真遥测等数据以低延迟方式实时推送至所有已连接的前端客户端。

**（11）AI 定时巡检。** 参与者：系统。系统按配置周期自动执行 AI 巡检任务，检查无人机在线状态、设备健康、日志异常、安全风险等，发现问题自动生成通知推送至消息中心（含 5 分钟/台冷却抑制）。

**（12）自动备份。** 参与者：系统。系统默认每 24 小时自动执行一次数据库备份，备份 SQL 内容直接存储于数据库 LONGTEXT 字段，启动时触发首次备份，支持开关控制。

**（13）碰撞规避检测。** 参与者：系统。仿真引擎在运行中对所有实例执行高频距离检测，当两架无人机距离低于安全阈值时自动触发航向/高度避让动作，防止碰撞发生。

**（14）禁飞区围栏检测。** 参与者：系统。仿真引擎每 tick 对所有运行中实例执行 Ray-Casting 点在多边形内检测，判定是否违反禁飞区约束，违规时记录地理围栏违规事件并触发 RL 惩罚信号。

**（15）阈值告警。** 参与者：系统。系统按配置间隔（默认 10s）采集 CPU/内存/磁盘指标，超过阈值时自动生成告警通知推送至前端，支持自定义阈值配置。

### 4.3 数据字典

**（1）无人机信息。** 数据类型：结构体。包含无人机唯一标识（ID）、名称、机型、Agent 标识（如 UAV-K9-005）、协议类型（MAVLink）、状态（在线/离线）、围栏配置、最近心跳时间、发现时间等。在线状态由后端根据 4G 连接的 MAVLink 心跳时间戳自动判定。

**（2）GPS 位置记录。** 数据类型：结构体。包含设备 ID、经度（longitude）、纬度（latitude）、高度（altitude）、速度（speed）、航向角（heading）、采集时间戳。

**（3）电池状态记录。** 数据类型：结构体。包含设备 ID、电量百分比（0-100%）、电压（V）、温度（℃）、健康度（%）、充放电状态、采集时间戳。

**（4）飞行任务。** 数据类型：结构体。包含任务 ID、名称、关联无人机 ID、航线坐标数组、飞行阶段（待命/起飞/巡航/执行/返航/降落）、动作列表、创建时间、状态。

**（5）航线规划方案。** 数据类型：结构体。包含方案 ID、起点坐标、终点坐标、途经点数组、总距离（km）、预计时长（min）、规划类型（LLM/手动）、禁飞区绕行标记。

**（6）禁飞区。** 数据类型：结构体。包含禁飞区 ID、名称、类型（圆形/多边形）、中心坐标或顶点坐标数组、半径（m，圆形适用）、生效状态。

**（7）仿真批次。** 数据类型：结构体。包含批次 ID、任务模板类型（巡逻/巡检/配送/测绘/搜救）、实例数量、运行状态（就绪/运行中/已停止）、创建时间。

**（8）仿真实例。** 数据类型：结构体。包含实例 ID、所属批次 ID、当前位置（经纬度+高度）、速度、航向、电量、飞行阶段、异常标签列表、航线航点数组。

**（9）RL 训练状态。** 数据类型：结构体。包含训练轮次（episode）、累计奖励（reward）、探索率（epsilon）、Q 表大小、经验缓冲区大小、训练耗时。

**（10）思维链记录。** 数据类型：结构体。包含链 ID、场景类型（6 类）、输入上下文、推理步骤数组（每步含思考/结论/置信度）、最终结论、创建时间。

**（11）告警记录。** 数据类型：结构体。包含告警 ID、类型（系统/设备/安全）、优先级（低/中/高/紧急）、标题、描述、状态（待处理/已处理/已忽略）、创建时间。

**（12）备份记录。** 数据类型：结构体。包含备份 ID、触发方式（自动/手动）、SQL 内容（LONGTEXT）、文件大小、状态（成功/失败）、创建时间。

**（13）通知消息。** 数据类型：结构体。包含通知 ID、类型（系统/告警/巡检/AI）、标题、内容、关联模块、已读状态、创建时间。

**（14）用户会话。** 数据类型：结构体。包含 Token 字符串、关联用户 ID、创建时间、过期时间（24h）、Redis 缓存键（`session:<token>`）。

**（15）知识库文档。** 数据类型：结构体。包含文档 ID、标题、内容（Markdown 格式）、来源文件名、导入时间，用于 RAG 检索引擎的 BM25 索引构建。

**（16）系统监控指标。** 数据类型：结构体。包含 CPU 使用率（%）、内存使用率（%）、磁盘使用率（%）、网络流入/流出速率（B/s）、采集时间戳。

### 4.4 硬件架构

本系统采用"**边缘感知—云端计算—前端展示**"三层硬件架构，各层设备与职责如下：

**边缘感知层（无人机端）：**

- 飞控系统：负责无人机底层飞行控制，通过 MAVLink 协议采集 GPS、电池、飞行阶段等遥测数据。
- 4G 通信模块：无人机端无需运行额外程序，飞控遥测数据经 4G LTE 模块直接上传至云端服务器，通信链路支持断线自动重连与心跳保活。
- 机载传感器：包括 GPS/GNSS 定位模块、惯性测量单元（IMU）、电池管理单元（BMS）与任务载荷（如云台相机），为平台提供多维度实时态势感知数据。

**云端计算层（后端服务）：**

- 应用服务器：运行 Go 后端单体服务（单二进制部署），内置 TCP Gateway 直接接收无人机 4G 上传的遥测数据并解析 MAVLink 协议，同时承载 API 网关、业务逻辑、并发调度、AI 推理等全部核心能力。
- MySQL 数据库：Aiven 云数据库服务，34 张业务表，连接池配置 25 最大连接 / 10 空闲连接。
- Redis 缓存：承担会话缓存、分布式限流、统计同步与通知缓存职能，连接池 20 连接 / 5 空闲。
- LLM 大模型服务：通过 API 调用外部大语言模型服务，支持 RPM 令牌桶限流（默认 30 次/分钟）。

**前端展示层（用户终端）：**

- 用户终端：标准 Web 浏览器（Chrome/Firefox/Edge），通过 HTTP/WebSocket 协议与云端服务交互。
- 显示设备：支持 PC、平板等多种终端设备访问，前端采用响应式布局适配不同屏幕尺寸。

关键组件和接口说明：4G TCP 上传接口（无人机→云端 TCP Gateway）、MAVLink 协议解析接口（TCP Gateway→业务逻辑）、RESTful API + WebSocket 接口（云端↔前端）、MySQL TCP 接口（应用↔数据库）、Redis TCP 接口（应用↔缓存）、LLM HTTPS API 接口（应用↔大模型服务）。

### 4.5 软件架构

本系统软件架构分为以下四个核心层次：

**（1）接入与路由层。** 基于 Go Gin 框架构建 HTTP 服务器，集成 CORS 跨域处理、Panic 自动恢复、结构化访问日志等中间件。路由层统一注册 RESTful API 端点与 WebSocket 升级端点，通过 Token 认证中间件与 IP 限流中间件保障接口安全。

**（2）业务逻辑层。** 22 个功能模块的核心业务逻辑实现，按领域划分为 handlers 包，涵盖无人机管理、飞行任务、AI 分析、仿真控制、备份恢复等全部 API 处理函数。业务层通过统一的数据库接口与并发框架接口与底层交互，保持业务逻辑的内聚性与可测试性。

**（3）智能引擎层。** 包含 LLM 大模型调用封装、CoT 思维链推理模块、RAG 检索增强生成引擎（BM25 算法 + 倒排索引）、仿真模拟引擎（含多实例管理、碰撞规避、地理围栏）、Q-Learning 强化学习训练器（含 11 维状态空间、经验回放）、高德地图 API 封装。

**（4）基础设施层。** 数据库连接管理（SQLite/MySQL 双驱动 + AdaptSQL 方言适配）、Redis 客户端封装（连接池 + 优雅降级）、统一并发框架（IO/CPU 双池 + 优先级队列 + WriteBatcher + Throttler + StatsCache）、WebSocket Hub 事件广播中心、系统监控指标采集。

**通信协议：** 前后端通过 HTTP RESTful API 交换业务数据，通过 WebSocket 协议实现实时数据推送（GPS/电池/监控/仿真遥测），数据格式统一为 JSON。

**优雅关闭机制：** 系统按序执行 flush batchers → stop caches → shutdown pool → shutdown HTTP，确保数据完整性与服务平滑退出。

### 4.6 非功能性需求

**安全性：**

- 用户密码采用 bcrypt 算法加密存储，Token 会话 24 小时自动过期。
- API 接口支持 Token 认证与基于 IP 的请求限流（Redis 分布式限流 + 内存降级）。
- SQL 注入防护（参数化查询）、输入验证、CORS 跨域安全策略。
- 备份恢复前自动创建安全备份，防止误操作导致数据丢失。

**可靠性：**

- 系统支持 7×24 连续运行，Panic 自动恢复机制保障服务不中断。
- Redis 不可用时所有功能自动降级为纯内存模式，系统正常运行不受影响。
- 数据库双驱动架构（SQLite/MySQL），支持从云端 MySQL 降级为本地 SQLite 运行。
- 全量数据库持久化，消除对本地文件系统的运行时依赖，支持无状态云端部署。

**可扩展性：**

- 模块化架构设计，22 个功能模块按领域独立开发，支持灵活扩展新模块。
- 统一并发框架（TaskPool）支持动态添加任务组与调整 Worker 数量。
- 数据库表结构支持在线迁移扩展，AdaptSQL 自动适配 SQL 方言。

**可部署性：**

- Go embed 将前端静态资源嵌入后端二进制文件，交付物为单一可执行文件（≤ 50 MB），部署门槛极低。
- 支持 Linux systemd 守护进程部署与 Windows 后台服务运行。
- 支持环境变量与 `.env` 文件两种配置方式，适配开发、测试与生产多环境。

**合规性：**

- 系统设计遵循《无人驾驶航空器飞行管理暂行条例》相关要求，内置禁飞区管控能力。
- CoT 思维链推理与操作日志提供完整的决策审计追溯能力。

## 5 产品使用

### 5.1 系统部署与启动

**环境准备：** 确保已安装 Go 1.25+ 运行环境，进入 `project/` 目录执行 `go mod tidy` 安装依赖。

**配置环境变量：** 编辑 `project/.env` 文件，配置数据库连接（MySQL DSN 或 SQLite 路径）、Redis 连接参数、LLM API Key、天地图 Key 等。关键配置项说明：

- `SC_DB_DRIVER`：默认值 `sqlite`，数据库驱动，可选 `sqlite` 或 `mysql`。
- `SC_LISTEN_ADDR`：默认值 `:8080`，服务监听地址与端口。
- `LLM_API_KEY`：默认值空，大模型 API Key，留空降级为直线规划。
- `REDIS_HOST`：默认值空，Redis 地址，留空禁用 Redis 功能。

**启动服务：** 在 `project/` 目录执行 `go run .`，启动成功后访问 `http://127.0.0.1:8080/`。系统启动时将自动执行数据库迁移（建表）、知识库导入（首次运行）、首次自动备份与后台巡检任务注册。

**编译部署：** 执行 `go build -o sc.exe .`（Windows）或 `go build -o sc .`（Linux）生成单体二进制文件，配合 `.env` 文件即可独立部署运行，无需额外安装 Node.js 或前端构建工具。

### 5.2 用户注册与登录

首次访问系统自动跳转至注册页面（`/app/register.html`），用户填写用户名与密码完成注册（密码以 bcrypt 加密存储）。注册成功后跳转至登录页面（`/app/login.html`），输入凭据完成登录。登录成功后系统颁发 Token 凭证（有效期 24 小时），后续所有 API 请求自动携带 Token 进行身份验证。Token 优先通过 Redis 缓存验证（命中直接返回），未命中时回退 MySQL 校验并回填缓存。

### 5.3 无人机管理与监控

登录后进入控制仪表盘（`/app/dashboard.html`），左侧导航栏可切换 22 个功能模块。

**无人机发现与管理：** 进入“无人机管理”模块，系统自动展示已发现的无人机列表（发现总数、在线数、离线数、GPS 数据推送状态）。无人机端 4G 模块接入后，后端自动识别并创建设备记录，无需手动添加。用户可通过名称搜索与状态筛选快速定位设备，点击“详情”查看单台无人机完整信息，点击“编辑”修改名称、围栏等配置。

**实时监控：** 在"GPS/位置信息"模块中查看无人机实时位置（天地图底图 + Leaflet 标记），在"电池监控"模块中查看电量/电压/温度趋势。两类数据均通过 WebSocket 实时推送，无需手动刷新。

**远程控制：** 在"远程桌面控制"模块中选择已注册设备，通过浏览器内嵌 VNC 客户端或 SSH 终端远程接入已注册设备进行运维操作。

### 5.4 飞行任务创建与执行

**第一步 — 航线规划：** 在"飞行任务管理"模块中进入任务创建向导。设定起点与终点坐标（默认郑州大学主校区），系统自动加载已配置的禁飞区。点击"智能规划"调用 LLM 生成多候选航线方案，航线自动经 NFZ 纠偏引擎修正绕行禁飞区域。用户对比评估后选择最优方案确认航线。

**第二步 — 任务配置：** 航线确认后进入任务配置页面，填写任务名称、关联无人机、选择动作类型（拍照/录像/投放/传感器采集）。动作执行支持两种模式：点选模式（在航线地图上点击选择执行位置，自动吸附最近航段）与间隔模式（设定起止范围与间隔距离，自动计算执行次数）。

**任务执行与追踪：** 任务创建后可查看飞行阶段状态机自动追踪的六个阶段流转，以及每个动作的执行状态（待执行/执行中/已完成）与完成进度。

### 5.5 AI 智能助手使用

点击页面右下角 AI 助手浮窗图标打开对话窗口。用户可直接输入自然语言问题（如"如何配置禁飞区？""当前系统有多少无人机在线？"），AI 助手自动从 RAG 知识库检索相关知识片段并结合 LLM 生成回答。支持快捷指令面板（常用操作一键触发）与模块快速跳转（输入模块名称自动导航）。对话窗口支持拖拽移动，位置在页面刷新后自动恢复。

### 5.6 仿真模拟与强化学习训练

**创建仿真批次：** 在"仿真模拟引擎"模块中选择任务模板（巡逻/巡检/配送/测绘/搜救）与实例数量，点击"创建批次"。系统根据任务模板自动生成航点与任务配置。

**运行与监控：** 启动仿真批次后，Leaflet 实时地图展示所有无人机实例的位置、航向与轨迹（Catmull-Rom 样条平滑 + 贝塞尔曲线航线）。可对单个实例注入异常（低电量/偏航/失联/温度异常），观察系统响应与碰撞规避行为。

**RL 训练：** 在仿真运行过程中启动强化学习训练，实时查看训练轮次、累计奖励曲线、探索率变化与 Q 表规模。训练收敛后可导出策略 JSON 文件，作为后续实机部署的先验决策参考。

## 6 测试标准

### 6.1 关键测试用例

**（1）并发管控压力测试。** 输入条件：仿真引擎同时创建并运行 200 台无人机实例，持续运行 72 小时。预期输出：所有实例正常运行，CPU 使用率 ≤ 80%，内存无泄漏，任务池队列不饱和。

**（2）WebSocket 推送延迟测试。** 输入条件：GPS 与电池数据以 200ms/500ms 节流间隔推送，50 个客户端同时连接。预期输出：端到端推送延迟 ≤ 200ms（局域网），消息无丢失、无乱序。

**（3）LLM 航线规划测试。** 输入条件：输入起终点坐标（跨越已配置禁飞区），调用智能规划接口。预期输出：生成至少 1 条合规航线方案，航线不穿越禁飞区，NFZ 纠偏引擎成功绕行并输出平滑曲线。

**（4）RAG 知识库问答测试。** 输入条件：通过 AI 助手提问“无人机低电量应如何处理？”。预期输出：AI 助手检索到电池安全相关知识片段，生成包含具体操作步骤的准确回答。

**（5）备份恢复完整性测试。** 输入条件：手动触发备份后修改部分数据，再从备份记录恢复。预期输出：恢复后数据与备份时刻完全一致，恢复前自动创建安全备份，恢复耗时 ≤ 30s。

**（6）碰撞规避测试。** 输入条件：两架仿真无人机航线交叉，运行中接近碰撞距离。预期输出：系统自动触发航向/高度避让，两机保持安全距离，无碰撞事件发生。

**（7）禁飞区围栏测试。** 输入条件：仿真实例航线穿越已配置禁飞区域。预期输出：Ray-Casting 检测正确识别违规，记录 GeofenceViolation 事件，RL 奖励函数输出惩罚信号。

**（8）并发任务池压力测试。** 输入条件：5000 个任务并发提交至 TaskPool（IO + CPU 混合）。预期输出：所有任务在超时时间内完成，无 Panic，队列使用率合理（< 80%），完成率 ≥ 99%。

**（9）Redis 降级测试。** 输入条件：运行中断开 Redis 连接。预期输出：系统自动降级为纯内存模式，会话验证/限流/统计缓存/通知缓存切换至内存实现，功能无异常。

**（10）认证安全测试。** 输入条件：使用过期 Token 或无效 Token 请求受保护接口。预期输出：接口返回 401 Unauthorized，不泄露任何业务数据。

**（11）RL 训练收敛测试。** 输入条件：启动 Q-Learning 训练 1000 轮。预期输出：累计奖励曲线呈上升趋势并趋于稳定，策略可正常导出为 JSON 格式。

**（12）限流阈值测试。** 输入条件：单 IP 以 4000 次/分钟频率请求 API。预期输出：前 3000 次正常响应，超限请求返回 429 Too Many Requests，Redis 与内存限流策略均生效。

## 7 测试风险与应对

**（1）云端 MySQL 高延迟。** 云数据库跨境网络 RTT 可能超过 200ms，导致任务池队列饱和。应对：已引入 `TrySubmit` 非阻塞提交机制，仿真遥测 DB 写入频率已降低，数据库连接池参数已优化（25 最大连接 / 10 空闲连接）。

**（2）LLM 外部服务不可用。** 大模型 API 可能出现超时、限流或服务中断。应对：已实现 RPM 令牌桶限流（默认 30 次/分钟），配置独立超时控制（默认 60s），未配置 API Key 或调用失败时自动降级为直线规划，不影响核心飞行任务流程。

**（3）Redis 缓存失效。** Redis 服务可能不可用或连接中断。应对：已建立全局降级策略，Redis 不可达时会话缓存、限流计数、统计同步、通知缓存全部自动降级为纯内存实现，系统功能与稳定性不受影响。

**（4）仿真实例恢复冻结。** 仿真引擎从快照恢复 Running 状态实例时可能出现 goroutine 未创建导致实例冻结。应对：已修复——恢复后先重置为 Stopped 再调用 Start()，确保 goroutine 正确创建。

**（5）通知风暴。** 无人机离线等异常场景下巡检与告警可能触发密集重复通知。应对：已引入 5 分钟/台冷却抑制机制，巡检前执行 DB ping + LLM HEAD 预检（离线时跳过巡检），前端通知铃铛增加离线检测与暂停轮询。

**（6）电池仿真精度异常。** 整数运算截断可能导致电量异常掉电（1%/s）。应对：已改用 `float64` 精确追踪电量，新增 idle 充电逻辑与满电自动重启机制，确保 7×24 连续仿真稳定运行。

**（7）前端 CDN 资源不可达。** 部分前端静态资源依赖外部 CDN 加载，弱网或离线环境下可能加载失败。应对：核心业务 JS/CSS 已内嵌，后续持续推进静态资源本地化；Go embed 机制确保核心页面离线可用。

**（8）跨数据库兼容问题。** SQLite 与 MySQL SQL 方言差异可能导致部分语句执行失败。应对：自研 AdaptSQL 兼容层自动转换 SQL 方言（`AUTOINCREMENT` → `AUTO_INCREMENT`、`INSERT OR REPLACE` → `REPLACE INTO` 等），已完成 34 张表全量审计与对齐验证。
